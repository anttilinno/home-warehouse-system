## E2E infra note (2026-06-13)

- **Full `bun run test:e2e` trips the auth rate limiter (20 req/min/IP, hardcoded `router.go:342`).** 10 login-bearing specs × 2 browsers + register/seeding POSTs exceed 20/min from one IP → back-half specs 429 and appear to "fail login". NOT a product bug. To run the full suite green: batch specs (e.g. by file) with a >60s cooldown, or restart the backend (clears the in-memory limiter) between batches. Each spec passes in isolation. A dev-only env override for the auth limit would make CI E2E runnable in one shot — candidate for Phase 17.
