# Requirements: v2.1 Feature Parity — Items, Loans & Scanning

**Milestone:** v2.1
**Status:** Active
**Last updated:** 2026-04-14

---

## v2.1 Requirements

### Items (ITEM)

- [x] **ITEM-01**: User can view a paginated list of inventory items (25/page) with search by name, SKU, and barcode
- [x] **ITEM-02**: User can filter items by category and location, and sort by name, SKU, created date, and updated date
- [x] **ITEM-03**: User can view item detail showing name, SKU, barcode, description, category, location, container, status, and notes
- [x] **ITEM-04**: User can create a new inventory item with name (required) and optional fields: SKU, barcode, description, category, location, container, status, notes
- [x] **ITEM-05**: User can edit an existing item's fields
- [x] **ITEM-06**: User can delete an item with a confirmation dialog
- [x] **ITEM-07**: User can archive and unarchive an item
- [x] **ITEM-08**: User can toggle archived items visibility in the items list

### Item Photos (PHOTO)

- [x] **PHOTO-01**: User can upload one or more photos to an item (JPEG/PNG/HEIC, auto-resized client-side, max 10 MB)
- [x] **PHOTO-02**: User can view item photos in a gallery viewer on the item detail page
- [x] **PHOTO-03**: User can delete an item photo
- [x] **PHOTO-04**: Item list and detail show the primary/thumbnail photo where available

### Borrowers (BORR)

- [x] **BORR-01**: User can view a list of borrowers with name and active loan count
- [x] **BORR-02**: User can create a new borrower with name (required) and optional email, phone, and notes
- [x] **BORR-03**: User can edit a borrower's details
- [x] **BORR-04**: User can delete a borrower (blocked with message if borrower has active loans)
- [x] **BORR-05**: User can view borrower detail showing active and historical loans

### Loans (LOAN)

- [x] **LOAN-01**: User can view loans in a tabbed list: Active, Overdue, and History
- [x] **LOAN-02**: User can create a loan by selecting an item and borrower, with optional due date and notes
- [x] **LOAN-03**: User can mark an active loan as returned
- [x] **LOAN-04**: User can edit a non-returned loan's due date and notes
- [x] **LOAN-05**: Item detail page shows the item's active loan and loan history
- [x] **LOAN-06**: Borrower detail page shows the borrower's active loans and loan history

### Taxonomy: Categories, Locations & Containers (TAX)

- [x] **TAX-01**: User can view categories in a hierarchical tree view with child indentation
- [x] **TAX-02**: User can create a category with name (required), optional parent category, and optional description
- [x] **TAX-03**: User can edit a category's name, parent, and description
- [x] **TAX-04**: User can archive/delete a category; UI shows usage count and warns if items are assigned
- [x] **TAX-05**: User can view locations in a hierarchical tree view with child indentation
- [x] **TAX-06**: User can create a location with name (required), optional parent location, and optional description
- [x] **TAX-07**: User can edit a location's name, parent, and description
- [x] **TAX-08**: User can archive/delete a location; UI shows usage count and warns if items are assigned
- [x] **TAX-09**: User can view containers grouped by location
- [x] **TAX-10**: User can create a container with name (required), location (required), and optional description
- [x] **TAX-11**: User can edit a container's name, location, and description
- [x] **TAX-12**: User can archive/delete a container; UI shows usage count and warns if items are assigned

### Navigation (NAV)

- [x] **NAV-01**: Retro sidebar navigation includes links to Items, Loans, Borrowers, and Taxonomy sections
- [x] **NAV-02**: Dashboard quick-access action cards link to Items list and Loans list (completing v2.0 wiring)

## Future Requirements (v2.2+)

- Barcode scanning — single-route `/scan`, QR+UPC/EAN/Code128 lookup, "not found → create" overlay, flashlight, manual fallback
- Offline/PWA support for frontend2
- Quick capture flow in frontend2
- Bulk item operations (bulk delete, bulk archive, bulk export)
- Saved filters / persistent search state
- CSV import/export in frontend2
- SSE real-time updates in frontend2
- Declutter assistant in frontend2

---

## Out of Scope (v2.1)

- Offline/PWA/IndexedDB — online-only, consistent with v2.0 approach
- Barcode scanning — deferred to v2.2
- Reminders and recurring loans — complexity not warranted for lean parity
- Loan approval workflows — not in frontend1 base flow
- Borrower avatars — cosmetic, defer
- Drag-drop taxonomy reorder — complexity not warranted
- Merge categories/locations — edge case, defer
- Icons/colors on taxonomy nodes — cosmetic, defer
- External UPC database lookup — out of scope
- NFC scanning — limited device support
- Hardware (USB/Bluetooth) barcode scanners — camera sufficient
