## Spacing Scale

Inherited verbatim from `57-UI-SPEC.md`. Phase-specific applications:

| Token | Value | Phase 58 usage |
|-------|-------|----------------|
| xs | 4px | Tree expand-chevron → node-label gap; inline badge horizontal padding |
| sm | 8px | Tree row vertical padding; tab body top padding after tab bar |
| md | 16px | Slide-over panel inner horizontal padding; form field vertical rhythm; container-group (per-location) internal padding |
| lg | 24px | Slide-over panel `padding-block`; tab-content top padding; container-group header → first row gap |
| xl | 32px | Container-group → next-group vertical break; empty-state internal padding |
| 2xl | 48px | Empty-state icon → heading gap |
| 3xl | 64px | Empty-state outer vertical padding when tab body has zero nodes |

**Tree indentation:** each depth level adds **24px** of `padding-inline-start` to the node row (matches `lg`). Max observable depth is 5 — deeper is clipped visually with an ellipsis badge (`+N more`) and still navigable via expand. No spacing exceptions outside the 4-multiple scale.

**Slide-over panel width:**
- Desktop (≥1024px): `480px` fixed width, right-docked, `100vh` tall
- Tablet (≥640px, <1024px): `60vw`, min `360px`
- Mobile (<640px): `100vw`, full-screen takeover (retro-cream background, standard top-bar remains)

## Color

Inherited 60/30/10 from v2.0 / Phase 57.

| Role | Value | Phase 58 usage |
|------|-------|----------------|
| Dominant (60%) | `#F5F0E1` (`--color-retro-cream`) | Page background, slide-over panel background, tree row default background |
| Secondary (30%) | `#2A2A2A` (`--color-retro-charcoal`) | Text, thick borders on tree rows and panel edges, tab-bar underline |
| Accent (10%) | `#D4A017` (`--color-retro-amber`) | See reserved list below |
| Destructive | `#CC3333` (`--color-retro-red`) | `delete permanently` secondary link on hover/focus; hard-delete confirm button; 409 toast border |

**Accent reserved for (explicit list — nothing else):**
1. Primary CTA buttons: `+ NEW CATEGORY`, `+ NEW LOCATION`, `+ NEW CONTAINER`, and the slide-over submit button (`CREATE CATEGORY` / `CREATE LOCATION` / `CREATE CONTAINER` on create flow; `SAVE CATEGORY` / `SAVE LOCATION` / `SAVE CONTAINER` on edit flow)
2. `:focus-visible` outline ring on every focusable control (2px solid amber, 2px offset)
3. Active tab indicator in `RetroTabs` (bottom border fill)
4. Primary button in **archive-first** confirm dialog (`ARCHIVE CATEGORY` / `ARCHIVE LOCATION` / `ARCHIVE CONTAINER`)
5. Expanded state marker on tree rows (left-edge 2px amber rail while the row's slide-over is open — visually anchors which row is being edited)

**Not-accent palette usage in this phase:**
- `--color-retro-orange` `#E67E22` — warning badge only, ONE site: the archive dialog header strip reading `HIDES FROM PICKERS`

- `--color-retro-blue` `#3366AA` — info badge only (`ROOT` marker on top-level tree nodes when tree depth ≥ 2)
- `--color-retro-gray` `#8B8B8B` — archived-node text color; disabled Edit/Archive buttons when a row is already being edited
- `--color-retro-hazard-yellow` `#FFD700` — hazard stripes appear ONLY on the secondary hard-delete confirm dialog header (not on archive-first)

## Focal Point — `/taxonomy` Route

1. **Primary focal point:** the active tab's **primary CTA button** (`+ NEW CATEGORY` / `+ NEW LOCATION` / `+ NEW CONTAINER`), amber-filled, rendered in the top-right of the tab body header. It is the only amber-filled element above the fold in the empty/small-tree case and draws the eye first.
2. **Secondary focus:** the tree itself (Categories/Locations) or the first location-group (Containers). Rows use charcoal text on cream; the expanded-row amber rail (color rule #5) is the only accent inside the tree region.
3. When a slide-over panel is open, focus shifts to the panel: the panel's submit button (`CREATE {ENTITY}` or `SAVE {ENTITY}`) becomes the only amber-filled element on screen, the backdrop overlays the tree at 40% charcoal, and the tree's amber rail on the edited row persists to anchor context.

### Slide-over panel (create/edit)
- Slides in from the right with a 150ms ease-out transform (skipped when `prefers-reduced-motion`)
- Header: panel title (20px / 600 / UPPERCASE) + close `×` button top-right
- Body: `RetroFormField`-wrapped controls in a single column, 16px vertical rhythm
- Footer: sticky bottom bar with `← BACK` (charcoal outline) on the left and the entity-specific submit button (amber fill) on the right — `CREATE CATEGORY` / `CREATE LOCATION` / `CREATE CONTAINER` on create flow, `SAVE CATEGORY` / `SAVE LOCATION` / `SAVE CONTAINER` on edit flow; 24px top padding
- Backdrop: charcoal at 40% opacity; click-outside cancels (with unsaved-changes guard — see Interaction Contracts)

## Interaction Contracts

- **Tab persistence:** active tab stored in URL hash (`#categories` | `#locations` | `#containers`). Default `#categories` when absent. Switching tabs updates the hash without full navigation; browser back/forward replays tab state.
- **Slide-over unsaved-changes guard:** if `formState.isDirty`, clicking `← BACK`, the close `×`, the backdrop, or pressing Esc opens a nested mini-confirm dialog: title `DISCARD CHANGES?`, body `Your edits to {Name} will be lost.`, escape `← BACK`, destructive `DISCARD` (red). No guard when pristine.
- **Tree expand/collapse:** per-row state is local component state. Expand/collapse is keyboard-accessible via Right/Left arrows on a focused tree row. Enter on a row opens the edit slide-over. Home/End jump to first/last visible row.
- **Archived-node rendering:** archived nodes appear inline within the tree structure (preserving hierarchy) but are collapsed under a single `ARCHIVED ({N})` section at the BOTTOM of the tree when the `Show archived` toggle is off. Toggling on interleaves them at their natural position with `line-through` + `gray` styling + `ARCHIVED` badge.
- **short_code auto-fill:** 300ms debounced derivation — `uppercase(first_3_chars_of_name) + '-' + auto_suffix`. Suffix is generated client-side (e.g., 3-digit random). User typing in the short_code field at any point severs the auto-link for the lifetime of that form instance.
- **Focus ring:** every focusable control gets `outline: 2px solid var(--color-retro-amber); outline-offset: 2px` on `:focus-visible`. Never remove outline without replacement.
- **Keyboard for RetroCombobox parent picker:** inherits Phase 57 contract (Up/Down/Home/End/Enter/Esc/Typeahead + `role="combobox"` + `aria-expanded`).
- **Slide-over ARIA:** `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing at panel title. Focus trap active. First focusable element receives focus on open (the first form input).
- **Icon-only buttons:** every icon-only control carries `aria-label` (see Row-level actions table).
- **Loading:** save-in-flight disables the submit button and swaps its label to `WORKING…` in mono (replacing `CREATE {ENTITY}` or `SAVE {ENTITY}` for the duration of the request). Row-level action buttons disable while their mutation is pending.
- **Cache invalidation after mutations:** each mutation invalidates the entity's `.all` query key (`categoryKeys.all`, `locationKeys.all`, `containerKeys.all`). Containers tab additionally invalidates `locationKeys.all` on any location mutation to refresh the filter picker.
- **Motion:** honor `prefers-reduced-motion` — slide-over transform, dialog fade/scale, toast slide all replaced with instant state swaps.
