# Phase 60 — UI Design Contract

> Visual and interaction contract for the Items section: paginated `/items` list with search/filter/sort/archived chip, slide-over create/edit panel, and `/items/:id` detail page with placeholder Photos and Loans sections. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

**Source of truth:** All design tokens inherit from `frontend2/src/styles/globals.css` (v2.0 retro system, locked) and the contracts of Phase 57 (primitives), Phase 58 (taxonomy archive-first/slide-over patterns), and Phase 59 (single-entity CRUD shape). Phase 60 introduces ONE new local feature component — `ItemsFilterBar` (filter chip + search input + category picker + sort picker) — composed entirely from existing retro primitives. No new global primitives. No new tokens.

**Upstream artifacts consumed:**
- `60-CONTEXT.md` — D-01 deferred fields (no location/container/status/notes), D-02 list params, D-03 DELETE handler, D-04 archive-first dialog, D-05 form fields (name/SKU/barcode/description/category), D-06 detail sections (header/core/Photos placeholder/Loans placeholder), D-07 list columns (Name, SKU, Category, Actions), D-08 filter chip for archived, D-09 filter bar
- `59-UI-SPEC.md` — slide-over panel layout, archive-first dialog pattern, row-action conventions, success/error toast shells, pagination URL state
- `58-UI-SPEC.md` — slide-over backdrop, archive-first dialog reuse, ARIA conventions
- `57-UI-SPEC.md` — typography scale (3 sizes / 2 weights), spacing scale, color 60/30/10, touch targets, RetroFormField/Combobox/Dialog/Pagination/EmptyState contracts
- `REQUIREMENTS.md` — ITEM-01..ITEM-08 acceptance criteria
- `frontend2/src/styles/globals.css` — `@theme` token block

## Focal Point — `/items` Route

1. **Primary focal point:** `+ NEW ITEM` button, amber-filled, rendered in the top-right of the list header. It is the only amber-filled element above the fold in the empty/small-list case and draws the eye first.
2. **Secondary focus:** the `ItemsFilterBar` immediately above the table — search input (mono placeholder, monospace caret), category dropdown, sort dropdown, "Show archived" chip. Charcoal text on cream; the chip's amber active state (when toggled on) is the only accent inside the bar.
3. **Tertiary focus:** the items table. Rows use charcoal text on cream; the active-row amber rail (color rule #4) is the only accent inside the table region; SKU column sets a vertical mono rhythm.
4. When a slide-over panel is open, focus shifts to the panel: the panel's submit button (`CREATE ITEM` or `SAVE ITEM`) becomes the only amber-filled element on screen, the backdrop overlays the table at 40% charcoal, and the table's amber rail on the edited row persists to anchor context.

## Focal Point — `/items/:id` Route

1. **Primary focal point:** the item name header at the top of the page (20px / 600 / UPPERCASE), with a 2px amber left-edge rail (color rule #5) and an `EDIT ITEM` button (amber fill) in the top-right — the button is the only amber-filled element above the fold.
2. **Secondary focus:** the `DETAILS` block (14px UPPERCASE label + SKU/barcode/description/category/timestamps), rendered in a `RetroPanel`. SKU and barcode rendered in mono; category resolved to its display name (or `—` if absent).
3. **Tertiary focus:** the `PHOTOS` and `LOANS` section headers with their placeholder `RetroEmptyState` bodies (Phase 61 wires Photos; Phase 62 wires Loans).

## Layout Contract

### List page shell — `/items`

```
┌────────────────────────────────────────────────────────────────────────────┐
│  ITEMS                                                     [+ NEW ITEM]    │
├────────────────────────────────────────────────────────────────────────────┤
│  ┌──── ItemsFilterBar ───────────────────────────────────────────────────┐ │
│  │ [🔍 Search name, SKU, or barcode…]  [CATEGORY ▾]  [SORT: NAME ▾]     │ │
│  │ [SHOW ARCHIVED · 0]                                                    │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│  ┌──── Table ────────────────────────────────────────────────────────────┐ │
│  │ NAME                  SKU                    CATEGORY        ACTIONS  │ │
│  ├───────────────────────────────────────────────────────────────────────┤ │
│  │ Cordless Drill        ITEM-1718-A4F2        Power Tools     [E][A]   │ │
│  │ Bench Vice            ITEM-1718-7B1C        Hand Tools      [E][A]   │ │
│  │ Spare Light Bulbs     ITEM-1718-9D03        —               [E][A]   │ │
│  │ [ARCHIVED] Old Saw    ITEM-1701-2C99        Hand Tools      [R][D]   │ │ (only with chip ON)
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│                  [← PREV  Page 1 of 4  NEXT →]                              │
└────────────────────────────────────────────────────────────────────────────┘
```

- **Page size:** 25 rows / page (ITEM-01); pagination control hidden when total ≤ 25
- **Row click target:** on mobile (<640px), tapping the row navigates to `/items/:id` (except when tap originates on an action button); on desktop, the name cell is a `<Link>` to the detail page and the rest of the row is passive
- **Action buttons on an ACTIVE row:** `EDIT` (pencil) + `ARCHIVE` (archive icon)
- **Action buttons on an ARCHIVED row:** `RESTORE` (undo icon) + `DELETE` (trash icon) — the `DELETE` button opens the **archive-first** dialog which, for archived rows, surfaces the hard-delete option as its primary path via the secondary `delete permanently` link
- **Empty table:**
  - Zero items, no archived either: replace table region with `RetroEmptyState` (see Copywriting)
  - Zero matches under active search/filter: replace rows with inline `RetroEmptyState` inside the table panel (heading `NO MATCHES`, body `No items match your filters. Clear them to see all items.`, action `CLEAR FILTERS` charcoal outline)
  - Zero active items, some archived exist (chip OFF): heading `NO ACTIVE ITEMS`, body `All items are currently archived. Toggle "Show archived" to view them.`, action `+ NEW ITEM` (amber)

### `ItemsFilterBar` (new local feature component)

A horizontal flex bar above the table. Composed entirely from existing retro primitives. Not a new global primitive.

Layout:
- Desktop (≥1024px): single row — `[search input flex-grow] [16px gap] [category dropdown 240px] [16px gap] [sort dropdown 200px] [16px gap] [show-archived chip auto-width]`
- Tablet (≥640, <1024px): two rows — row 1 `[search input full-width]`; row 2 `[category] [sort] [chip]` with 16px gaps
- Mobile (<640px): three rows — row 1 `[search input]`; row 2 `[category dropdown full-width] [16px below] [sort dropdown full-width]`; row 3 `[chip auto-width, left-aligned]`

Components used inside the bar:
| Slot | Primitive | Notes |
|------|-----------|-------|
| Search input | `RetroInput` with prepended search icon | `type="search"`, mono placeholder text, debounce 300ms (planner may tune); clears on `×` button inside the input |
| Category dropdown | `RetroCombobox` over `categoriesApi.list()` | Async option loading; first option is `All categories` (charcoal); selected category shown in trigger label |
| Sort dropdown | `RetroSelect` | Static options: `Name`, `SKU`, `Created`, `Updated`; trigger label `SORT: {value}` (uppercase value) |
| "Show archived" chip | inline component composed of `<button>` + retro border tokens | See "Filter chip" pattern below |

**Filter chip pattern** (reusable shape — Phase 60 introduces the convention):
- Off state: `border: var(--border-width-retro-thick) solid var(--color-retro-charcoal)`, `background: var(--color-retro-cream)`, `color: var(--color-retro-charcoal)`, label text 14px / 600 / UPPERCASE sans + count in mono
- On state: `border-color: var(--color-retro-amber)`, `color: var(--color-retro-amber)`, optional 2px inset amber dotted underline on the count
- Hover: 2-2-0 charcoal box-shadow (matches `--shadow-retro-raised`)
- Pressed: inset shadow (matches `--shadow-retro-pressed`)
- Focus-visible: 2px amber outline + 2px offset (standard focus ring)
- Padding: 8px horizontal, 8px vertical (mobile compresses to ensure 44px min-height)
- Icon: optional eye/eye-off icon on the leading edge

- **Absent optional fields** (no SKU is impossible — auto-generated; no barcode, no description, no category): render the label row with value `—` in `--color-retro-gray` (mono 16px for barcode; sans 16px for description/category) — DO NOT hide rows; the DETAILS block always renders all six rows.

- **Timestamps:** mono font, ISO-like format `YYYY-MM-DD HH:MM` (planner may switch to user's `regionalFormats` setting if trivial; otherwise default to ISO; no localised relative-time strings in Phase 60).

### Slide-over panel (create/edit)

Same shell as Phase 58/59. Phase 60 specifics:

- Header: panel title `NEW ITEM` or `EDIT ITEM` (20px / 600 / UPPERCASE) + close `×` button top-right
- Body: single-column `RetroFormField` stack, 16px vertical rhythm:
  1. Name (required) — `RetroInput`
  2. SKU (required field, auto-generated on open) — `RetroInput`, mono font, helper text `Auto-generated — editable`
  3. Barcode (optional) — `RetroInput`, mono font, `inputMode="text"`
  4. Description (optional) — `RetroTextarea`, 4 rows visible
  5. Category (optional) — `RetroCombobox` async-searching `categoriesApi.list()`
- Footer: sticky bottom bar — `← BACK` (charcoal outline) on the left, submit on the right (`CREATE ITEM` on create, `SAVE ITEM` on edit)
- Backdrop: charcoal at 40% opacity; click-outside cancels (with unsaved-changes guard — see Interaction Contracts)

### Form schema — Item

| Field | Control | Required | Validation |
|-------|---------|----------|------------|
| Name | `RetroFormField` + `RetroInput` | **yes** | min 1 char, max 200 chars |
| SKU | `RetroFormField` + `RetroInput` (mono) | **yes** | min 1 char, max 64 chars; pattern: `[A-Za-z0-9_-]+` (no whitespace); auto-prefilled with `ITEM-{base36 timestamp}-{4-char base36 random}` (planner finalises generator) — fully editable |
| Barcode | `RetroFormField` + `RetroInput` (mono) | no | if present, max 64 chars; pattern: `[A-Za-z0-9]+` (no whitespace) |
| Description | `RetroFormField` + `RetroTextarea` (4 rows) | no | max 2000 chars |
| Category | `RetroFormField` + `RetroCombobox` | no | if present, must be a valid category UUID returned by `categoriesApi.list()` |

## Interaction Contracts

- **List pagination:** `RetroPagination` at bottom of the table; default page size 25 (ITEM-01); page state stored in URL query `?page=N`; initial load defaults to page 1. Pagination only visible when total > 25.
- **Search input:** controlled, debounced; on each keystroke, after the debounce window the URL query `?q=…` updates and React Query refetches the list with the new `search` param. Empty input removes the query param. Clearing via the `×` button resets focus to the input.
- **Category filter:** URL query `?category=<uuid>`; switching to `All categories` removes the param. Combobox loads category list once via React Query (not on every keystroke); options filter client-side.
- **Sort:** URL query `?sort=<field>&dir=<asc|desc>`; default `?sort=name&dir=asc`. Sort dropdown changes the field; the small `↑`/`↓` icon button toggles direction.
- **Archived visibility chip:** URL query `?archived=1` when ON, removed when OFF. Off by default (CONTEXT D-08 — chip-not-checkbox; default hides archived). When toggled ON, archived rows appear interleaved with active rows in current sort order, each prefixed with an `ARCHIVED` mono badge in the NAME cell and rendered in `--color-retro-gray` with `line-through` on the name. (Phase 60 deviates from Phase 59's "always at bottom" rule — items honour the active sort order even when archived, because users sort items by recency or SKU and expect monotonic order.)
- **Filter clear:** the `CLEAR FILTERS` button in the no-matches empty state removes ALL of `q`, `category`, `archived` from the URL but preserves `page=1` and `sort`/`dir`.
- **Slide-over unsaved-changes guard:** if `formState.isDirty`, clicking `← BACK`, the close `×`, the backdrop, or pressing Esc opens a nested mini-confirm dialog: title `DISCARD CHANGES?`, body `Your edits to {Name} will be lost.` (use `New item` when in create mode and the name is blank), escape `← BACK`, destructive `DISCARD` (red). No guard when pristine.
- **SKU auto-generation:** on `panel.open("create")`, the form's `defaultValues.sku` is computed once at open time using the planner-finalised generator (suggested: `ITEM-${Date.now().toString(36).toUpperCase()}-${4 random base36 chars upper}`). On `panel.open("edit", item)`, `defaultValues.sku = item.sku` (no regeneration). The SKU field is always editable; users can replace the auto-generated value.
- **Archive-first dialog reuse:** the same dialog component opens for both archive (active row) and delete (archived row). In archived mode the primary button flips to `DELETE ITEM` (red) and the hazard-stripe header activates; in active mode it is `ARCHIVE ITEM` (amber) with the orange `HIDES FROM DEFAULT VIEW` strip.
- **Optimistic state:** create/edit/archive/restore/delete mutations invalidate `itemKeys.all` on success; use `useMutation` `onSuccess` without optimistic updates (match Phase 58/59 baseline). Hard-delete additionally removes the detail query (`itemKeys.detail(id)`) from cache and, if invoked from the detail page, navigates the user to `/items` after the success toast.
- **Detail page navigation:** `EDIT ITEM` button opens the slide-over panel in-place on the detail route (panel overlays the detail page); saving keeps the user on the detail page and refreshes the query. Cancelling returns to the detail page unchanged. `ARCHIVE` from the detail page opens the archive-first dialog inline; on success, the detail page re-renders with the archived header treatment (no navigation).
- **Focus ring:** every focusable control gets `outline: 2px solid var(--color-retro-amber); outline-offset: 2px` on `:focus-visible`. Never remove outline without replacement.
- **Slide-over ARIA:** `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing at panel title. Focus trap active. First focusable element receives focus on open (the Name input).
- **Filter bar ARIA:** the filter bar is wrapped in `<form role="search">` for the search input; the chip is `<button type="button" aria-pressed={archived ? "true" : "false"}>`; the sort direction toggle is `<button type="button" aria-pressed={dir === "desc"}>`.
- **Table keyboard navigation:** row focus with Tab cycles through name-link → EDIT → ARCHIVE (or RESTORE/DELETE). Enter on the name link navigates to detail. Enter on an action button invokes it.
- **Icon-only buttons:** every icon-only control carries `aria-label` (see Row-level action buttons table; sort direction toggle; chip leading icon).
- **Loading:**
  - Initial list load: `RetroPanel` skeleton — three placeholder rows with mono dashes in each cell, no shimmer (matches Phase 58/59 plain-loading approach)
  - Save-in-flight: disables the submit button and swaps its label to `WORKING…` in mono. Row-level action buttons disable while their mutation is pending (gray text, `cursor: not-allowed`).
- **Cache invalidation after mutations:** all item mutations invalidate `itemKeys.all`. Archive/restore/delete additionally invalidate `itemKeys.detail(id)` if a detail query is mounted. Category list is NOT re-fetched on item mutations.
- **Motion:** honor `prefers-reduced-motion` — slide-over transform, dialog fade/scale, toast slide, chip color transition all replaced with instant state swaps.
- **Phase 61 seam:** the `PHOTOS` section on the detail page renders a `RetroEmptyState` placeholder in Phase 60. The placeholder body copy (`Photos will appear here after Phase 61.`) is swapped for a photo-gallery view in Phase 61 — no structural markup change is required; the section header and container panel remain.
- **Phase 62 seam:** the `LOANS` section on the detail page renders a `RetroEmptyState` placeholder in Phase 60. Same swap rules as Photos seam.
