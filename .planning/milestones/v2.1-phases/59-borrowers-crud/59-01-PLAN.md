| Boundary | Description |
|----------|-------------|
| Browser → huma API | Untrusted input crosses here: workspace ID (from session middleware, not URL), borrower ID (UUID from URL param), archived query param (bool) |
| API → Postgres | sqlc-parameterised queries only; no string concatenation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-01 | Tampering | `ListBorrowers` SQL archived filter | mitigate | sqlc.narg produces a parameterised query; `$2` (limit), `$3` (offset) stay bound params; no string concat |
| T-59-02 | Authorization | POST /borrowers/{id}/archive — cross-workspace archive | mitigate | `svc.Archive` calls `svc.GetByID(ctx, id, workspaceID)` first, which returns ErrNotFound when the borrower does not belong to the workspace in session context |
| T-59-03 | Authorization | POST /borrowers/{id}/restore — cross-workspace restore | mitigate | Same mediation via `svc.Restore → GetByID(workspaceID)` |
| T-59-04 | Authorization | DELETE /borrowers/{id} — cross-workspace delete | mitigate | Same mediation via `svc.Delete → GetByID(workspaceID)` before HasActiveLoans check |
| T-59-05 | Information Disclosure | `borrower.archived`/`borrower.restored` event payloads | mitigate | Events contain only entity ID + workspace ID + user ID + display name; no PII (email/phone/notes) leaked into event streams |
| T-59-06 | Elevation of Privilege | Unauthenticated archive/restore/delete | mitigate | First line of each handler: `appMiddleware.GetWorkspaceID(ctx)` returns 401 when no workspace is in session context; `appMiddleware.GetAuthUser(ctx)` guards event publish path |
| T-59-07 | DoS | Enumerating IDs via DELETE to learn which exist | accept | Standard REST 404-vs-400 information leak is already present across all workspace endpoints; workspace-scoping prevents cross-tenant leakage, which is the real boundary |
| T-59-08 | Tampering | `archived=true` query param to view other workspaces' archived rows | mitigate | `ListBorrowers` SQL always includes `workspace_id = $1`; workspace comes from session middleware not URL |
| T-59-09 | Repudiation | No audit trail on archive/restore | mitigate | Broadcaster publishes `borrower.archived`/`restored` events with user_id + display name; this provides the audit signal consumed by downstream listeners |
