## Focal Point — `/borrowers` Route

1. **Primary focal point:** `+ NEW BORROWER` button, amber-filled, rendered in the top-right of the list header. It is the only amber-filled element above the fold in the empty/small-list case and draws the eye first.
2. **Secondary focus:** the borrower table. Rows use charcoal text on cream; the active-row amber rail (color rule #4) is the only accent inside the table region.
3. When a slide-over panel is open, focus shifts to the panel: the panel's submit button (`CREATE BORROWER` or `SAVE BORROWER`) becomes the only amber-filled element on screen, the backdrop overlays the table at 40% charcoal, and the table's amber rail on the edited row persists to anchor context.

## Focal Point — `/borrowers/:id` Route

1. **Primary focal point:** the borrower name header at the top of the page (20px / 600 / UPPERCASE), with a 2px amber left-edge rail (color rule #5) and an `EDIT BORROWER` button (amber fill) in the top-right — the button is the only amber-filled element above the fold.
2. **Secondary focus:** the `CONTACT` block (14px UPPERCASE label + email/phone/notes in mono), rendered in a `RetroPanel`.
3. **Tertiary focus:** the `ACTIVE LOANS` and `LOAN HISTORY` section headers with their placeholder `RetroEmptyState` bodies (Phase 62 wires real data).

## Layout Contract

### List page shell — `/borrowers`

```
┌────────────────────────────────────────────────────────────────┐
│  BORROWERS                                                     │
│  {0 ARCHIVED · Show archived} [toggle]       [+ NEW BORROWER]  │
├────────────────────────────────────────────────────────────────┤
│  ┌──── Table ────────────────────────────────────────────────┐ │
│  │ NAME           EMAIL              PHONE         ACTIONS   │ │
│  ├───────────────────────────────────────────────────────────┤ │
│  │ Alice Smith    alice@example.com  —             [E][A]    │ │
│  │ Bob Jones      —                  +372 555 0101 [E][A]    │ │
│  │ Carol White    carol@ex.com       +372 555 0202 [E][A]    │ │
│  │ [ARCHIVED] Dan Brown  —  —                      [R][D]    │ │ (only with Show archived on)
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  [pagination controls — RetroPagination, bottom-center]         │
└────────────────────────────────────────────────────────────────┘
```

- Row click target: on mobile (<640px), tapping the row navigates to `/borrowers/:id` (except when tap originates on an action button); on desktop, the name cell is a `<Link>` to the detail page and the rest of the row is passive
- Action buttons on an ACTIVE row: `EDIT` (pencil) + `ARCHIVE` (archive icon)
- Action buttons on an ARCHIVED row: `RESTORE` (undo icon) + `DELETE` (trash icon) — the `DELETE` button opens the **archive-first** dialog which, for archived rows, surfaces the hard-delete option as its primary path via the secondary `delete permanently` link
- Empty table (zero non-archived borrowers): replace table region with `RetroEmptyState` (see Copywriting)

### Slide-over panel (create/edit)

Same shell as Phase 58. Phase 59 specifics:

- Header: panel title `NEW BORROWER` or `EDIT BORROWER` (20px / 600 / UPPERCASE) + close `×` button top-right
- Body: single-column `RetroFormField` stack, 16px vertical rhythm:
  1. Name (required) — `RetroInput`
  2. Email (optional) — `RetroInput` with `type="email"`
  3. Phone (optional) — `RetroInput` with `type="tel"`, mono font variant
  4. Notes (optional) — `RetroTextarea`, 4 rows visible
- Footer: sticky bottom bar — `← BACK` (charcoal outline) on the left, submit on the right (`CREATE BORROWER` on create, `SAVE BORROWER` on edit)
- Backdrop: charcoal at 40% opacity; click-outside cancels (with unsaved-changes guard — see Interaction Contracts)

### Form schema — Borrower

| Field | Control | Required | Validation |
|-------|---------|----------|------------|
| Name | `RetroFormField` + `RetroInput` | **yes** | min 1 char, max 120 chars |
| Email | `RetroFormField` + `RetroInput` (`type="email"`) | no | if present, must match email regex; max 254 chars |
| Phone | `RetroFormField` + `RetroInput` (`type="tel"`, mono) | no | if present, max 40 chars; no format enforced (free-form) |

## Interaction Contracts

- **List pagination:** `RetroPagination` at bottom of the table; default page size 25, page-size picker deferred (CONTEXT does not mandate one). Page state stored in URL query `?page=N`; initial load defaults to page 1. Pagination only visible when total > 25.
- **Archived visibility toggle:** `Show archived ({N})` chip-toggle in list header. Off by default (CONTEXT D-03). Toggling on appends archived rows at the BOTTOM of the table (never interleaved with active), each prefixed with an `ARCHIVED` mono badge and rendered in `--color-retro-gray` with `line-through` on the name. Toggle state persists via URL query `?archived=1`.
- **Slide-over unsaved-changes guard:** if `formState.isDirty`, clicking `← BACK`, the close `×`, the backdrop, or pressing Esc opens a nested mini-confirm dialog: title `DISCARD CHANGES?`, body `Your edits to {Name} will be lost.` (use `New borrower` when in create mode and name is blank), escape `← BACK`, destructive `DISCARD` (red). No guard when pristine.
- **Archive-first dialog reuse:** the same dialog component opens for both archive (active row) and delete (archived row). In archived mode the primary button flips to `DELETE BORROWER` (red) and the hazard-stripe header activates; in active mode it is `ARCHIVE BORROWER` (amber) with the orange `HIDES FROM LOAN PICKERS` strip.
- **Optimistic state:** create/edit/archive/restore mutations invalidate `borrowerKeys.all` on success; use `useMutation` `onSuccess` without optimistic updates (match Phase 58 baseline). Hard-delete additionally removes the detail query (`borrowerKeys.detail(id)`) from cache.
- **Detail page navigation:** `EDIT BORROWER` button opens the slide-over panel in-place on the detail route (panel overlays the detail page); saving keeps the user on the detail page and refreshes the query. Cancelling returns to the detail page unchanged.
- **Focus ring:** every focusable control gets `outline: 2px solid var(--color-retro-amber); outline-offset: 2px` on `:focus-visible`. Never remove outline without replacement.
- **Slide-over ARIA:** `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing at panel title. Focus trap active. First focusable element receives focus on open (the Name input).
- **Table keyboard navigation:** row focus with Tab cycles through name-link → EDIT → ARCHIVE (or RESTORE/DELETE). Enter on the name link navigates to detail. Enter on an action button invokes it.
- **Icon-only buttons:** every icon-only control carries `aria-label` (see Row-level action buttons table).
- **Loading:** save-in-flight disables the submit button and swaps its label to `WORKING…` in mono. Row-level action buttons disable while their mutation is pending (gray text, `cursor: not-allowed`).
- **Cache invalidation after mutations:** all borrower mutations invalidate `borrowerKeys.all`. Archive/restore/delete additionally invalidate `borrowerKeys.detail(id)` if a detail query is mounted.
- **Motion:** honor `prefers-reduced-motion` — slide-over transform, dialog fade/scale, toast slide all replaced with instant state swaps.
- **Phase 62 seam:** the `ACTIVE LOANS` and `LOAN HISTORY` sections on the detail page render `RetroEmptyState` placeholders in Phase 59. The placeholder body copy (`Loan data will be available soon.`) is swapped for a loans-wired view in Phase 62 — no structural markup change is required; the section headers and container panels remain.
