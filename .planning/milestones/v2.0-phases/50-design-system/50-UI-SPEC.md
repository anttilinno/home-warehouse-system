# Phase 50 — UI Design Contract

> Visual and interaction contract for the retro design system component library. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (fully custom retro components) |
| Icon library | none (inline SVG or text glyphs per component) |
| Font (body) | system-ui, -apple-system, sans-serif (`--font-sans`) |
| Font (data/mono) | IBM Plex Mono, JetBrains Mono, ui-monospace, monospace (`--font-mono`) |

Source: STATE.md decision "Fully custom component library -- shadcn/ui fights retro aesthetic"

---

## Spacing Scale

Declared values (all multiples of 4, already defined in `globals.css` `@theme` block):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, close button offset from edge |
| sm | 8px | Compact element spacing, badge padding, input icon gap |
| md | 16px | Default element spacing, panel internal padding axis |
| lg | 24px | Section padding, panel body padding, toast container offset |
| xl | 32px | Layout gaps, demo page section spacing |
| 2xl | 48px | Major section breaks on demo page |
| 3xl | 64px | Page-level spacing (not used in component internals) |

Exceptions:
- Touch targets: interactive elements use `h-[44px]` minimum (11 x 4px) for mobile tap targets
- Tab height: `h-[36px]` (9 x 4px) -- smaller than buttons, consistent with file-folder tab aesthetic
- Close button: `w-[24px] h-[24px]` (6 x 4px) -- small decorative button in panel corner
- Toast vertical padding: `py-[12px]` (3 x 4px) -- midpoint between sm (8px) and md (16px), chosen for compact toast height without crowding the left-border accent strip

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `--font-sans` | Paragraph text, descriptions, toast messages |
| Label | 14px | 700 (bold) | 1.2 | `--font-sans` | Button text, tab labels, badge text -- always UPPERCASE |
| Data | 14px | 400 (regular) | 1.4 | `--font-mono` | Input values, table cell data, monospace content |
| Heading | 20px | 700 (bold) | 1.2 | `--font-sans` | Panel titles, dialog titles, demo page section headings |

Rules:
- All labels and button text render in `uppercase` via Tailwind `uppercase` utility
- Monospace font (`--font-mono`) is used exclusively for user-editable data and table cells
- No font size below 12px anywhere in the system
- Two weights only: 400 (regular) and 700 (bold) -- no semibold, no light

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `retro-cream` | #F5F0E1 | Panel backgrounds, card backgrounds, input backgrounds, page surfaces |
| Secondary (30%) | `retro-charcoal` | #2A2A2A | Page background behind panels, dark containers, demo page background |
| Accent (10%) | `retro-amber` | #D4A017 | Primary button hover, focus-visible ring, active tab indicator |
| Destructive | `retro-red` | #CC3333 | Danger button background, error messages, close (X) buttons, validation errors |
| Border | `retro-ink` | #1A1A1A | All thick borders on every component, text color on cream surfaces |
| Disabled | `retro-gray` | #8B8B8B | Inactive tab background, disabled button/input background, placeholder text |
| Success | `retro-green` | #4A7C4A | Success toast left border, success badge background |
| Info | `retro-blue` | #3366AA | Info toast left border, info badge background |
| Hazard | `retro-hazard-yellow` / `retro-hazard-black` | #FFD700 / #1A1A1A | Hazard stripe pattern only (decorative dividers, panel headers) |

Accent reserved for:
- Primary button hover state (`hover:bg-retro-amber`)
- Focus-visible ring on all interactive elements (`focus-visible:outline-retro-amber`)
- Active state indicators where amber highlight is needed

## Accessibility Contracts

Close buttons (RetroPanel, RetroDialog): must render with `aria-label="Close"` since they use a visual "X" glyph without visible text label.

Decorative icon slots (RetroInput icon prefix): icons placed in the prefix slot are decorative and must include `aria-hidden="true"` to prevent screen readers from announcing them.

---

## Interaction States (Global Rules)

Every interactive component must implement ALL of these states:

| State | Visual Treatment |
|-------|-----------------|
| default | Component at rest with `shadow-retro-raised` where applicable |
| hover | Background color shift (variant-specific), `cursor-pointer` |
| active/pressed | `shadow-retro-pressed` replaces raised shadow |
| disabled | `bg-retro-gray`, `cursor-not-allowed`, `shadow-none`, reduced opacity text |
| focus-visible | `outline-2 outline-offset-2 outline-retro-amber` (amber ring, 2px offset) |

Focus-visible must use `outline` (not `ring`) to avoid conflicting with thick borders.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Demo page heading | "RETRO COMPONENT LIBRARY" |
| Demo page subheading | "SYSTEM STATUS: ALL COMPONENTS OPERATIONAL" |
| Demo section labels | Component name in uppercase, e.g., "RETROBUTTON", "RETROPANEL" |
| Demo empty state (table) | "NO DATA LOADED" |
| Demo toast examples | Success: "OPERATION COMPLETE" / Error: "SYSTEM ERROR DETECTED" / Info: "NEW TRANSMISSION RECEIVED" |
| Demo dialog title | "CONFIRM ACTION" |
| Demo dialog body | "Are you sure you want to proceed with this operation?" |
| Demo dialog confirm | "CONFIRM" (primary button) |
| Demo dialog cancel | "ABORT" (neutral button) |
| Demo button labels | "EXECUTE", "ABORT MISSION", "STANDBY" |
| Demo badge labels | "ACTIVE", "OFFLINE", "WARNING", "CRITICAL", "NOMINAL" |
| Demo input placeholder | "ENTER DESIGNATION..." |
| Demo input error | "INVALID INPUT DETECTED" |

All demo copy uses industrial/military tone consistent with the BAM game aesthetic.

**Style justification:** Single-word and short-phrase military jargon CTA labels (CONFIRM, ABORT, EXECUTE, STANDBY) are intentional per the BAM retro industrial aesthetic. These labels are deliberately terse and imperative -- they mirror the command-style vocabulary of the game world rather than following conventional "friendly" UX copy patterns.

**Destructive action handling:** In the demo context, all destructive actions (close X buttons on panels/dialogs, dialog cancel, danger buttons) are safe-to-dismiss and do not require confirmation. They either close a UI element or reset a demo state with no persistent consequences. The demo dialog's confirm/abort flow demonstrates the confirmation pattern visually but does not guard real data.

## Demo Page Contract

The `/demo` route serves as the living style guide and visual validation target.

| Property | Value |
|----------|-------|
| Route | `/demo` (public, no auth required) |
| Background | `bg-retro-charcoal` full page |
| Layout | Single column, max-width 960px, centered |
| Section spacing | 48px (`2xl`) between component sections |
| Section format | Heading (component name) + HazardStripe divider + interactive examples |

**Primary focal point:** The RetroButton section at the top of the demo page is the primary focal point. It is the first component section rendered after the page heading, showcasing all three variants (neutral, primary, danger) and all interactive states side by side. This gives the viewer immediate confidence in the design system's core interaction model before scrolling to more complex components.

Each component section must demonstrate:
1. All variants (e.g., neutral/primary/danger for buttons)
2. All interactive states (hover, pressed, disabled, focus)
3. Edge cases (long text, empty content, error states)
