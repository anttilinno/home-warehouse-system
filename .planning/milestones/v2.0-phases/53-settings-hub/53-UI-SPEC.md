# Phase 53 — UI Design Contract

> Visual and interaction contract for the Settings Hub: hub-and-spoke navigation with eight subpages (Profile, Security, Appearance, Language, Formats, Notifications, Data). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (fully custom retro components from Phase 50) |
| Icon library | none (inline SVG or text glyphs) |
| Font (body) | system-ui, -apple-system, sans-serif (`--font-sans`) |
| Font (data/mono) | IBM Plex Mono, JetBrains Mono, ui-monospace, monospace (`--font-mono`) |

Source: STATE.md decision "Fully custom component library -- shadcn/ui fights retro aesthetic"; Phase 50 UI-SPEC.

---

## Spacing Scale

Declared values (all multiples of 4, inherited from Phase 50 / `globals.css` `@theme` block):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap between toggle option buttons |
| sm | 8px | Hub row vertical padding, gap between hub rows, gap between session rows, compact form field spacing |
| md | 16px | Default element spacing, hub row horizontal padding, subpage form field gaps |
| lg | 24px | Subpage content padding (via RetroPanel), section group spacing within subpages |
| xl | 32px | Gap between hub category panels, gap between major subpage sections |
| 2xl | 48px | Not used in settings internals |
| 3xl | 64px | Not used in settings internals |

Exceptions:
- Hub row height: 48px (`h-[48px]`) -- consistent with top bar height, comfortable tap target
- Avatar display: 80px x 80px (`w-[80px] h-[80px]`) -- 20 x 4px, large enough to see avatar detail in retro square
- Avatar initials font: 32px (`text-[32px]`) inside 80px square -- 8 x 4px, legible retro square
- Toggle option button height: 44px (`h-[44px]`) -- matches RetroButton touch target standard

---

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Heading | 20px | 700 (bold) | 1.2 | `--font-sans` | RetroPanel titles ("PROFILE", "SECURITY", "ACCOUNT", "PREFERENCES") |
| Body | 14px | 400 (regular) | 1.5 | `--font-sans` | Form field descriptions, session info text, preview values in hub rows, descriptive text in Data subpage |
| Label | 14px | 700 (bold) | 1.2 | `--font-sans` | Hub row labels, form field labels, section headings within subpages ("CHANGE PASSWORD", "ACTIVE SESSIONS") -- always UPPERCASE |
| Data | 14px | 400 (regular) | 1.4 | `--font-mono` | Input field values, session timestamps, format previews, IP addresses, provider emails |

Rules (inherited from Phase 50):
- All labels, button text, hub row labels, and section headings render in `uppercase` via Tailwind `uppercase` utility
- Monospace font (`--font-mono`) used for user-editable data in inputs and data display (timestamps, IPs, format previews)
- Two weights only: 400 (regular) and 700 (bold)
- No font size below 12px

### Profile Subpage (SET-02, D-03, D-04)

| Property | Value |

| Route | `/settings/profile` |
| Panel title | "PROFILE" |

**Avatar Section:**

| Property | Value |
|----------|-------|

### Appearance Subpage (SET-04, D-07)

| Property | Value |
|----------|-------|
| File location | `features/settings/AppearancePage.tsx` |
| Route | `/settings/appearance` |
| Panel title | "APPEARANCE" |

**Theme Toggle:**

Three-option toggle group (uses shared ToggleGroup component):

| Property | Value |
|----------|-------|
| Section label | "THEME" |
| Options | "LIGHT" (`light`), "DARK" (`dark`), "SYSTEM" (`system`) |
| Active option | `bg-retro-amber shadow-retro-pressed text-retro-ink` |
| Inactive option | `bg-retro-cream shadow-retro-raised text-retro-ink` |
| Save behavior | Immediate on click -- no explicit save button. Calls `PATCH /users/me/preferences` with `{ theme }`, then `refreshUser()`, then applies `data-theme` attribute on `<html>` |

Theme application logic:
- "light" -> `document.documentElement.setAttribute("data-theme", "light")`
- "dark" -> `document.documentElement.setAttribute("data-theme", "dark")`
- "system" -> read `window.matchMedia('(prefers-color-scheme: dark)').matches`, set `data-theme` to "dark" or "light" accordingly, add `change` event listener

Note: If `globals.css` does not yet define `[data-theme="dark"]` CSS variable overrides, the toggle saves the preference but has no visual effect. The control is functional and the preference persists for future dark theme implementation.

## Toggle Group Component Pattern

Multiple subpages use the same toggle group pattern (theme, language, date format, time format, number format, notification categories). Extract a shared component:

| Property | Value |
|----------|-------|
| File location | `features/settings/ToggleGroup.tsx` |

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Hub nav | Wrap hub row panels in `<nav aria-label="Settings navigation">` |
| Hub rows | Each `<Link>` has descriptive text via the label; no additional `aria-label` needed |
| Toggle groups | `role="radiogroup"` on container with `aria-label` matching section label; `role="radio"` + `aria-checked` on each option |
| Disabled toggles | `aria-disabled="true"` on notification category toggles when master is off |
| Avatar image | `alt="User avatar"` on `<img>` |
| Avatar fallback | `aria-label="User avatar"` on the initials container |
| File inputs | Hidden file inputs: `aria-label="Upload avatar image"` and `aria-label="Import workspace file"` |
| Delete dialog | Focus trap (native `<dialog>` handles this); focus moves to dialog content on open, returns to trigger on close |
| Session current badge | `<RetroBadge>` text "CURRENT" is readable by screen readers; no additional aria needed |
| Form fields | Each `<RetroInput>` receives a visible `<label>` element above it with `htmlFor` linking to the input |
| Form errors | `aria-describedby` on input linking to error message element; `role="alert"` on error text |
| Live format preview | `aria-live="polite"` on format preview text so screen readers announce changes |
| Password fields | `autocomplete="current-password"` and `autocomplete="new-password"` on respective fields |
| Back button | Standard button semantics; label "BACK" is sufficient |

## Interaction States

All interactive elements follow global rules from Phase 50:

| State | Visual Treatment |
|-------|-----------------|
| default | Component at rest with `shadow-retro-raised` where applicable |
| hover | Background shifts per component type; `cursor-pointer` |
| active/pressed | `shadow-retro-pressed` replaces raised shadow |
| disabled | `bg-retro-gray`, `cursor-not-allowed`, `shadow-none`, `opacity-50` on text |
| focus-visible | `outline-2 outline-offset-2 outline-retro-amber` |

Toggle-specific states:
- Active option: `bg-retro-amber shadow-retro-pressed` (always pressed-in look)
- Inactive option: `bg-retro-cream shadow-retro-raised` (resting state)
- Clicking an already-active option: no-op (no state change, no API call)

Hub row states:
- Hover: entire row shifts to `bg-retro-amber`
- Pressed: `shadow-retro-pressed`
- Focus-visible: amber outline ring
