## Spacing Scale

Declared values (all multiples of 4, inherited from Phase 50 / `globals.css` `@theme` block):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, activity feed line gap |
| md | 16px | Default element spacing, card internal padding, grid gaps |
| lg | 24px | Section padding, stat panel body padding, page content padding |
| xl | 32px | Gap between dashboard sections (stats row, activity, actions) |
| 2xl | 48px | Not used in dashboard internals |
| 3xl | 64px | Not used in dashboard internals |

Exceptions:
- Stat panel number: `text-[48px]` (12 x 4px) -- large display number for HUD stat value
- Activity feed line height: `leading-tight` (~1.25) -- compact terminal log lines

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Display | 48px | 700 (bold) | 1.0 | `--font-mono` | Stat panel numbers (e.g., "247") |
| Heading | 20px | 700 (bold) | 1.2 | `--font-sans` | Panel titles ("ACTIVITY LOG"), page heading ("DASHBOARD") |
| Body | 14px | 400 or 700 | 1.2 | `--font-sans` / `--font-mono` | Quick-action card labels (bold, sans, UPPERCASE), sidebar nav items (bold, sans, UPPERCASE), activity feed log lines (regular, mono, `leading-tight`), stub page messages (regular, mono) |
| Label | 12px | 700 (bold) | 1.2 | `--font-mono` | Stat panel labels ("ITEMS", "CATEGORIES", "LOCATIONS") -- always UPPERCASE, letter-spacing `tracking-widest` |

Rules (inherited from Phase 50):
- All labels, nav items, and button text render in `uppercase` via Tailwind `uppercase` utility
- Monospace font (`--font-mono`) for all data display: stat numbers, stat labels, activity feed lines
- Two weights only: 400 (regular) and 700 (bold)
- No font size below 12px

### Activity Feed (D-06, D-07, D-08)

Terminal-styled log of recent inventory actions, refreshed via SSE.

| Property | Value |
|----------|-------|
| File location | `features/dashboard/ActivityFeed.tsx` |
| Component | `<RetroPanel showHazardStripe title="ACTIVITY LOG">` with dark background override |
| Background | `bg-retro-charcoal` -- requires overriding RetroPanel's default `bg-retro-cream` via `!bg-retro-charcoal` (Tailwind important modifier) or adding a `variant` prop to RetroPanel |
| Title color | `text-retro-cream` (white-on-dark since panel is charcoal) |
| Text | `font-mono text-[14px] leading-tight text-retro-cream` |
| Line spacing | `space-y-xs` (4px between log lines) |
| Data source | `GET /workspaces/{id}/analytics/activity?limit=10` |
| Refresh trigger | SSE `EventSource` to `GET /api/workspaces/{id}/sse` with `withCredentials: true` |
| SSE scope | Dashboard component only -- open on mount, close on unmount |
| Max entries | 10 (API-limited) |

Log line format: `[HH:MM] ACTION entity_type: entity_name`

| Part | Formatting |
|------|-----------|
| Timestamp | Local time, 24-hour, `[HH:MM]` format via `Date.toLocaleTimeString("default", { hour: "2-digit", minute: "2-digit", hour12: false })` |
| Action | UPPERCASED (e.g., "CREATE", "UPDATE", "DELETE") |
| Entity type | Lowercase as returned from API |
| Entity name | As returned from API; omitted if null |

Examples:
- `[14:32] CREATE item: Hammer`
- `[09:15] UPDATE location: Garage`
- `[11:47] DELETE category`

Empty state: `> NO ACTIVITY YET` in `text-retro-gray font-mono`

## Interaction States

All interactive elements follow global rules from Phase 50:

| State | Visual Treatment |
|-------|-----------------|
| default | Component at rest with `shadow-retro-raised` where applicable |
| hover | Background shifts to `bg-retro-amber` for nav items; button hover per variant |
| active/pressed | `shadow-retro-pressed` replaces raised shadow |
| disabled | `bg-retro-gray`, `cursor-not-allowed`, `shadow-none` |
| focus-visible | `outline-2 outline-offset-2 outline-retro-amber` |

Quick-action cards: the `<Link>` wrapper should not apply its own focus styling -- focus-visible applies to the `<RetroButton>` inside. The card itself has no interactive states beyond the button.

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Stat panels | Each stat panel must have an accessible label: `aria-label="{label}: {value}"` on the panel container (e.g., `aria-label="Items: 247"`) |
| Activity feed | `<section aria-label="Activity log">` landmark wrapping the feed |
| Activity feed empty state | `role="status"` on the empty state message so screen readers announce it |
| Quick-action cards | Each `<Link>` must have an `aria-label` describing the destination (e.g., `aria-label="Add item"`) since the card contains redundant text in label and button |
| Sidebar new items | NavLink with `aria-current="page"` (provided by React Router NavLink) |
| Loading state | Stat panels show "---" as loading placeholder -- no spinner needed, no aria-busy required since the loading state is brief |

## RetroPanel Dark Variant Note

The activity feed requires a dark terminal look (`bg-retro-charcoal`) but `RetroPanel` has `bg-retro-cream` hardcoded in its class string (line 21 of RetroPanel.tsx: `bg-retro-cream border-retro-thick...`). The `className` prop is appended after the base classes, but Tailwind CSS 4 does not guarantee later classes override earlier ones in the same specificity tier.

Recommended approach: Use `!bg-retro-charcoal` (Tailwind important modifier) in the `className` prop. This avoids modifying the shared RetroPanel component for a single use case. If the important modifier does not work reliably, fall back to a plain `<div>` styled directly for the activity feed container.

The title text color must also be overridden to `text-retro-cream` since the panel title defaults to `text-retro-ink` (invisible on charcoal). This requires either the important modifier (`!text-retro-cream`) or passing a custom title element via `children` instead of the `title` prop.
