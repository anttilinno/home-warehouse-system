# Phase 51 — UI Design Contract

> Visual and interaction contract for the app layout shell: sidebar, top bar, mobile drawer, loading bar, and error boundary. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (fully custom retro components from Phase 50) |
| Icon library | none (inline SVG or text glyphs) |
| Font (body) | system-ui, -apple-system, sans-serif (`--font-sans`) |
| Font (data/mono) | IBM Plex Mono, JetBrains Mono, ui-monospace, monospace (`--font-mono`) |

Source: STATE.md decision "Fully custom component library -- shadcn/ui fights retro aesthetic"; Phase 50 UI-SPEC.

## Spacing Scale

Declared values (all multiples of 4, inherited from Phase 50 / `globals.css` `@theme` block):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, nav item icon-to-label gap |
| md | 16px | Default element spacing, sidebar internal padding, top bar horizontal padding |
| lg | 24px | Section padding, content area padding |
| xl | 32px | Layout gaps, sidebar width gutter |
| 2xl | 48px | Top bar height |
| 3xl | 64px | Page-level spacing (not used in layout internals) |

Exceptions:
- Sidebar width (desktop): 240px (60 x 4px) -- fixed width for nav panel
- Top bar height: 48px (`h-2xl`) -- matches 2xl spacing token
- Mobile hamburger button: 44px x 44px (`h-[44px] w-[44px]`) -- mobile tap target, matches RetroButton height
- Loading bar height: 4px (`h-[4px]`) -- thin progress indicator, matches xs token
- Mobile breakpoint: 768px (`md:` Tailwind breakpoint) -- below this, sidebar becomes drawer

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `--font-sans` | User name in top bar, sidebar placeholder text |
| Label | 14px | 700 (bold) | 1.2 | `--font-sans` | Nav item labels, logout button text -- always UPPERCASE |
| Heading | 20px | 700 (bold) | 1.2 | `--font-sans` | App title "HOME WAREHOUSE" in top bar, error boundary heading |

Rules (inherited from Phase 50):
- All labels and nav item text render in `uppercase` via Tailwind `uppercase` utility
- Monospace font (`--font-mono`) used only for error details in the error boundary
- Two weights only: 400 (regular) and 700 (bold)

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `retro-charcoal` | #2A2A2A | Page background behind all layout elements, error boundary background |
| Secondary (30%) | `retro-cream` | #F5F0E1 | Sidebar background, top bar background, content panel backgrounds |
| Accent (10%) | `retro-amber` | #D4A017 | Active nav item background, loading bar color, focus-visible ring |
| Destructive | `retro-red` | #CC3333 | Logout button (danger variant), error boundary accent |
| Border | `retro-ink` | #1A1A1A | All thick borders on sidebar, top bar, nav items, drawer |
| Disabled | `retro-gray` | #8B8B8B | Inactive nav items (future use) |

Accent reserved for:
- Active nav item background highlight (`bg-retro-amber` + `shadow-retro-pressed`)
- Route loading progress bar (`bg-retro-amber`)
- Focus-visible ring on interactive elements (`focus-visible:outline-retro-amber`)

Source: Phase 50 UI-SPEC color contract; CONTEXT.md D-05, D-09.

**Color application note:** This phase inverts the 60/30 ratio compared to Phase 50. In the design system demo page, cream panels sit on a charcoal background (charcoal dominant). The app layout continues this: charcoal is the page background (dominant), cream is used for the sidebar panel and top bar (secondary surfaces). This is consistent -- the overall viewport is charcoal, with cream used for structural UI panels.

## Layout Component Contracts

### AppShell (LAY-01, D-11)

The root authenticated layout. Renders sidebar + top bar + content area via React Router v7 `<Outlet>`.

| Property | Value |
|----------|-------|
| File location | `components/layout/AppShell.tsx` |
| Structure | Flex row: Sidebar (left, fixed width) + Main column (top bar + content) |
| Background | `bg-retro-charcoal` full viewport |
| Min height | `min-h-dvh` |

Route structure:
- AppShell is the `element` of a parent `<Route>` that wraps all authenticated child routes
- `/login`, `/auth/callback`, `/demo` remain outside AppShell (top-level routes)
- `RequireAuth` guard wraps the AppShell route (or is integrated into AppShell itself)

### Sidebar (LAY-01, D-04, D-05, D-06)

| Property | Value |
|----------|-------|
| Width (desktop) | 240px fixed |
| Background | `bg-retro-cream` |
| Border | 3px solid `retro-ink` on the right edge (`border-r-retro-thick border-retro-ink`) |
| Shadow | `shadow-retro-raised` |
| Height | Full viewport height (`h-dvh`), fixed position on desktop |
| Padding | 16px (`p-md`) |
| Visibility | Visible on `md:` breakpoint and above; hidden below `md:` |

Nav items (Phase 51 scope -- two items only):

| Item | Label | Route |
|------|-------|-------|
| 1 | "DASHBOARD" | `/` |
| 2 | "SETTINGS" | `/settings` |

Nav item styling:

| State | Classes |
|-------|---------|
| Default | `bg-retro-cream text-retro-ink border-retro-thick border-retro-ink shadow-retro-raised px-md py-sm font-bold uppercase text-[14px] w-full text-left cursor-pointer` |
| Active | `bg-retro-amber shadow-retro-pressed` (pressed-in look, per D-05) |
| Hover (inactive) | `hover:bg-retro-amber` (same amber tint on hover) |
| Focus-visible | `outline-2 outline-offset-2 outline-retro-amber` |

Nav items are rendered as `<NavLink>` from React Router, using the `isActive` prop to apply active styling.

Nav item spacing: 8px gap (`gap-sm`) between items in a vertical flex column.

### Top Bar (LAY-01, D-07, D-08)

| Property | Value |
|----------|-------|
| Height | 48px (`h-[48px]`) |
| Background | `bg-retro-cream` |
| Border | 3px solid `retro-ink` on the bottom edge (`border-b-retro-thick border-retro-ink`) |
| Shadow | `shadow-retro-raised` |
| Padding | 0 16px (`px-md`) |
| Layout | Flex row, `items-center`, `justify-between` |
| Position | Sticky top (`sticky top-0`) |
| Z-index | `z-30` (per Phase 50 z-index scale) |

Left section:
- App title: "HOME WAREHOUSE" in 20px bold uppercase `text-retro-ink`
- On mobile (below `md:`): hamburger button replaces or sits beside the title

Right section:
- User display name (`user.full_name`) in 14px regular `text-retro-ink`
- User avatar: 32px x 32px circle. If `user.avatar_url` is non-null, render an `<img>` with `rounded-full object-cover`. If null, render a circle with `bg-retro-charcoal text-retro-cream` showing first letter of `full_name` in 14px bold uppercase
- Logout button: `<RetroButton variant="neutral">` with label "LOGOUT". Uses `useAuth().logout()` on click. Per D-08 (Claude's discretion), neutral variant chosen -- it blends with the top bar cream background and avoids visual alarm fatigue from a permanent red button in the chrome.
- Gap between avatar and name: 8px (`gap-sm`). Gap between user info and logout: 16px (`gap-md`).

### Mobile Hamburger Button (D-01)

| Property | Value |
|----------|-------|
| Size | 44px x 44px |
| Background | `bg-retro-cream` |
| Border | 3px solid `retro-ink` |
| Shadow | `shadow-retro-raised`, `active:shadow-retro-pressed` |
| Icon | Text glyph hamburger icon or three horizontal lines (inline SVG, 3 bars, each 2px thick, `stroke-retro-ink`) |
| Visibility | Visible below `md:` breakpoint only (`md:hidden`) |
| Position | Left side of top bar, before app title |
| Focus-visible | `outline-2 outline-offset-2 outline-retro-amber` |
| Aria | `aria-label="Open navigation"` when closed, `aria-label="Close navigation"` when open |

Uses `<RetroButton>` component with custom sizing via className override.

### Mobile Drawer (D-01, D-02, D-03, LAY-02)

| Property | Value |
|----------|-------|
| Width | 240px (same as desktop sidebar) |
| Height | Full viewport (`h-dvh`) |
| Background | `bg-retro-cream` |
| Border | 3px solid `retro-ink` on the right edge |
| Shadow | `shadow-retro-raised` |
| Position | Fixed, `left-0 top-0`, `z-30` |
| Animation | Slide in from left: `transform transition-transform duration-200 ease-out`. Closed: `translate-x-[-100%]`. Open: `translate-x-0`. |
| Content | Identical nav items as desktop sidebar |

Backdrop:

| Property | Value |
|----------|-------|
| Background | `bg-black/50` |
| Position | Fixed, `inset-0`, `z-20` (behind drawer) |
| Behavior | Click anywhere on backdrop closes drawer (D-02) |
| Animation | Fade in: `transition-opacity duration-200`. Closed: `opacity-0 pointer-events-none`. Open: `opacity-100`. |

Drawer closes on:
1. Tapping the backdrop
2. Tapping a nav item (navigating)
3. Pressing Escape key
4. Pressing the hamburger button again

### Loading Bar (D-09, LAY-03)

| Property | Value |
|----------|-------|
| Height | 4px (`h-[4px]`) |
| Background | `bg-retro-amber` |
| Position | Fixed, `top-0 left-0`, full width, `z-40` (above top bar) |
| Animation | Width grows from 0% to 90% over ~300ms with `ease-out`, then holds. On route load complete, snaps to 100% and fades out over 200ms. |
| Visibility | Only visible during route transitions. Hidden by default. |
| Layout impact | None -- no layout shift (D-09) |

Implementation: state-driven CSS width + opacity transitions. A `useNavigation()` hook (React Router v7) or a custom `useRouteLoading()` hook tracks pending navigations.

### Error Boundary (D-10, LAY-03)

| Property | Value |
|----------|-------|
| File location | `components/layout/ErrorBoundary.tsx` |
| Background | `bg-retro-charcoal` full screen |
| Layout | Flex center, `min-h-dvh items-center justify-center p-lg` |
| Panel | `<RetroPanel showHazardStripe>` centered, `max-w-[480px] w-full` |
| Heading | "SYSTEM ERROR" in 20px bold uppercase `text-retro-ink` |
| Error message | Displayed in `font-mono text-[14px] text-retro-ink` inside a `bg-retro-charcoal text-retro-cream p-md border-retro-thick border-retro-ink` code block |
| Retry button | `<RetroButton variant="neutral">` with label "RETURN TO BASE", navigates to `/` or calls `error.reset()` |
| Spacing | 16px (`gap-md`) between heading, error block, and button |

The error boundary uses React's `ErrorBoundary` class component pattern (or React Router's `errorElement`). It catches uncaught errors in the component tree within AppShell.

## Copywriting Contract

| Element | Copy | i18n Key Pattern |
|---------|------|------------------|
| App title (top bar) | "HOME WAREHOUSE" | `layout.appTitle` |
| Nav: Dashboard | "DASHBOARD" | `layout.nav.dashboard` |
| Nav: Settings | "SETTINGS" | `layout.nav.settings` |
| Logout button | "LOGOUT" | `layout.logout` |
| Hamburger aria (open) | "Open navigation" | `layout.nav.open` |
| Hamburger aria (close) | "Close navigation" | `layout.nav.close` |
| Error heading | "SYSTEM ERROR" | `layout.error.heading` |
| Error body | "Something went wrong. Return to the home screen or reload the page to try again." | `layout.error.body` |
| Error retry | "RETURN TO BASE" | `layout.error.retry` |
| Loading aria | "Loading page" | `layout.loading` |

Destructive actions in this phase:
- **Logout**: Single-click action via RetroButton. No confirmation dialog -- logout is low-risk and easily reversible (user can log back in). The button label "LOGOUT" is clear about the action.

Empty states: Not applicable for this phase. The layout shell always renders structural UI (sidebar, top bar). Content area empty states are the responsibility of the child page components (Phase 52, 53).

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Sidebar nav | `<nav>` landmark with `aria-label="Main navigation"` |
| Top bar | `<header>` landmark |
| Content area | `<main>` landmark |
| Active nav item | `aria-current="page"` (provided by React Router `<NavLink>`) |
| Hamburger button | `aria-label` toggles between "Open navigation" / "Close navigation"; `aria-expanded` reflects drawer state |
| Mobile drawer | Focus trap when open -- focus moves to first nav item on open, returns to hamburger on close |
| Loading bar | `role="progressbar"` with `aria-label="Loading page"` |
| Error boundary | Focus moves to error heading on render for screen reader announcement |
| Skip link | Hidden "Skip to main content" link as first focusable element, targets `<main id="main-content">` |

## Interaction States

All interactive elements in the layout follow the global rules from Phase 50:

| State | Visual Treatment |
|-------|-----------------|
| default | Component at rest with `shadow-retro-raised` where applicable |
| hover | Background shifts to `bg-retro-amber` for nav items, cursor pointer |
| active/pressed | `shadow-retro-pressed` replaces raised shadow |
| disabled | `bg-retro-gray`, `cursor-not-allowed`, `shadow-none` |
| focus-visible | `outline-2 outline-offset-2 outline-retro-amber` |
