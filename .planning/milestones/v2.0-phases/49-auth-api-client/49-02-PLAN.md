Start the dev server with `cd frontend2 && bun run dev`, then present the following verification checklist to the user. This is a human-verify checkpoint -- no code changes unless the user reports issues.

**Check 1: Route guard and auth page appearance**
- Navigate to http://localhost:5173/
- Expected: Automatically redirected to /login
- Expected: Charcoal background, centered cream panel (max 420px), hazard stripe bar at top
- Expected: Two file-folder tabs at top (LOGIN active/cream, REGISTER inactive/gray)
- Expected: Decorative red X button in top-right corner of panel
- Expected: Email and Password fields with inline SVG icons and thick 3px borders
- Expected: LOG IN button with retro beveled style
- Expected: OR divider then Google and GitHub OAuth buttons
- Compare to BAM reference: .planning/references/retro-ui/5.png

**Check 2: Tab toggle**
- Click the REGISTER tab
- Expected: REGISTER tab becomes cream (no bottom border), LOGIN tab becomes gray
- Expected: Four fields appear: Full Name, Email, Password, Confirm Password
- Expected: CREATE ACCOUNT button shown
- Click LOGIN tab again to switch back

**Check 3: Login error display**
- On login tab, enter any email and password, click LOG IN
- Expected: Inline red error text appears between fields and submit button
- Expected: No page reload occurred

**Check 4: Register password mismatch**
- Switch to REGISTER tab, fill all fields but enter different passwords, click CREATE ACCOUNT
- Expected: "Passwords do not match." error shown immediately (no API call)

**Check 5: OAuth button navigation**
- Click "SIGN IN WITH GOOGLE" button
- Expected: Full page navigation (address bar changes), NOT an internal SPA route

**Check 6: Mobile responsive**
- Resize browser to less than 640px width
- Expected: Panel fills width with margins, all elements readable and tappable
