# Phase 49 Plan 01: Vite Proxy Fix, API Client Tests, and i18n Summary

Vite proxy now strips `/api` prefix for correct backend routing; 9-test API client suite covers all auth transport scenarios; 27 Estonian auth translations extracted and ready.

## What Was Built

### Task 1: Vite Proxy Rewrite + API Client Tests

**Vite proxy fix (`frontend2/vite.config.ts`):**
Added `rewrite: (path: string) => path.replace(/^\/api/, "")` to the `/api` proxy configuration. Previously, requests to `/api/auth/login` would proxy to `http://localhost:8080/api/auth/login` (404). Now they correctly reach `http://localhost:8080/auth/login`.

**API client unit tests (`frontend2/src/lib/__tests__/api.test.ts`):**
Created 9 tests covering:
1. `get()` - GET request with `credentials: "include"` and correct headers
2. `post()` - POST request with JSON body serialization
3. Error handling - non-200 non-401 throws with parsed error message
4. 401 refresh retry - three fetch calls: original, refresh, retry
5. Concurrent 401 deduplication - refresh called only once for parallel requests
6. Refresh failure - throws "Session expired" when refresh returns 401
7. 204 No Content - resolves to `undefined` without JSON parsing
8. `patch()` - PATCH request with JSON body
9. `del()` - DELETE request

### Task 2: i18n Extraction and Estonian Translations

Ran `lingui extract` to extract 28 messages from auth source files into `.po` catalogs. Added Estonian (`et`) translations for all 27 auth messages, including:
- Form labels: EMAIL ADDRESS, PASSWORD, FULL NAME, CONFIRM PASSWORD
- Buttons: LOG IN, CREATE ACCOUNT, REGISTER, PROCESSING...
- OAuth: SIGN IN WITH GOOGLE, SIGN IN WITH GITHUB, OR
- Errors: Invalid credentials, duplicate account, network failure, generic error
- Callback: AUTHENTICATING..., AUTHENTICATION FAILED, RETURN TO LOGIN

Compiled catalogs with `lingui compile` (generated `.ts` files remain gitignored per `.gitignore`).
