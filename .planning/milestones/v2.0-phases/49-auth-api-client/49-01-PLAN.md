**Vite proxy fix:**

In `frontend2/vite.config.ts`, add the `rewrite` function to the `/api` proxy configuration. The current config at lines 20-27 is:

```typescript
server: {
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
    },
  },
},
```

Change it to:

```typescript
server: {
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
      rewrite: (path: string) => path.replace(/^\/api/, ""),
    },
  },
},
```

This ensures `/api/auth/login` is proxied to `http://localhost:8080/auth/login` (stripping the `/api` prefix), because the Go backend routes are registered at `/auth/login`, `/auth/register`, `/auth/refresh`, `/auth/logout`, `/users/me`, etc. -- all without an `/api` prefix.

**API client unit tests:**

Create `frontend2/src/lib/__tests__/api.test.ts` with the following test cases. Use `vi.stubGlobal('fetch', vi.fn())` to mock the global fetch function:

1. **get() makes GET request with credentials: "include"** -- call `get("/users/me")`, verify fetch was called with `{method: "GET", credentials: "include", headers: {"Content-Type": "application/json"}}`, mock 200 response with `{id: "u1"}`, verify return value matches.

2. **post() makes POST request with JSON body** -- call `post("/auth/login", {email: "a@b.com", password: "p"})`, verify fetch was called with `{method: "POST", body: '{"email":"a@b.com","password":"p"}', credentials: "include"}`, mock response with `{token: "t", refresh_token: "rt"}`.

3. **request throws on non-200 non-401 response** -- mock fetch returning 500 with body `{detail: "Internal error"}`, call `get("/fail")`, expect it to throw with message "Internal error".

4. **401 triggers refresh then retries original request** -- First call to fetch returns 401. Mock `setRefreshToken` to have set a refresh token first via the module. Second fetch call (to `/auth/refresh`) returns 200 with `{token:"t2", refresh_token:"rt2"}`. Third fetch call (retry) returns 200 with `{id:"u1"}`. Verify 3 fetch calls total. Because api.ts uses a module-level `storedRefreshToken` variable, set it by calling `setRefreshToken("rt1")` before the test, then verify the refresh call sends `{refresh_token: "rt1"}` in the body.

5. **concurrent 401s deduplicate refresh** -- Set refresh token. Mock fetch to return 401 for first two calls, then 200 for refresh, then 200 for both retries. Call `get("/a")` and `get("/b")` simultaneously with `Promise.all`. Verify the refresh endpoint was called only once (not twice).

6. **refresh failure throws "Session expired"** -- Set refresh token. Mock fetch: first call returns 401, second call (refresh) returns 401. Call `get("/fail")`, expect it to throw with message "Session expired".

7. **handles 204 No Content responses** -- Mock fetch returning 204 with no body and content-type not including "application/json". Call `post("/auth/logout")`, verify it resolves to undefined.

Import from `@/lib/api` using `{ get, post, patch, del, setRefreshToken, getRefreshToken }`. Use `beforeEach` to clear mocks and reset the refresh token to null via `setRefreshToken(null)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Vite proxy -> Backend | Auth credentials (email/password) cross from browser through proxy to backend |
| Backend -> Browser (cookies) | HttpOnly JWT cookies set by backend, sent automatically via credentials: "include" |
| OAuth provider -> Backend -> Browser | OAuth authorization code passes through URL params |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Information Disclosure | api.ts | mitigate | All requests use `credentials: "include"` for HttpOnly cookies; tokens never stored in localStorage; refresh token stored in module-level variable (memory only, lost on reload) |
| T-49-02 | Tampering | vite.config.ts proxy | accept | Vite proxy is dev-only; production serves via same origin or proper reverse proxy; rewrite rule only affects development |
| T-49-03 | Spoofing | AuthCallbackPage.tsx | mitigate | OAuth CSRF handled by backend via state cookie; frontend uses `useRef(false)` guard against React StrictMode double-exchange; one-time codes in Redis with GETDEL |
| T-49-04 | Spoofing | LoginForm.tsx redirect | mitigate | Post-login redirect uses `location.state.from.pathname` which is an internal React Router state, not a URL parameter; open redirect not possible |
| T-49-05 | Denial of Service | api.ts refresh | mitigate | Concurrent 401 refresh calls deduplicated via shared `refreshPromise`; prevents refresh endpoint flooding |
| T-49-06 | Tampering | CSRF on auth endpoints | accept | Backend uses SameSite=Lax cookies; auth endpoints are POST-only; acceptable ASVS L1 posture for cookie-based auth |
</threat_model>
