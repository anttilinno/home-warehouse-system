# Requirements: Home Warehouse System

**Defined:** 2026-04-08
**Core Value:** Reliable inventory access anywhere -- online or offline -- with seamless sync

## v2.0 Requirements

Requirements for Retro Frontend milestone. Each maps to roadmap phases.

### Scaffold

- [ ] **SCAF-01**: Vite 8 + React 19 project scaffolded in `/frontend2` with dev server proxying to Go backend
- [ ] **SCAF-02**: React Router v7 (library mode) configured with route structure matching frontend1 pages
- [ ] **SCAF-03**: Tailwind CSS 4 configured with `@theme` block defining retro design tokens (colors, borders, shadows, fonts)
- [ ] **SCAF-04**: Lingui v5 configured with EN + ET locale support, message extraction, and compile-time catalogs

### Auth

- [ ] **AUTH-01**: User can log in with email and password and see authenticated routes
- [ ] **AUTH-02**: User is redirected to login when accessing protected routes without a session
- [ ] **AUTH-03**: API client handles JWT tokens from HttpOnly cookies with automatic refresh
- [ ] **AUTH-04**: User can register a new account
- [ ] **AUTH-05**: User can log out

### Design System

- [x] **SYNC-01**: Quick capture works fully offline — items queued in IndexedDB
- [x] **SYNC-02**: Photos stored as blobs in IndexedDB for offline display
- [x] **SYNC-03**: Photos upload automatically after item syncs to server (chained sync)
- [x] **SYNC-04**: Offline-captured items appear in item list with pending indicator

### Dashboard

- [x] **COMP-01**: Quick-captured items are flagged as "needs details" in the database
- [x] **COMP-02**: User can filter item list to show only "needs details" items
- [x] **COMP-03**: User can mark an item as complete (remove "needs details" flag)
- [x] **COMP-04**: User sees session summary when ending quick capture (count + thumbnails)

### Settings

- [ ] **SET-01**: Settings hub with retro tabbed/panel navigation to 8 subpages
- [ ] **SET-02**: Profile settings -- user can edit name, email, and avatar
- [ ] **SET-03**: Security settings -- user can change password and view active sessions
- [ ] **SET-04**: Appearance settings -- theme toggle (retro light/retro dark)
- [ ] **SET-05**: Language settings -- switch between EN and ET
- [ ] **SET-06**: Regional Formats settings -- date, time, and number format preferences
- [ ] **SET-07**: Notifications settings -- per-category notification toggles
- [ ] **SET-08**: Data & Storage settings -- cache management and manual sync trigger

### Layout

- [ ] **LAY-01**: App shell with retro-styled sidebar navigation and top bar
- [ ] **LAY-02**: Mobile-responsive layout that maintains retro aesthetic on small screens
- [ ] **LAY-03**: Loading states and error boundaries with retro styling

## v2.1+ Requirements

Deferred to future milestones. Tracked but not in current roadmap.

### Items & Inventory

- **ITEM-01**: User can list, create, edit, and delete items with photo upload
- **ITEM-02**: User can search items with fuzzy matching

### Extended Features

- **EXT-02**: Repair logs with photos and warranty tracking
- **EXT-03**: Barcode/QR scanning with camera
- **EXT-04**: Quick capture (rapid item entry)
- **EXT-05**: Declutter assistant (unused item detection)

### Integration

- **INT-01**: Google + GitHub OAuth login

- **INT-03**: CSV import/export
- **INT-04**: Bulk photo operations

### Offline (future)

- **OFFL-01**: PWA installable with service worker
- **OFFL-02**: IndexedDB offline storage and sync
- **OFFL-03**: Offline mutations with conflict resolution

## Out of Scope

| Feature | Reason |
|---------|--------|
| Offline/PWA support | Deferred -- online-only for v2.0, reduces 30-40% complexity |
| shadcn/ui components | Fights retro aesthetic -- fully custom components instead |
| Server-side rendering | SPA is sufficient -- no SEO need behind auth |
| Dark mode (standard) | Retro IS the theme -- optional retro-dark variant only |
| Drag-and-drop inventory grid | Terrible UX for 100+ items, accessibility issues |
| CRT shader/scanline effects | Performance and accessibility concerns |
| Russian language | Start with EN + ET, add RU in future milestone |
