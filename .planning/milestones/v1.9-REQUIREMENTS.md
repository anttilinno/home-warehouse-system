# Requirements: Home Warehouse System

**Defined:** 2026-04-08
**Core Value:** Reliable inventory access anywhere -- online or offline -- with seamless sync

## v2.0 Requirements

Requirements for Retro Frontend milestone. Each maps to roadmap phases.

### Scaffold

- [x] **SCAF-01**: Vite 8 + React 19 project scaffolded in `/frontend2` with dev server proxying to Go backend
- [x] **SCAF-02**: React Router v7 (library mode) configured with route structure matching frontend1 pages
- [x] **SCAF-03**: Tailwind CSS 4 configured with `@theme` block defining retro design tokens (colors, borders, shadows, fonts)
- [x] **SCAF-04**: Lingui v5 configured with EN + ET locale support, message extraction, and compile-time catalogs

### Auth

- [x] **AUTH-01**: User can log in with email and password and see authenticated routes
- [x] **AUTH-02**: User is redirected to login when accessing protected routes without a session
- [x] **AUTH-03**: API client handles JWT tokens from HttpOnly cookies with automatic refresh
- [x] **AUTH-04**: User can register a new account
- [x] **AUTH-05**: User can log out

### Design System

- [x] **DS-01**: RetroButton component with neutral, primary, danger, and secondary variants and distinct hover/press/disabled states
- [x] **DS-02**: RetroPanel component with optional hazard stripe header and close button
- [x] **DS-03**: RetroInput component with icon prefix slot, monospace text, and inline validation error state
- [x] **DS-04**: RetroCard lightweight content container component
- [x] **DS-05**: RetroDialog modal using native `<dialog>` element with imperative open/close API via ref
- [x] **DS-06**: RetroTable data table with retro charcoal header and alternating row styling
- [x] **DS-07**: RetroTabs file-folder style tab bar with controlled active tab
- [x] **DS-08**: RetroToast ToastProvider context and useToast hook with auto-dismiss and color variants
- [x] **DS-09**: RetroBadge inline status badge with 5 color variants (default, success, danger, warning, info)
- [x] **DS-10**: HazardStripe configurable decorative divider using diagonal yellow-black gradient

### Dashboard

- [x] **DASH-01**: Dashboard HUD panels display total items, categories, and locations counts fetched from the analytics API
- [x] **DASH-02**: Terminal-styled activity feed shows recent inventory actions; SSE EventSource triggers live re-fetch without page reload
- [x] **DASH-03**: Quick-access action cards for Add Item, Scan Barcode, and View Loans navigate to /items, /scan, /loans respectively

### Settings

- [x] **SET-01**: Settings hub with retro tabbed/panel navigation to 8 subpages
- [x] **SET-02**: Profile settings -- user can edit name, email, and avatar
- [x] **SET-03**: Security settings -- user can change password and view active sessions
- [x] **SET-04**: Appearance settings -- theme toggle (retro light/retro dark)
- [x] **SET-05**: Language settings -- switch between EN and ET
- [x] **SET-06**: Regional Formats settings -- date, time, and number format preferences
- [x] **SET-07**: Notifications settings -- per-category notification toggles
- [x] **SET-08**: Data & Storage settings -- cache management and manual sync trigger

### Layout

- [x] **LAY-01**: App shell with retro-styled sidebar navigation and top bar
- [x] **LAY-02**: Mobile-responsive layout that maintains retro aesthetic on small screens
- [x] **LAY-03**: Loading states and error boundaries with retro styling

## v2.1+ Requirements

Deferred to future milestones. Tracked but not in current roadmap.

### Items & Inventory

- **ITEM-01**: User can list, create, edit, and delete items with photo upload
- **ITEM-02**: User can search items with fuzzy matching
- **ITEM-03**: Taxonomy CRUD (categories, locations, containers, borrowers)

### Extended Features

- **EXT-01**: Loans management (loan/return tracking)
- **EXT-02**: Repair logs with photos and warranty tracking
- **EXT-03**: Barcode/QR scanning with camera
- **EXT-04**: Quick capture (rapid item entry)
- **EXT-05**: Declutter assistant (unused item detection)

### Integration

- **INT-01**: Google + GitHub OAuth login
- **INT-02**: SSE real-time notifications
- **INT-03**: CSV import/export
- **INT-04**: Bulk photo operations

### Offline (future)

- **OFFL-01**: PWA installable with service worker
- **OFFL-02**: IndexedDB offline storage and sync
- **OFFL-03**: Offline mutations with conflict resolution

## Out of Scope

| Feature | Reason |
|---------|--------|
| Offline/PWA support | Deferred -- online-only for v2.0, reduces 30-40% complexity |
| shadcn/ui components | Fights retro aesthetic -- fully custom components instead |
| Server-side rendering | SPA is sufficient -- no SEO need behind auth |
| Dark mode (standard) | Retro IS the theme -- optional retro-dark variant only |
| Drag-and-drop inventory grid | Terrible UX for 100+ items, accessibility issues |
| CRT shader/scanline effects | Performance and accessibility concerns |
| Russian language | Start with EN + ET, add RU in future milestone |
