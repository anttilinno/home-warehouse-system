# Step 1 Summary — Retro Terminal Token Mapping

## Outcome

Appended a single new `[data-theme="retro-terminal"] { … }` rule block to
`frontend/app/globals.css` (lines 147–224), positioned immediately after the
existing `.dark` block and before `/* Animations */ @keyframes wiggle`. The
block:

- Defines the 13 premium-terminal source variables (`--bg-base`, `--bg-panel`,
  `--bg-panel-2`, `--bg-elevated`, `--bg-active`, `--fg-dim`, `--fg-mid`,
  `--fg-base`, `--fg-bright`, `--fg-glow`, `--amber`, `--accent-warn`,
  `--accent-danger`, `--accent-info`) as hex literals — scoped to the theme
  block so they cannot leak into `:root` / `.dark`.
- Aliases all shadcn tokens (`--background`, `--foreground`, `--card*`,
  `--popover*`, `--primary*`, `--secondary*`, `--muted*`, `--accent*`,
  `--destructive`, `--border`, `--input`, `--ring`, `--chart-1..5`,
  `--sidebar*`) onto the source variables via `var(...)`.
- Sets `--radius: 0` and `--font-sans: var(--font-mono)` for sharp corners +
  monospace body while the theme is active.
