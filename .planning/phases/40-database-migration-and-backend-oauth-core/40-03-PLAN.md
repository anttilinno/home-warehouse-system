  <name>Task 1: Create OAuth handler with initiate, callback, exchange, and account management endpoints</name>
  <files>
    backend/internal/domain/auth/oauth/handler.go
  </files>
  <action>
    Create `backend/internal/domain/auth/oauth/handler.go` following the patterns from `user/handler.go`.

    **Handler struct and constructor:**
    1. `Handler` struct with fields: service *Service, jwtService *shared.JWTService, sessionSvc SessionService (interface), redisClient RedisClient (interface), cfg *config.Config, providerConfigs map[string]*oauth2.Config.
    2. Define `RedisClient` interface: `Set(ctx, key, value string, expiration time.Duration) error`, `GetDel(ctx, key string) (string, error)`. This avoids importing redis directly.
    3. Define `SessionService` interface for session creation (same pattern as user handler): `CreateSession(ctx, input) (*session.Session, error)`.
    4. Constructor `NewHandler(svc *Service, jwtSvc *shared.JWTService, sessionSvc SessionService, redisClient RedisClient, cfg *config.Config) *Handler` that builds providerConfigs map from cfg (call GoogleConfig and GitHubConfig from providers.go with cfg.GoogleClientID, cfg.GoogleClientSecret, cfg.BackendURL etc.).

    **Initiate endpoint (raw Chi handler, not Huma -- redirect-based):**
    5. `func (h *Handler) Initiate(w http.ResponseWriter, r *http.Request)`:
       - Extract provider from `chi.URLParam(r, "provider")`.
       - Get provider config from h.providerConfigs[provider]. If not found, return 400.
       - Generate PKCE verifier: `oauth2.GenerateVerifier()`.
       - Generate CSRF state: 32 bytes random, base64url encoded (use `crypto/rand`).
       - Store state and verifier in single HttpOnly cookie: `oauth_state` with value `{state}|{verifier}`. Cookie settings: Path="/", MaxAge=600 (10 min), HttpOnly=true, Secure=(cfg.AppURL starts with "https"), SameSite=Lax.
       - Build auth URL: `providerConfig.AuthCodeURL(state, oauth2.S256ChallengeOption(verifier))`.
       - Redirect: `http.Redirect(w, r, url, http.StatusTemporaryRedirect)`.

    **Callback endpoint (raw Chi handler -- redirect-based):**
    6. `func (h *Handler) Callback(w http.ResponseWriter, r *http.Request)`:
       - Extract provider from `chi.URLParam(r, "provider")`.
       - Get provider config. If not found, redirect to frontend error page.
       - Read `oauth_state` cookie. If missing, redirect to frontend with error=invalid_state.
       - Parse cookie value: split by `|` to get state and verifier.
       - Validate state matches `r.URL.Query().Get("state")`. If mismatch, redirect to frontend with error=invalid_state (Pitfall 8-C).
       - Delete the cookie immediately (set MaxAge=-1).
       - Check for OAuth error in query params: if `r.URL.Query().Get("error")` is non-empty, redirect to frontend with error=authorization_cancelled (or the specific error).
       - Exchange authorization code for token: `providerConfig.Exchange(r.Context(), r.URL.Query().Get("code"), oauth2.VerifierOption(verifier))`.
       - Fetch user profile: switch on provider, call `FetchGoogleProfile` or `FetchGitHubProfile`.
       - Resolve user: `h.service.FindOrCreateUser(r.Context(), profile)`. Handle errors:
         - ErrEmailNotVerified -> redirect to frontend with error=email_not_verified
         - Other errors -> redirect to frontend with error=server_error
       - Generate JWT tokens: `h.jwtService.GenerateToken(user)` and `h.jwtService.GenerateRefreshToken(user)`.
       - Create session (same as login handler): call h.sessionSvc.CreateSession with user agent, IP, refresh token hash.
       - Generate one-time code: 32 bytes random, base64url encoded.
       - Store in Redis: key=`oauth_code:{code}`, value=`{accessToken}|{refreshToken}`, TTL=60 seconds.
       - Redirect to frontend: `{cfg.AppURL}/auth/callback?code={code}`.

    **Exchange endpoint (Huma typed handler -- JSON):**
    7. Define input/output types:
       - `ExchangeInput` struct with `Body struct { Code string }`.
       - `ExchangeOutput` struct matching login response: `Body struct { AccessToken, RefreshToken, TokenType string; ExpiresIn int; User UserInfo }`.
    8. `func (h *Handler) ExchangeCode(ctx context.Context, input *ExchangeInput) (*ExchangeOutput, error)`:
       - Get code from input.Body.Code. If empty, return 400.
       - Read from Redis: `h.redisClient.GetDel(ctx, "oauth_code:"+code)`. If not found (expired or already used), return 401.
       - Parse value: split by `|` to get accessToken and refreshToken.
       - Return ExchangeOutput with tokens and user info.
       - ALSO set auth cookies (access_token, refresh_token) on the response, matching the login handler cookie pattern. Look at how user/handler.go sets cookies on the huma response writer.

    **List accounts endpoint (Huma typed handler, JWT-protected):**
    9. `ListAccountsOutput` struct with Body containing `Accounts []AccountResponse`.
    10. `AccountResponse` struct: Provider, ProviderUserID, Email, DisplayName, AvatarURL, CreatedAt.
    11. `func (h *Handler) ListAccounts(ctx context.Context, input *struct{}) (*ListAccountsOutput, error)`:
        - Get userID from context (same pattern as user handler).
        - Call `h.service.ListAccounts(ctx, userID)`.
        - Map to response.

    **Unlink account endpoint (Huma typed handler, JWT-protected):**
    12. `UnlinkInput` struct with `Provider string` from URL path param.
    13. `func (h *Handler) UnlinkAccount(ctx context.Context, input *UnlinkInput) (*struct{}, error)`:
        - Get userID from context.
        - Get user to check hasPassword: `h.service.userSvc.GetByID(ctx, userID)` -- or pass hasPassword as param. Better: let the service handle this. Add userID to UnlinkAccount call, service loads user internally.
        - Actually, review the service.UnlinkAccount from Plan 02 -- it takes hasPassword as a param. The handler should load the user first to get HasPassword(), then pass it to the service.
        - Call `h.service.UnlinkAccount(ctx, userID, input.Provider, user.HasPassword())`.
        - Return 204 No Content on success.
        - Handle ErrCannotUnlinkLastAuth -> 409 Conflict.
        - Handle ErrOAuthAccountNotFound -> 404.

    **Error redirect helper:**
    14. Create `redirectWithError(w, r, appURL, errorCode string)` helper that redirects to `{appURL}/auth/callback?error={errorCode}`. Error codes: `invalid_state`, `authorization_cancelled`, `email_not_verified`, `server_error`, `provider_unavailable`.

    15. Verify: `cd backend && go build ./...`

    3. Register OAuth routes in four groups:

    Group 1 -- OAuth initiate (public, no rate limit, just redirects):
    ```go
    r.Get("/auth/oauth/{provider}", oauthHandler.Initiate)
    ```

    Group 2 -- OAuth callback (public, rate-limited at 10/min per IP):
    ```go
    r.Group(func(r chi.Router) {
        oauthCallbackRL := appMiddleware.NewRateLimiter(10, time.Minute)
        r.Use(appMiddleware.RateLimit(oauthCallbackRL))
        r.Get("/auth/oauth/{provider}/callback", oauthHandler.Callback)
    })
    ```

    Group 3 -- OAuth exchange (public, rate-limited with existing auth rate limiter):
    ```go
    // Register POST /auth/oauth/exchange as a Huma endpoint within a rate-limited group
    // Follow the pattern used for other Huma endpoints in the router
    ```
    Use Huma for the exchange endpoint since it's JSON-based. Follow the existing pattern for how Huma routes are registered in the router (look at how login/register endpoints are set up).

    Group 4 -- OAuth account management (JWT-protected):
    ```go
    r.Group(func(r chi.Router) {
        r.Use(appMiddleware.JWTAuth(jwtService))
        // Register Huma endpoints for ListAccounts and UnlinkAccount
    ```
