# Phase 4 — UI Design Contract

> Visual and interaction contract for the **Retro OS Pastel atom library**:
> form family (Select, Combobox, Textarea, Checkbox, FileInput, FormField),
> overlay family (Dialog, ConfirmDialog, Menu/Popover), feedback family (Toast,
> EmptyState, StatusDot, status pills), data family (Table selection,
> Pagination, Tabs), filter atoms (FilterBar, FilterPopover, BulkActionBar,
> SavedFilters), and the `/demo` review surface. Generated by gsd-ui-researcher
> (autonomous run), verified by gsd-ui-checker.

> **Direction re-anchor governs this spec.** ROADMAP / REQUIREMENTS / older
> CONTEXT prose for Phase 4 still carries scrapped **Premium Terminal**
> vocabulary ("sketch 005 chrome", "hazard-stripe", "`// HEADER` slot",
> "monospace anchor"). Per `04-CONTEXT.md`, `MANIFEST.md`, and the
> sketch-findings skill (BINDING), **sketches 001-005 are HISTORY ONLY**; the
> canonical direction is **Retro OS Pastel (006-008)**. Observable truths
> survive (panels have header slots, danger variants exist, numeric columns are
> tabular); exact treatment re-anchors to retro-os. Each re-interpretation is
> recorded as **Resolved (autonomous)**. This spec also stays consistent with
> the Phase 3 chrome language (keycaps, bevels, titlebar semantics, modal
> stack) — it extends, never forks, the shipped primitives.

## Design System

| Property | Value |
|----------|-------|

`retro/`, re-exported from `retro/index.ts`. The `/demo` page lives at
`frontend2/src/routes/demo/` (route surface, not an atom).

## Spacing Scale

The Phase 2 4px grid (`--sp-1..6`), surfaced as Tailwind utilities `p-sp-1 …
p-sp-6`, `gap-sp-*`, `m*-sp-*`. **No raw px spacing and no new tokens** — any
fixed pixel value below is a justified chrome dimension, not a new token.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| sp-1 | 4px | Checkbox-to-label gap, badge inner gap, dot offset, pager button gap, toast stack gap |
| sp-2 | 8px | FormField label→control gap, menu item padding, popover padding, filter chip gap, toast inner padding |
| sp-3 | 12px | Dialog body padding rhythm, toolbar/filter-bar padding, pager strip padding, BulkActionBar padding |
| sp-4 | 16px | Dialog body inset, demo section gutter, FileInput drop-zone padding |
| sp-5 | 24px | EmptyState vertical padding, demo section vertical rhythm |
| sp-6 | 32px | Demo page section breaks |

**Exceptions (fixed chrome dimensions, autonomous — justified):**

| Dimension | Value | Justification |
|-----------|-------|---------------|
| Checkbox box | 16×16px | System-7 square checkbox; matches the 14px Silkscreen titlebar corner-box scale and the 11px label cap height. Touch target met via a `≥24px` padded hit area (label is clickable). |
| Toast panel width | `min(360px, calc(100vw - var(--sp-4) * 2))` | Mini-window readable line length; clamps on mobile so it never touches the viewport edge. |
| Toast region offset (desktop) | `bottom: calc(36px + var(--sp-3))` | Sits **above the 36px Bottombar** (Phase 3 height) so chrome never overlaps. |
| Toast region offset (mobile) | `bottom: calc(env(safe-area-inset-bottom) + 56px + var(--sp-4))` | Sits **above the 56px FAB** (Phase 3) + safe area. |
| StatusDot | 8px (header), 6px (rail badge dot) | Matches the Phase 3 ONLINE dot + rail badge-dot scale (visual consistency). |
| Dialog width (default) | `min(520px, 92vw)` | Same as the Phase 3 F1 dialog — one dialog width contract. |
| ConfirmDialog width | `min(420px, 92vw)` | Narrower; a confirm is a sentence + two buttons, not a panel. |
| Pager button | `padding: 2px 9px`, 12px mono | Verbatim from sketch 008 `.pager .pg` — validated pager density. |
| Inline table action button | `padding: 2px 8px`, 11px | Verbatim from sketch 008 `.rtable .btn` — validated row-action density. |

## Typography

Reuses the locked retro-os contract (Phase 2/3). Display = Silkscreen; body =
Plex Sans (400/600/700); data = Plex Mono (400/600, `tabular-nums`). No new
roles; this table maps existing roles onto the new atoms.

| Role | Face | Size | Weight | Case | Usage in this phase |
|------|------|------|--------|------|---------------------|
| Dialog / toast title | Silkscreen | 16px | 400 | UPPERCASE | RetroDialog + RetroToast titlebar titles (Window titlebar) |
| Section / panel header | Silkscreen | 16px | 400 | UPPERCASE | `/demo` family section headers (Window titlebars) |
| Field label | IBM Plex Sans | 12px | 700 | UPPERCASE, `tracking-[0.08em]` | RetroFormField label, FilterBar labels (matches shipped RetroInput label) |
| Group / menu label | IBM Plex Sans | 10px | 700 | UPPERCASE, `tracking-[0.14em]` | Menu/popover group headers, SavedFilters group label, EmptyState eyebrow |
| Badge / pill / chip | IBM Plex Sans | 11px | 700 | UPPERCASE, `tracking-[0.07em]` | Status pills, filter chips, BulkActionBar count, tab labels |
| Body / control text | IBM Plex Sans | 14px | 400 | sentence | Select/Combobox/Textarea/menu-item text, dialog body, EmptyState body |
| Hint text | IBM Plex Sans | 12px | 400 | sentence | RetroFormField hint line (`text-fg-muted`) |
| Error text | IBM Plex Sans | 12px | 600 | sentence | RetroFormField + control error message (`text-danger`) — matches shipped RetroInput error |
| Data / counts / live | IBM Plex Mono | 12px | 400/600 | as-is | Pager meta, table numeric columns, StatusDot `sse: ● live` text, toast timestamps (`tabular-nums`) |

**Hard rules enforced (sketch-findings, BINDING):** Silkscreen never below
16px, never mixed case, never body copy; `--fg-faint` is decorative/disabled
only (never running text — see disabled treatments); pastel fills carry ink
text only — any colored text on white/cream uses the `*-deep` companions; radii
0 except badges/pills/chips (2px).

## Color

60/30/10 on the retro-os surface system. All values are Phase 2 tokens; **spec
references utilities, never raw hex** (hex shown for reference only).

| Role | Token (utility) | Hex | Usage |
|------|-----------------|-----|-------|
| Dominant (60%) | `bg-bg-desktop` | #fdf6ec | App/desktop behind windows; scrim base under dialogs (`bg-fg-ink/40`) |
| Surface (30%) | `bg-bg-panel` / `bg-bg-panel-2` | #fff / #f7f0e2 | Atom bodies, dialog bodies, menus (panel). Panel-2 for recessed strips: FilterBar, pager, BulkActionBar, table header. |
| Accent (10%) | `bg-titlebar-blue` | #b8d8e8 | **Reserved (see list below).** |
| Destructive | `text-danger` (#b73348) on `bg-danger-bg` (#fbe3e8) | — | ConfirmDialog (pink titlebar + danger confirm button), danger toast, DANGER pill, destructive bulk actions |

**Accent (`titlebar-blue`) reserved for — explicit, never "all interactive":**
1. **Selected table row** fill (`bg-titlebar-blue` + ink rule) — validated sketch 008.
2. **Current pager page** (`.pg.is-current`: blue fill + `shadow-hard-ink`) — verbatim sketch 008.
3. **Active tab** folder fill (RetroTabs current tab).
4. **Selected/highlighted menu or combobox option** (keyboard/hover active descendant).
5. **Default RetroDialog titlebar** (informational dialogs — same rule as the Phase 3 F1 dialog).
6. **Active SavedFilters preset chip** and **active FilterBar filter chip**.
7. **Focused element fill where a fill is the focus affordance** (e.g. selected combobox row); the focus *ring* is always ink (see Accessibility), independent of accent.

### Status & titlebar semantics (MANIFEST-locked — do not improvise)

| Semantic | Titlebar token | Where used in this phase |
|----------|----------------|--------------------------|
| Default / informational | `titlebar-blue` | RetroDialog default; INFO toast |
| Positive / inventory / success | `titlebar-mint` | Success/OK toast; success ConfirmDialog (rare) |
| Attention / **danger / destructive** | `titlebar-pink` | **RetroConfirmDialog** (destructive); DANGER toast |
| Warning | `titlebar-butter` | WARN toast |
| Chromeless utility | `plain` (`bg-bg-panel-2`, no pinstripe titlebar, or titlebar omitted) | Menu/popover, FilterPopover, SavedFilters menu — "plain = chromeless utility" per MANIFEST |

**Resolved (autonomous): "hazard-stripe" → retro-os danger treatment.** The
stale premium-terminal "hazard-stripe" maps to the danger token family:
destructive surfaces take a **pink titlebar** (ConfirmDialog) and/or
`bg-danger-bg`/`text-danger` (danger toast, DANGER pill, destructive confirm
button). No diagonal hazard stripes — those are a CRT/industrial idiom alien to
System 7. The pinstripe in retro-os is the *titlebar* texture only, color-coded
by semantic.

### Status pill variants (TUI-04 — exact token pairs, AA-safe)

Pills extend the shipped **RetroBadge** chrome (pastel fill + 1px ink border +
2px radius + 11px UPPERCASE **ink** text). Variant = fill token only; text is
always `text-fg-ink` on the pastel fill (AA-safe by the locked palette).

| Variant | Fill token | Text | Reference hex pair | AA |
|---------|-----------|------|--------------------|-----|
| OK | `bg-ok-bg` | `text-fg-ink` | #26262e on #ddf0e4 | reused (Phase 2 verified) |
| WARN | `bg-warn-bg` | `text-fg-ink` | #26262e on #f9eccb | reused |
| INFO | `bg-info-bg` | `text-fg-ink` | #26262e on #ddeaf3 | reused |
| DANGER | `bg-danger-bg` | `text-fg-ink` | #26262e on #fbe3e8 | reused |

**Resolved (autonomous): pill text color.** CONTEXT/MANIFEST says variants map
to deep companions (`OK=mint-deep` etc.). That mapping is the **colored-text-on-
white** rule — but a pill is a *pastel fill*, and the hard rule is "pastel fills
carry ink text ONLY." So inside a filled pill, text is **ink**, matching the
shipped RetroBadge exactly. The `*-deep` companions apply when status text sits
on white/cream **without** a fill (e.g. an inline "● Overdue" label in a table
cell: `text-accent-pink-deep`, no fill). Both forms are specified; neither
introduces a new pairing — all already pass `tokens.test.ts`. **No NEW contrast
pairings introduced this phase.**

## Form Family

All form atoms share: sunken (inverted-bevel) field chrome, 12px UPPERCASE
`tracking-[0.08em]` `text-fg-muted` label, ink focus ring, danger flip on error
— consistent with the shipped **RetroInput**.

### Shared field states (every interactive form atom)

| State | Treatment |
|-------|-----------|
| Default | `border-2 border-border-ink bg-bg-panel bevel-sunken text-fg-ink` |
| Focus-visible | `focus-visible:outline focus-visible:outline-3 focus-visible:outline-offset-1 focus-visible:outline-titlebar-blue` (matches shipped RetroInput) |
| Error | `border-danger bg-danger-bg`, `aria-invalid="true"`, `aria-describedby` → error id; error text `text-[12px] font-semibold text-danger` |
| Disabled | `disabled:opacity-50 disabled:cursor-not-allowed`; placeholder/value text may use `text-fg-faint` (the ONE place faint is allowed — disabled) |

### RetroSelect

**Resolved (autonomous): native `<select>` skinned, NOT custom listbox.** A
native select gives free keyboard/mobile/AT behavior and zero focus-trap risk.
Skin: same sunken-field chrome as RetroInput + a `▾` glyph affordance
(`absolute right`, `pointer-events-none`, ink). The dropdown panel itself is
OS-rendered (unavoidable with native select — accepted, it's the one place the
OS chrome shows and it's the correct trade for accessibility). For cases needing
a styled popup list, use **RetroCombobox** instead. RetroSelect = label + native
select + optional error, wrapped by RetroFormField.

### RetroCombobox

Hand-rolled: a **RetroInput** (text trigger, sunken) + a **chromeless-utility
popover** listbox (see Menu/Popover below). Open on focus/type/`▾` click.
- Listbox = `plain` popover: `bg-bg-panel border-2 border-border-ink bevel-raised` (raised, it floats), `max-h-[240px] overflow-y-auto`, anchored under the input, `z-40`.
- Option rows: `px-sp-2 py-[5px] text-[14px] text-fg-ink`; hover/active-descendant = `bg-titlebar-blue` (the accent-selected rule); selected option carries a `✓` glyph.
- Keyboard: ↑/↓ move active descendant, Enter selects, Esc closes **via `useModalStack`** (no own listener), Tab closes and commits. `role="combobox" aria-expanded aria-controls`; listbox `role="listbox"`, options `role="option" aria-selected`.
- Empty filter → a single `text-fg-muted text-[12px] px-sp-2 py-sp-2` row: `No matches.`
- Error/disabled: inherits the shared field states (error flips the input).

### RetroTextarea

Same sunken chrome as RetroInput, multi-line: `min-h-[88px] resize-y px-[10px]
py-[7px] text-[14px] font-body`. Wrapped by RetroFormField. `mono` prop (Plex
Mono) for code/notes, mirroring RetroInput's `mono`. Error/disabled per shared
states.

### RetroCheckbox

System-7 square box, pressed-bevel when checked.
- Box: **16×16px**, `border-2 border-border-ink`. Unchecked = `bg-bg-panel bevel-sunken` (a recessed empty box). Checked = `bg-titlebar-blue bevel-pressed` with a centered ink `✓` glyph (pressed = "it's been pushed in").
- Layout: `inline-flex items-center gap-sp-1`; the whole label is the click target (`≥24px` tall hit area — touch-safe).
- Indeterminate (for table header select-all partial): box shows an ink `–` dash, `bg-bg-panel-2`.
- Focus-visible: 2px ink outline on the box. Disabled: `opacity-50`, faint glyph.
- `<input type="checkbox" className="sr-only">` drives state (native semantics); the visual box is `aria-hidden` and the input is the accessible control.

### RetroFileInput

**Resolved (autonomous): drop-zone treatment, drag-drop supported.** CONTEXT
grants "drag-drop support level" discretion. Decision: a **sunken drop zone** +
a bevel "Browse…" button — both the dashed zone and the button trigger the
hidden native file input.
- Zone: `border-2 border-dashed border-border-ink bg-bg-panel-2 bevel-sunken p-sp-4 text-center`. Body copy 14px `text-fg-muted`: `Drop files here, or` + a `BevelButton` (neutral) labeled `BROWSE…`.
- Dragover state: `border-titlebar-blue bg-info-bg` + ink-text prompt `Release to add`.
- Selected files list: each = a row with mono filename + `tabular-nums` size + a `✕` remove BevelButton (neutral, small).
- Error (wrong type / too large): RetroFormField error treatment; reject the file and surface `text-danger` copy.
- Disabled: `opacity-50`, zone not droppable.

### RetroFormField (label / hint / error wrapper)

The composition wrapper every control above plugs into (RetroInput already
inlines label+error; RetroFormField generalizes it so Select/Combobox/Textarea/
FileInput share one layout). In-window error treatment per sketch 007.
- Layout: vertical stack, `gap-sp-1`.
- **Label** (top): 12px UPPERCASE `tracking-[0.08em]` `text-fg-muted`, `htmlFor` → control id.
- **Control** (middle): the field, given the shared states.
- **Hint** (optional, below control when no error): 12px `text-fg-muted` `font-body`.
- **Error** (replaces hint when present): 12px `font-semibold text-danger`, `id` referenced by the control's `aria-describedby`, prefixed with an ink `✕` glyph. **Error uses `text-danger` (#b73348 / danger-deep family) on the field's `bg-danger-bg`/cream** — the sketch-007 in-window error idiom. Required-field marker: an ink `*` after the label (not red — color is reserved for the actual error).

## Overlay Family

All overlays route ESC through the Phase 3 `useModalStack` (capture-phase
arbiter) — **no overlay owns a document-level ESC listener** (LOCKED). All
render through the Phase 3 portal over a `bg-fg-ink/40` scrim (dialogs) or
anchored (menus/popovers).

### RetroDialog

A retro-os **Window** rendered centered-modal over the scrim. Generalizes the
Phase 3 F1 dialog.
- Chrome: `Window` with `border-2 border-border-ink bevel-raised`, pinstriped titlebar, corner boxes. Width `min(520px, 92vw)`, body `p-sp-4`, scrollable if tall.
- **Titlebar color is semantic by purpose** (MANIFEST):
  - Informational / form dialog → **blue** (default).
  - Positive/success acknowledgement → **mint**.
  - Warning → **butter**.
  - Destructive → **pink** (but prefer **RetroConfirmDialog** for destructive confirms).
- A close box in the titlebar `actions` slot is interactive (`✕`, `aria-label="Close"`); clicking it = `onClose` (same path ESC takes).
- `role="dialog" aria-modal="true" aria-labelledby` → titlebar id. Focus trapped; focus restored to the invoking control on close.
- Footer action row (optional): right-aligned `BevelButton`s, `gap-sp-2`, `border-t-2 border-border-ink pt-sp-3` if the body scrolls.

**Resolved (autonomous): "in-window" everything.** All Phase 4 dialogs are
Windows — there is no bare-div modal in retro-os. The "panel has a header slot"
observable truth from the stale spec is satisfied by the Window titlebar.

### RetroConfirmDialog

A **RetroDialog preset for destructive/decision confirms** (CONTEXT-specified).
- **Titlebar = pink** (`titlebarVariant="pink"`) — danger semantic per MANIFEST.
- Body: a single sentence (14px body). Optional `text-danger` consequence line.
- Footer: two `BevelButton`s — **confirm** (`variant="danger"`: `bg-danger-bg text-danger`) + **cancel** (`variant="neutral"`).
- **Focus defaults to Cancel** (safe default — CONTEXT-specified). Enter on Cancel cancels; the destructive action requires explicit confirm-button activation.
- Width `min(420px, 92vw)`.
- Non-destructive decisions (e.g. "Discard changes?") may use a **butter** titlebar + neutral/primary confirm instead — the danger styling is reserved for genuinely destructive actions.

### Menu / Popover primitive

The shared chromeless-utility floating panel (combobox listbox, FilterPopover,
SavedFilters menu, user-menu items, table row-action menu all build on it).
- Surface: `bg-bg-panel border-2 border-border-ink bevel-raised` (raised — it floats above content), `min-w-[160px]`, `z-40`. **No pinstriped titlebar** ("plain = chromeless utility" per MANIFEST) — an optional 10px UPPERCASE `tracking-[0.14em]` `text-fg-muted` group header sits inside instead.
- Item: `flex items-center gap-sp-2 px-sp-2 py-[6px] text-[14px] text-fg-ink cursor-pointer`. Hover/active = `bg-titlebar-blue`. Disabled item = `text-fg-muted cursor-not-allowed`. Danger item (e.g. Delete) = `text-danger`, hover `bg-danger-bg`.
- Separator: `border-t border-table-rule my-sp-1`.
- Anchored to its trigger (flip up/down to stay in viewport); on tap-outside or ESC (**via `useModalStack`**) it closes. `role="menu"` (action menus) or `role="listbox"` (selection popovers); items `role="menuitem"`/`role="option"`. Trigger gets `aria-haspopup aria-expanded`.
- Focus: opening moves focus into the panel (first item / active option); closing restores focus to the trigger.

## Feedback Family

### RetroToast (TUI — skinned toast system)

Hand-rolled (sonner declined). Mini **Window-chrome** panels in a fixed region.
- **Region placement:** desktop = **bottom-right, above the Bottombar** (`bottom: calc(36px + var(--sp-3)); right: var(--sp-4); z-50`); mobile (<768px) = **bottom-center/right above the FAB** (`bottom: calc(env(safe-area-inset-bottom) + 56px + var(--sp-4))`). Stacks upward, newest on top, `gap-sp-1`.
- **Each toast = a small Window**: `border-2 border-border-ink bevel-raised`, width `min(360px, calc(100vw - 2*var(--sp-4)))`, a thin pinstriped titlebar carrying a 16px Silkscreen type label + a `✕` close box, body `p-sp-2` 14px body copy.
- **Semantic titlebar stripe by toast type** (MANIFEST color semantics):

  | Type | Titlebar | Title text | Lead glyph |
  |------|----------|-----------|-----------|
  | success | `mint` | `DONE` | `✓` |
  | info | `blue` | `INFO` | `●` |
  | warning | `butter` | `WARN` | `⚠` |
  | danger | `pink` | `ERROR` | `✕` |

- Behavior: auto-dismiss (default 5s; danger/error = manual-dismiss only, never auto, so failures aren't missed) + manual close box. Hover pauses the auto-dismiss timer. Optional action link (one `BevelButton` neutral, small) e.g. `UNDO`.
- A11y: container `role="region" aria-label="Notifications"`; each toast `role="status"` (polite) for info/success, `role="alert"` (assertive) for warning/danger. Close box `aria-label="Dismiss"`.
- Motion: slide-up + fade in (120ms ease), fade out on dismiss; **respects `prefers-reduced-motion`** → instant.

### RetroEmptyState

Centered placeholder for empty lists/tables/filtered results.
- Layout: `flex flex-col items-center gap-sp-2 py-sp-5 px-sp-4 text-center`.
- Optional eyebrow: 10px UPPERCASE `tracking-[0.14em]` `text-fg-muted`.
- Glyph: a large unicode placeholder in a 1px ink-bordered square thumb frame (`◇` / `▢`, 32px), `text-fg-faint` (decorative — faint is allowed here).
- Heading: 16px Silkscreen UPPERCASE `text-fg-ink`.
- Body: 14px `text-fg-muted`, one sentence + next step.
- Optional action: a `BevelButton` (primary) — the recovery CTA.

### RetroStatusDot (TUI-03 — visual primitive, NO SSE wiring)

The `sse: ● live` panel-header indicator + the live blink.
- Composition: `font-mono text-[12px]` `text-fg-muted` key (`sse:`) + a dot + an optional state word.
- **Dot:** 8px square (radius 0), ink-bordered: live = `bg-titlebar-mint`, idle/offline = `bg-fg-faint`, error = `bg-danger`. (Matches the Phase 3 ONLINE dot scale + color family.)
- **Step-end blink (live only):** a CSS `steps()` keyframe so the dot blinks hard on/off (no smooth fade — System-7 has no easing on a status LED):
  ```css
  @keyframes status-blink { 50% { opacity: 0; } }
  .status-dot--live { animation: status-blink 1.4s steps(1, end) infinite; }
  ```
  **Respects `prefers-reduced-motion`** → animation removed, dot stays solid mint.
- State text (optional): `live` (`text-accent-mint-deep`), `offline` (`text-fg-muted`), `error` (`text-danger`) — colored text on panel/cream uses the deep companions, never on a fill.
- **No SSE wiring** — props-driven (`state: "live" | "idle" | "error"`); the dashboard wires it in Phase 6.

### Status pills

See the **Color → Status pill variants** table above. Built on RetroBadge
chrome; OK/WARN/INFO/DANGER fills; ink text; 2px radius.

## Data Family

### RetroTable — selection states (TUI-06)

Extends the shipped RetroTable + `.rtable` globals.css chrome. Selection is
driven by a `useTableSelection(rows)` hook (click selects, **Shift+Click**
range-selects, **Ctrl/Cmd+Click** toggles) exposing `{ selected, actions }`.

| State | Treatment (reuse `.rtable` rules) |
|-------|-----------------------------------|
| Even-row stripe | `bg-table-stripe` (#fcf8f0) — existing |
| Hover | `bg-info-bg` — existing |
| **Selected row** | **full `bg-titlebar-blue` fill + ink bottom rule** (`tr[aria-selected="true"]`) — existing rule, verbatim sketch 008 |
| **Range-select (Shift)** visual | every row in the contiguous range gets `aria-selected="true"` → the same blue fill (a solid blue block reads as "this span is selected"); the anchor + focus row additionally carry a 2px ink left-border accent so the range endpoints are legible |
| **Header checkbox** | a `RetroCheckbox` in the leading `<th>`: unchecked = none selected, checked = all selected, **indeterminate dash** = partial selection (the `–` box) |
| Per-row checkbox | leading `<td>` `RetroCheckbox`; toggles that row without disturbing others |
| Focus (keyboard) | the active row gets a 2px ink `outline` (focus-visible), independent of selection fill |

- **`tabular-nums` columns:** numeric/code cells use the existing `.mono` class (`font-mono` + `font-variant-numeric: tabular-nums`) — Qty, Barcode, timestamps. Right-align numeric columns (`text-right`).

## Accessibility

| Concern | Treatment |
|---------|-----------|
| Focus-visible | EVERY interactive atom (Select, Combobox trigger + options, Textarea, Checkbox box, FileInput zone + buttons, dialog controls, menu items, tabs, pager buttons, filter chips, toast close) gets a 2px ink ring: `focus-visible:outline focus-visible:outline-2 focus-visible:outline-border-ink focus-visible:outline-offset-2`. Form fields keep the shipped RetroInput 3px `outline-titlebar-blue` focus. Never `outline: none` without a replacement. |
| Disabled | `aria-disabled` / native `disabled`; `opacity-50 cursor-not-allowed`; faint text allowed only here (the one sanctioned `--fg-faint` text use). |
| Error | `aria-invalid="true"` on the control + `aria-describedby` → error id; error text `text-danger`, prefixed `✕`; danger-bg field flip. Never color-only — the `✕` glyph + message carry the signal for color-blind users. |
| ESC discipline | EVERY overlay (Dialog, ConfirmDialog, Menu/Popover, Combobox listbox, FilterPopover, SavedFilters menu) routes ESC through the Phase 3 `useModalStack` capture-phase arbiter (LOCKED). No overlay owns a document-level ESC listener. Integration test on `/demo`: dialog → drawer → menu stack pops topmost-first; logout never fires on bare ESC. |
| Focus trap + restore | Dialogs/ConfirmDialog trap focus and restore to the invoker on close. Menus/popovers move focus in on open, restore to trigger on close. |
| Dialog/menu ARIA | Dialog `role="dialog" aria-modal aria-labelledby`; menu `role="menu"`/`menuitem`; listbox `role="listbox"`/`option aria-selected`; combobox `role="combobox" aria-expanded aria-controls`; tabs `role="tablist"/"tab"/"tabpanel"` with roving tabindex. |
| Toast | container `role="region" aria-label="Notifications"`; info/success `role="status"` (polite), warning/danger `role="alert"` (assertive). Danger toasts never auto-dismiss. |
| Table selection | `aria-multiselectable`, `aria-selected` rows, header checkbox `aria-label`, keyboard row focus outline distinct from selection fill. |
| StatusDot | the dot is decorative chrome (not text) — AA text bar N/A; the state word (`live`/`offline`/`error`) uses deep companions / muted / danger on panel = AA-safe text. |
| Reduced motion | Toast slide/fade, StatusDot blink, dialog/popover transitions all respect `@media (prefers-reduced-motion: reduce)` → instant / no blink. |
| Touch targets | Checkbox hit area ≥24px (label clickable); mobile bulk-bar buttons + FileInput button ≥44px; toast close box ≥24px. |
| Contrast — **NO NEW pairings** | Every text pair reuses Phase 2 tokens already verified by `tokens.test.ts`: ink-on-panel, ink-on-panel-2, ink-on-blue (selected row / active tab / current pager / selected option), ink-on-ok/warn/info/danger-bg (pills), danger-on-danger-bg + danger-on-cream (errors), deep companions on white (inline status text), muted-on-panel (hints/meta). **No NEW contrast pairing is introduced by this phase** — flag for AA test: none. |
