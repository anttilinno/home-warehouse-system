useExpiringQuery() (features/inventory/hooks/useExpiringQuery.ts) — URL-param-driven; returns a
UseQueryResult of `{ items: ExpiringEntry[]; total: number }` (PLUS a `days` field). For the side-rail
card you want the COUNT (`total` / items.length) and a link to `/inventory/expiring`. It reads `?days`
from the URL (default 30) — on the dashboard route that param is absent → default window, which is fine.

useMaintenanceDueQuery(days?) (features/maintenance/hooks/useMaintenanceQuery.ts) → `{ items:
DueSchedule[]; isLoading; isError }`. Each DueSchedule carries `is_overdue` (SERVER flag — render it,
never recompute) + item_name. Card count = items.length; overdue count = items.filter(is_overdue).
