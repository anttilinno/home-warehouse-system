  <name>Task 1: Budget manifest + check script + self-tests (RED→GREEN)</name>
  <files>frontend2/bundle-budget.json, scripts/check-bundle-budget.mjs, scripts/__tests__/check-bundle-budget.test.mjs</files>
  <behavior>
    Self-tests (node --test, fixture .js files written to a temp dir, NOT a real build):
    - Under-budget fixtures (all chunks below ceiling) → exit 0, prints OK line.
    - One chunk over budget (e.g. an oversized index-*.js fixture) → exit 1, stderr names the chunk and shows a positive Δ.
    - messages bucket takes the LARGEST single messages-*.js (alternates), NOT the sum: given messages-en (19KB) + messages-et (16KB) + messages-ru (15KB) all under 22000, exit 0; bump ONE locale over 22000 → exit 1 (proves it is per-locale, not additive).
    - A dist dir missing an EXPECTED budgeted chunk (e.g. no scanner-*.js) → handle deterministically: treat as 0 bytes for that chunk (under budget) and note it, do NOT crash. (Lazy chunks can legitimately be empty.)
    - Unknown/unbudgeted files (*Page-*.js) are ignored, never error.
  </behavior>
  <action>
Write `frontend2/bundle-budget.json` — a JSON object mapping logical chunk →
gz-byte ceiling, EXACTLY: main 215000, charts 120000, scanner 65000, palette
20000, messages 22000 (OQ-2). Add a top-level `_comment` field noting these are
gzip bytes, baseline+headroom, and that `messages` is per-locale (largest single
catalog), not additive.

Write `scripts/check-bundle-budget.mjs` — pure node (no deps), modeled on
`scripts/check-i18n-catalog.mjs` style:
- Accept an optional positional arg: a directory of `*.js` files to measure
  (used by self-tests). Default (no arg) = `frontend2/dist/assets`. Resolve via
  `import.meta.url` + `resolve(__dirname, "..")` like the i18n guard.
- Read the manifest from `frontend2/bundle-budget.json` (resolve from REPO_ROOT,
  not from the measured dir — the manifest path is fixed).
- For each `*.js` in the measured dir: gzip its bytes with node's
  `zlib.gzipSync` and bucket by filename prefix — `index-` → main, `charts-` →
  charts, `scanner-` → scanner, `palette-` → palette, `messages-` → messages.
  Files matching no budgeted prefix (e.g. `*Page-*.js`) are ignored.
- For the `messages` bucket take the LARGEST single `messages-*.js` gz size
  (they are alternate locale catalogs, not loaded together) — do NOT sum them.
  All other buckets: if multiple files share a prefix, sum them (defensive;
  normally one each).
- Compare each bucket's measured gz bytes to its ceiling. Print a delta report
  line per budgeted chunk: `chunk: cur/budget (Δ <signed bytes vs budget>)`.
- Exit 1 if ANY bucket exceeds its ceiling (stderr lists the over-budget ones);
  else print an OK summary and exit 0. A budgeted chunk with no matching file =
  0 bytes (under budget) — note it, never crash.

Write `scripts/__tests__/check-bundle-budget.test.mjs` — node --test, modeled on
`scripts/__tests__/check-i18n-catalog.test.mjs`: build temp dirs of fixture
`*.js` files (`writeFileSync` with byte payloads sized so their gz output lands
under/over the ceilings — use incompressible random-ish bytes via
`crypto.randomBytes` so gz size is predictable ≈ raw size), `spawnSync` the
script against each, assert status + stderr substrings per the behavior block.
Cover: all-under (exit 0), one-over (exit 1 + Δ + chunk name), messages
per-locale-not-additive (both directions), missing-chunk-tolerated (exit 0),
unbudgeted-file-ignored.

CONSTRAINT: do NOT name any new file/dir with the substrings `sync`, `idb`, or
`offline` (lint:imports / FOUND-02 substring guard). "bundle-budget" is clean.
Commit: `feat(17-01): bundle budget manifest + check script + self-tests (POL-04)`.
  </action>
