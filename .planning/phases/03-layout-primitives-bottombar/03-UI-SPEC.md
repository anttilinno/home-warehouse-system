# Phase 3 ‚Äî UI Design Contract

> Visual and interaction contract for the Retro OS Pastel application chrome:
> AppShell, TopBar, Sidebar, Bottombar, PageHeader, FAB, and the F1 help
> dialog. Generated by gsd-ui-researcher (autonomous run), verified by
> gsd-ui-checker.

> **Direction re-anchor governs this spec.** ROADMAP / REQUIREMENTS text for
> SHELL-* and BAR-* carries scrapped Premium Terminal vocabulary (`// GROUP`
> comment labels, `[KEY] LABEL` amber chips, `// ROUTE`, "amber-on-near-black",
> "30√ó30 beveled square", "HOME WAREHOUSE"). Per `03-CONTEXT.md` and
> `MANIFEST.md`, **sketches 001-005 are HISTORY ONLY**; the canonical direction
> is **Retro OS Pastel (006-008)**. Every visual treatment below re-anchors to
> retro-os. Structure (grid, composition, SSOT, guards, modal stack, responsive
> split) survives verbatim; vocabulary does not. Each re-interpretation is
> recorded as **Resolved (autonomous)**.

## Typography

Three display/heading tiers + body + data, exactly as the locked retro-os
contract. Two body weights only (400 regular, 600 semibold); display is its
own face. Plex Mono 400/600 for data.

| Role | Face | Size | Weight | Line Height | Case | Usage |
|------|------|------|--------|-------------|------|-------|
| Display / brand / titlebar | Silkscreen | 16px (brand 20px) | 400 | 1.15 | UPPERCASE | TopBar brand mark, window/titlebar titles, F1 dialog title |
| Group label | IBM Plex Sans | 10px | 700 | 1.2 | UPPERCASE, `tracking-[0.14em]` | Sidebar group headers (replaces `// GROUP`) |
| Label / chip | IBM Plex Sans | 11px | 700 | 1.2 | UPPERCASE, `tracking-[0.1em]` | Key-cap chip labels, PageHeader meta keys, ONLINE label |
| Body / nav item | IBM Plex Sans | 13px | 600 | 1.5 | sentence | Nav item text, user pill name, dialog body |
| Body (running) | IBM Plex Sans | 14px | 400 | 1.5 | sentence | Help dialog descriptions, drawer copy |
| Data / clocks | IBM Plex Mono | 12px | 400/600 | 1.2 | as-is | SESSION/LOCAL clocks, LAST SYNC, key glyph in chip, nav counts (`tabular-nums`) |

**Hard rules enforced (sketch-findings, BINDING):** Silkscreen never below
16px, never mixed case, never body copy; `--fg-faint` decorative/disabled only;
pastel fills carry ink text only ‚Äî any colored text on white/cream uses the
`*-deep` companions; radii 0 except badges (2px).

**Resolved (autonomous): SHELL-04 "monospace amber labels" ‚Üí retro-os.**
Amber is dead Premium-Terminal palette. Sidebar group labels render as
sketch-006 section labels: 10px Plex Sans 700 uppercase, `tracking-[0.14em]`,
`text-fg-muted`, with a 1px dotted `border-fg-faint` bottom rule. The OBSERVABLE
truth (group labels exist and read as group labels) is preserved; the `//`
glyph and amber color are dropped.

## Layout ‚Äî AppShell Grid (SHELL-01, SHELL-02)

**2√ó3 CSS Grid**, single `data-collapsed` attribute toggles rail width (no JS
measure phase ‚Äî SHELL-02).

```
grid-template-areas:
  "topbar  topbar"
  "sidebar main"
  "sidebar bottombar";
grid-template-columns: var(--sidebar-w) 1fr;   /* --sidebar-w: 232px; [data-collapsed] ‚Üí 60px */
grid-template-rows:    40px 1fr 36px;           /* TopBar / content / Bottombar */
height: 100dvh;
```

- TopBar spans both columns (full width).
- Sidebar runs full content height (`sidebar` area spans rows 2‚Äì3).
- **Bottombar spans only the main column** (row 3, column 2) ‚Äî SHELL-01 explicit.
- Collapse: `[data-collapsed] { --sidebar-w: 60px; }` with
  `transition: grid-template-columns 160ms ease`. Animating a CSS variable that
  feeds `grid-template-columns` is GPU-cheap and needs no layout measurement.
- `main` is the scroll container (`overflow-y: auto; min-width: 0`); TopBar,
  Sidebar, Bottombar stay fixed in the viewport.

**Resolved (autonomous): mobile grid (<768px, SHELL-06, D-05).**
At `<768px` the grid collapses to a single column and single content row:
`grid-template-areas: "topbar" / "main";`. Sidebar leaves the grid and becomes
a fixed overlay drawer (off-canvas left, `translateX(-100%)` ‚Üí `0`, with a
`bg-fg-ink/40` scrim). **Bottombar is not rendered** (D-06 desktop-only); the
FAB renders instead (D-05). TopBar gains a hamburger toggle for the drawer.

## Sidebar / Navigator (SHELL-04)

A `plain`-titlebar `Window` titled **NAVIGATOR** (sketch 006), full content
height, `bg-bg-panel`, body `p-sp-2`. Extends the existing `Sidebar.tsx`.

### Group labels (replaces `// GROUP`)
10px Plex Sans 700 uppercase, `tracking-[0.14em]`, `text-fg-muted`,
`border-b border-dotted border-fg-faint pb-[3px]`, `mx-sp-2 mb-sp-1`. Groups:
**OVERVIEW**, **INVENTORY**, **SYSTEM** (legacy structure, direction-agnostic).

### Nav item states
Base: `flex items-center gap-sp-2 px-sp-2 py-[5px] text-[13px] font-semibold`,
glyph + label + optional `font-mono text-[11px] tabular-nums` count.

| State | Treatment |
|-------|-----------|
| Default | `border border-transparent text-fg-ink` |
| Hover | `border-border-ink bg-bg-panel-2` |
| Pressed | `bg-bg-pressed` |
| Active (current route) | `border-border-ink bg-titlebar-blue shadow-hard-ink` + `aria-current="page"` ‚Äî **the active bevel** |
| Disabled (route not built) | `text-fg-muted cursor-not-allowed`, dashed faint border on hover, `aria-disabled`, `title="Not built yet"` |

**Resolved (autonomous): SHELL-04 "left-border bevel + glow".** "Glow" is a
Premium-Terminal CRT effect (no glow under retro-os ‚Äî hard shadows only). The
active indicator is the validated sketch-006 bevel: pastel-blue fill + ink
border + `shadow-hard-ink`. No box-shadow glow, no blur. The OBSERVABLE truth
(per-route active emphasis that reads instantly) is preserved via the bevel.

### Collapse-to-rail (60px, SHELL-02)
At `[data-collapsed]`: labels and counts hide (`overflow-hidden`, width
collapses), only the 28px glyph cell remains, centered. Group labels collapse
to a 1px ink rule (a hairline divider) between groups. Active item keeps its
`bg-titlebar-blue` fill on the glyph cell. **Badge dot mode:** an item carrying
a count/unread renders a 6px ink-bordered pastel dot at the glyph cell's
top-right instead of the numeral. A collapse toggle button (chevron `‚Äπ` / `‚Ä∫`)
sits in the Navigator titlebar `actions` slot. Tooltips (native `title`) surface
the hidden label on hover in rail mode.

### User menu footer
Sketch-006 footer: `border-t-2 border-border-ink p-sp-2`, 28px pink Silkscreen
avatar + name (13px 600) + email (11px `text-fg-muted` truncate). In rail mode
collapses to the avatar only. (The TopBar user pill and this footer are the same
identity; both are valid retro-os patterns ‚Äî footer is the carried-over legacy
pattern, TopBar pill is the menu entry point.)

### Mobile drawer (<768px, SHELL-06)
Same Navigator window, rendered as a fixed off-canvas overlay
(`min(280px, 86vw)`), slide-in from left over a `bg-fg-ink/40` scrim. Opened by
the TopBar hamburger; dismissed by scrim click, ESC (via modal stack), or a
nav selection. `role="dialog" aria-modal="true"`. Focus trapped while open;
focus returns to the hamburger on close.

## Bottombar (BAR-01..05, desktop ‚â•768px only ‚Äî D-06)

36px strip in row 3 / column 2, `bg-bg-panel-2`, `border-t-2 border-border-ink`,
`px-sp-3`, flex row, `gap-sp-2`, `align-items-center`. Mounts on every
authenticated route. **Not rendered <768px** (FAB replaces it).

### Key-cap chip (replaces `[KEY] LABEL`)
Beveled System-7 keycap:
- Container: `inline-flex items-center gap-sp-1 border-2 border-border-ink bg-bg-panel px-sp-2 py-[2px] bevel-raised-ink`.
- Key glyph: `font-mono text-[12px] font-semibold` in a small inset cell
  (`bg-bg-panel-2`, `border border-border-ink`, e.g. `N`, `F1`).
- Label: `font-body text-[11px] font-bold uppercase tracking-[0.1em] text-fg-ink`.
- Hover: `brightness-103`. Active/press: `active:translate-x-px active:translate-y-px active:bg-bg-pressed active:bevel-pressed` (matches BevelButton).
- **Current/focused chip:** `bg-titlebar-blue` face.
- **`danger: true` chip:** `bg-danger-bg text-danger` (e.g. a delete shortcut).
- Chips are buttons (`<button>`), keyboard-focusable, each `aria-keyshortcuts`
  set to its key so AT announces the binding.

### F1 HELP chip
Always present, **right-anchored** (after a `flex-1` spacer). Same keycap chrome,
key glyph `F1`, label `HELP`. Click and F1-keydown both open the help dialog
(BAR-05). An `ESC` chip sits to its right (key glyph `ESC`, label `BACK`) ‚Äî both
F1 and ESC stay right-anchored under overflow (SHELL-06, Success Criterion 5).

### SESSION / LOCAL clocks (BAR-01)
Right cluster, after the F1/ESC pair: two `font-mono text-[12px] tabular-nums`
readouts, each with an 11px uppercase Plex Sans key (`SESSION`, `LOCAL`).
- **SESSION** = elapsed since login (app-mount fallback this phase), `hh:mm:ss`.
- **LOCAL** = wall clock (`toLocaleTimeString("et-EE")`, sketch-006 locale).
- One `setInterval` (1s) updates both. `tabular-nums` prevents column jitter
  (Phase 2 glyph-test convention).

### Overflow strategy (narrow desktop, Claude's Discretion ‚Üí Resolved)
**Resolved (autonomous): overflow = sheet, not paginate.** When route shortcut
chips exceed available width (768‚Äì~1024px desktops), the route chips collapse
behind a single **"‚ãØ MORE"** keycap that opens an upward bevel-window sheet
listing the overflowed chips (full keycap rows). F1/ESC and the clocks NEVER
overflow ‚Äî they are pinned to the right cluster outside the scroll/collapse zone
(Success Criterion 5 satisfied). A sheet beats pagination here: it keeps every
shortcut one interaction away without a stateful page index, and reuses the F1
dialog's window-overlay machinery (one overlay primitive, less code).

## PageHeader (SHELL-05)

Rendered at the top of `main`, above route content. `bg-bg-panel-2`,
`border-b-2 border-border-ink`, `px-sp-3 py-sp-2`, flex row baseline-aligned.

### Breadcrumb (replaces `// {ROUTE}`)
**Resolved (autonomous): breadcrumb glyph.** The `//` comment glyph is Premium
Terminal. Retro-os breadcrumb reads as a window path: Silkscreen-flavored is too
loud for a path, so use 11px Plex Sans 700 uppercase `tracking-[0.1em]`
`text-fg-ink` segments joined by an ink `‚Ä∫` separator (e.g.
`OVERVIEW ‚Ä∫ DASHBOARD`). The current/leaf segment is `text-fg-ink`; ancestors are
`text-fg-muted`. OBSERVABLE truth (breadcrumb shows current route) preserved.

### Meta line (SESSION / LAST SYNC)
Right-aligned, `font-mono text-[12px] tabular-nums text-fg-muted`:
`SESSION {hh:mm:ss}  ¬∑  LAST SYNC {hh:mm:ss}`. Keys (`SESSION`, `LAST SYNC`) in
11px Plex Sans 700 uppercase; values in Plex Mono. The `¬∑` middot replaces the
stale `//` divider. LAST SYNC is a placeholder static value this phase (binds to
live SSE in Phase 6). SESSION mirrors the Bottombar SESSION clock (shared timer).

### Radial / context menu (D-07)
On tap, the FAB opens a **context-aware action set** sourced from the same
`useShortcuts` SSOT the Bottombar reads (D-08 ‚Äî one source, two surfaces).
**Resolved (autonomous): radial ‚Üí bevel stack.** A true radial fan reads as a
material-design idiom alien to System 7. Instead the FAB expands an upward
**vertical stack of bevel keycap buttons** (each a route action, labeled,
`bg-bg-panel` keycaps with the same chrome as Bottombar chips), animated with a
short staggered translate-up + opacity (120ms, `ease`), the FAB glyph rotating
`+ ‚Üí √ó`. A `bg-fg-ink/40` scrim backs the open menu; scrim tap or ESC closes it
(via the modal stack). Empty action set ‚Üí FAB renders a single default action
(route-appropriate "Add"). This keeps the retro-os idiom while satisfying
"context-aware menu adapts per route."

## F1 Help Dialog (BAR-05, TUI-01/02)

A retro-os `Window` rendered as a centered modal over a `bg-fg-ink/40` scrim.

| Property | Value |
|----------|-------|
| Titlebar | **blue** (`titlebarVariant="blue"`) ‚Äî informational, not a warning. Title `KEYBOARD SHORTCUTS` (Silkscreen 16px uppercase). |
| Chrome | Window default: `border-2 border-border-ink bevel-raised`, pinstriped blue titlebar, decorative corner box(es). |
| Width | `min(520px, 92vw)` |
| Body | `p-sp-4`, scrollable if tall. |

**Resolved (autonomous): titlebar color = blue (utility/default), not pink.**
MANIFEST: blue = default, pink = attention/danger. The help dialog is
informational reference, so it takes the default blue titlebar ‚Äî NOT pink
(pink is reserved for attention/destructive surfaces). A future destructive
confirm dialog (e.g. delete-account) would use pink.

### Shortcut list layout
Grouped by scope (Global, then current route). Each row:
`grid grid-cols-[auto_1fr] gap-sp-3 items-center py-[3px]`:
- **Left:** the key-cap chip (same component as Bottombar ‚Äî visual consistency).
- **Right:** action description, 14px Plex Sans 400 `text-fg-ink`.
Group headers reuse the 10px uppercase `tracking-[0.14em] text-fg-muted` label
treatment. Global group includes `F1 ‚Äî Toggle this help`, `ESC ‚Äî Close / back`.

### Modal stack & ESC (TUI-02, BAR-05)
- A small modal-stack context (push on open, pop on close). ESC pops the
  **topmost** overlay only; reaches other handlers only when the stack is empty.
- **Logout is NEVER bound to bare ESC** ‚Äî only via the user-menu "Log out" item
  with confirm-before-logout. While any modal/drawer/menu is open, ESC closes
  it and does not log out (Success Criterion 4).
- F1 toggles the help dialog (open if closed, close if topmost).

| Concern | Treatment |
