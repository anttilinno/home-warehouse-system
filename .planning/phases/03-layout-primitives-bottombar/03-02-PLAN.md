<objective>
Build the modal-stack ESC-ordering provider — the single arbiter of ESC across
the F1 help dialog, the mobile drawer, the FAB menu, and any future overlay. ESC
pops the topmost overlay only; a bare ESC with nothing open is a no-op and NEVER
reaches logout (BAR-05, Success Criterion 4). This is a Wave 0 correctness
artifact: its ESC-ordering test ships before any overlay uses it.

Purpose: Centralize ESC ordering so independent per-overlay handlers never race
(03-RESEARCH.md Pitfall 4). One listener, one stack, deterministic pop order.
Output: `src/components/modal/` module — provider, convenience hook, tests, barrel.
</objective>
