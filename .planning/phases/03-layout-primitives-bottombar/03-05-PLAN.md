<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registered shortcut data → rendered chips/menu | Bottombar/dialog/FAB render shortcut label/key; actions are app-defined closures |
| ESC/F1 keystroke → overlay control | F1/ESC drive overlay open/close; ESC must never reach logout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | shortcut actions in chips/FAB/dialog | mitigate | Actions are static app-defined closures registered by feature code, never built from URL/event data |
| T-03-12 | Denial of Service (self-inflicted) | ESC reaching logout while an overlay is open | mitigate | F1 dialog + FAB push onto the modal stack; ESC pops the top only; logout is never bound to ESC (tests assert no-logout) |
| T-03-13 | Resource exhaustion | duplicate F1/keydown listeners | mitigate | Single F1 listener owner in the dialog; provider owns letter dispatch; Bottombar adds no letter listener (Pitfall 2); cleanup on unmount |
| T-03-14 | Tampering/XSS | shortcut label/key text | mitigate | Rendered as escaped JSX via ShortcutChip; no dangerouslySetInnerHTML |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this plan; no `motion`/`lucide` added (locked decisions); Package Legitimacy Audit: N/A |

Client-side chrome only; no endpoints, no credentials.
</threat_model>
