Backend surface (backend/.../borrower/handler.go, verified 2026-06-13), all under /workspaces/{wsId}:
- GET /borrowers?page=&limit=&archived=  → BARE { items: BorrowerResponse[] }  (NO total/total_pages). limit max 100 (422 over). page default 1, limit default 50, archived default false. NO q param on list.
- GET /borrowers/search?q=&limit=  → bare { items } (forward-compat / pickers only; list page does NOT call this).
- GET /borrowers/{id}  → BorrowerResponse
- POST /borrowers  → { name (req 1..255), email? (format email), phone?, notes? }
- PATCH /borrowers/{id}  → all-optional { name?, email?, phone?, notes? }
- DELETE /borrowers/{id}  → 400 "cannot delete borrower with active loans" when blocked (NOT 409)
- BorrowerResponse: { id, workspace_id, name, email?, phone?, notes?, is_archived, created_at, updated_at }

lib/api.ts exports: get/post/patch/del helpers, HttpError (carries .status), parseError (detail||message).
useWorkspace(): { currentWorkspaceId: string | null, ... }

RESEARCH.md Code Examples (lines ~399-555) contain the full reference bodies for
borrowersApi, useBorrowersQuery, useBorrowerMutations, the zod schema, and the MSW
handlers. Use them as the implementation template.
