<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Auth forms | Email/password from untrusted user input |
| OAuth buttons -> External providers | Full page navigation to Google/GitHub |
| OAuth callback -> Backend exchange | One-time code from URL params |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-07 | Spoofing | AuthCallbackPage.tsx | mitigate | Use `useRef` guard to prevent double OAuth code exchange (React Strict Mode). Backend GETDEL ensures one-time use of codes. |
| T-49-08 | Spoofing | OAuthButtons.tsx | accept | OAuth CSRF protection handled entirely by backend (state cookie + PKCE). Frontend just initiates navigation. |
| T-49-09 | Information Disclosure | LoginForm.tsx | mitigate | Never log or display passwords. Error messages are generic ("invalid credentials") to prevent email enumeration. Backend controls actual error messages. |
| T-49-10 | Spoofing | AuthPage.tsx post-login redirect | mitigate | Redirect target comes from React Router `location.state.from` (internal location object), not from URL query params. Cannot be manipulated by external links. |
| T-49-11 | Tampering | RegisterForm.tsx | accept | Client-side password match validation is UX only. Backend validates all registration data server-side. |
</threat_model>
