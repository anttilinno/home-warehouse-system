## Interaction States

### Form Submission Flow
1. User fills fields and clicks submit
2. Button text changes to "PROCESSING..." and button becomes disabled
3. On success: auth context updates, React Router navigates to `/` (dashboard)
4. On error: button re-enables, inline error message appears above button

### Tab Switching
1. Click inactive tab
2. Active tab changes appearance (cream background, no bottom border)
3. Previous tab becomes inactive (gray background)
4. Form fields below swap (login fields vs register fields)
5. Any existing error message clears on tab switch
6. Form field values are NOT preserved across tab switches

### OAuth Flow
1. User clicks OAuth button
2. Full page navigation to `/api/auth/oauth/{provider}` (via Vite proxy in dev)
3. Provider auth flow occurs (external page)
4. Redirect back to `/auth/callback?code=...`
5. Callback page shows "AUTHENTICATING..." state
6. On success: navigate to `/`
7. On error: show error message with "RETURN TO LOGIN" button

### Route Guard Redirect
1. Unauthenticated user navigates to protected route
2. Show nothing (or brief loading) while auth state loads
3. Once confirmed unauthenticated, redirect to `/login`
4. After successful login, redirect back to originally requested route (via `location.state.from`)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Focus visible | 2px `retro-amber` outline on all focusable elements |
| Tab order | Tab labels -> form fields (top to bottom) -> submit button -> OAuth buttons |
| Form labels | Each input has a visually hidden `<label>` element with `htmlFor` matching input `id` |
| Error announcement | Error message container has `role="alert"` and `aria-live="assertive"` |
| Button states | `aria-disabled="true"` on submit button during loading (not just visual) |
| OAuth buttons | `aria-label` includes provider name for screen readers |
| Decorative X | `aria-hidden="true"` on the non-functional close button |
| Color contrast | All text meets WCAG AA: ink (#1A1A1A) on cream (#F5F0E1) = 11.3:1; red (#CC3333) on cream = 4.6:1 |
