# Phase 56 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note:** This phase is pure infrastructure (TanStack Query wiring, typed API modules, `postMultipart` helper). The only rendered surface is `/api-demo` — a public, developer-facing smoke-test route that renders one real React Query fetch with explicit loading / success / error states. No end-user features ship in this phase. The contract below governs that single dev route plus any strings surfaced from the API layer (e.g. error messages thrown from `HttpError`).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (fully custom retro component library — see v2.0 decision) |
| Preset | not applicable |
| Component library | `@/components/retro` barrel (RetroButton, RetroCard, RetroPanel, RetroBadge, RetroInput, RetroTable, RetroToast, HazardStripe) |
| Icon library | none required for this phase (text-only dev route) |
| Font | `system-ui, -apple-system, sans-serif` for body; `IBM Plex Mono, JetBrains Mono, ui-monospace, monospace` for monospaced data (source: `frontend2/src/styles/globals.css`) |

Source of truth for tokens: `frontend2/src/styles/globals.css` (`@theme` block). No new tokens introduced in this phase.

---

## Spacing Scale

Declared values (must be multiples of 4) — pre-existing in `globals.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding on `/api-demo` |
| xl | 32px | Layout gaps between demo panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none.

---

## Typography

Retro aesthetic — monospace for data rows, sans for prose. Two weights only.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes: Data rows inside the smoke-test list (e.g., item names, ids) use `font-mono` at the Body size/weight. All `/api-demo` strings pass through the Lingui `t` macro (project convention).

---

## Color

60/30/10 split mapped to existing retro tokens:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F0E1` (retro-cream) | Page background, panel surfaces |
| Secondary (30%) | `#2A2A2A` (retro-charcoal) | Text, borders, headings |
| Accent (10%) | `#D4A017` (retro-amber) | Primary CTA background (Retry button), success badge on "data loaded" state |
| Destructive | `#CC3333` (retro-red) | Error-state panel border and error-state badge only |

Accent reserved for:
- The single "Retry" RetroButton on the `/api-demo` error state
- The "OK" / success indicator shown when the React Query fetch resolves

Destructive reserved for:
- The error-state panel border / badge when the fetch fails

No other surface on `/api-demo` uses accent or destructive colors.

## Copywriting Contract

All strings wrapped in the Lingui `t` macro. Dev-route copy is deliberately plain (no marketing tone).

| Element | Copy |
|---------|------|
| Page title (`/api-demo`) | "API Demo" |
| Primary CTA | "Retry fetch" |
| Loading state | "Loading data from API…" |
| Empty state heading | "No data returned" |
| Empty state body | "The endpoint responded successfully but returned zero records. Create a record in the backend, then retry." |
| Success state heading | "Data loaded" |
| Error state | "Could not reach the API. Check your network and the backend URL, then retry." |
| Destructive confirmation | not applicable — no destructive actions in this phase |
