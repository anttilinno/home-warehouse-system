# Phase 63 — UI Design Contract

> Visual and interaction contract for Phase 63 (Navigation & Polish). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase intent is polish, not invention.** Phase 63 wires the retro Sidebar and Dashboard to every v2.1 surface, ensures empty-state parity across the six list pages, and closes the Lingui catalog gap. No new retro primitives are introduced; this contract pins the *existing* design tokens so downstream additions (two new sidebar links, one rerouted quick-action card, audit of six empty states, EN/ET catalog sweep) stay on-pattern.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom retro component library — v2.0 locked decision: shadcn/ui fights the retro aesthetic) |

**Upstream source:** `frontend2/src/styles/globals.css` (`@theme` block) — all tokens below are already declared and in production use across phases 48–62. This phase MUST NOT add new CSS variables or redefine existing ones.

## Spacing Scale

Declared values (all multiples of 4 — matches the 8-point scale with an xs=4 leaf for icon gaps):

| Token | Value | Usage (enforced pattern) |
|-------|-------|--------------------------|
| xs  | 4px  | Icon-to-label gaps inside buttons (`gap-xs`), compact action-cluster spacing |
| sm  | 8px  | NavLink vertical padding (`py-sm`), stacked control spacing, badge offsets (`ml-sm`) |
| md  | 16px | NavLink horizontal padding (`px-md`), default card/panel padding, grid gap between quick-action cards |
| lg  | 24px | Page padding (`p-lg`), vertical rhythm between page sections (`gap-lg`), RetroEmptyState inner gap |
| xl  | 32px | Dashboard section gaps (`gap-xl`) between stats → activity → quick actions |
| 2xl | 48px | Not used in Phase 63 |
| 3xl | 64px | RetroEmptyState outer padding (`p-3xl`) — do not override |

**Touch-target exception (non-negotiable, inherited from phases 58–62):** action buttons use `min-h-[44px] min-w-[44px]` on mobile and `lg:min-h-[36px] lg:min-w-[36px]` on desktop. This is an intentional 44px mobile floor per iOS HIG / WCAG 2.5.5 that sits outside the 4px scale for icon-only affordances. Phase 63 must preserve this on any new or edited interactive element.

**Forbidden:** inline pixel values that are not multiples of 4, except the 44/36px touch-target floor above. Use the declared tokens only.

## Typography

Fixed to the four sizes and two weights already in use across phases 48–62. Phase 63 MUST NOT introduce a new size or weight.

| Role | Size | Weight | Line Height | Case | Font | Where used in Phase 63 |
|------|------|--------|-------------|------|------|------------------------|
| Micro | 12px | 700 (bold) | default (Tailwind) | UPPERCASE | sans or mono | Action button labels, chip/counter micro-text |
| Body / NavLink | 14px | 700 (bold) | default | UPPERCASE | sans | **Sidebar NavLinks (all six)**, QuickActionCard labels, inline action buttons |
| Body | 16px | 400 (regular) | 1.5 (default) | sentence case | sans | RetroEmptyState body copy, error panel body copy |
| Heading | 20px | 600 (semibold) or 700 (bold) | 1.2 (tight) | UPPERCASE | sans | Page H1 (`LOANS`, `ITEMS`, `BORROWERS`, `TAXONOMY`), RetroEmptyState title, error panel title |

Weight contract (exactly two weights allowed):
- **400 (regular)** — body copy (empty-state body, error body)
- **600 (semibold) or 700 (bold)** — headings, NavLinks, labels, button text, badges. Semibold is reserved for page H1 (`text-[20px] font-semibold uppercase tracking-wider`); bold is used everywhere else uppercase. This dual treatment predates Phase 63 and must not be harmonised in this phase.

Heading letter-spacing: `tracking-wider` is only applied to page H1 nodes that already use it (`ItemsListPage`, `LoansListPage`, `BorrowersListPage`, `TaxonomyPage`). Do not add or remove.

## Color

Retro palette locked in v2.0 (`src/styles/globals.css`). The 60/30/10 split is realized as follows:

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#F5F0E1` | `--color-retro-cream` | Page background for list pages, Sidebar background, TopBar background, RetroEmptyState surface, RetroPanel surface, inactive NavLink, QuickActionCard surface |
| Secondary (30%) | `#2A2A2A` | `--color-retro-charcoal` | AppShell outer frame (`bg-retro-charcoal` on `min-h-dvh`), the charcoal field the sidebar sits on. Supplementary ink `#1A1A1A` (`--color-retro-ink`) is used for 3px borders and default text. |
| Accent (10%) | `#D4A017` | `--color-retro-amber` | **Reserved-for list (exhaustive — do NOT apply to any other element in Phase 63):** (1) active Sidebar NavLink background (`bg-retro-amber`), (2) inactive Sidebar NavLink hover state (`hover:bg-retro-amber`), (3) focus-visible outline on NavLink (`focus-visible:outline-retro-amber`), (4) RetroButton `variant="primary"` hover/pressed states as already defined in `RetroButton.tsx`. |
| Destructive | `#CC3333` | `--color-retro-red` | Not used directly by Phase 63 surfaces. Already owned by existing error panels (`text-retro-red`) and the destructive flows inherited from phases 58–62 (ArchiveDeleteFlow, LoanReturnFlow). Phase 63 introduces zero new destructive actions. |

Supporting tokens already in use, **not** counted in the 60/30/10 split (documented here so the checker does not flag them as unplanned additions):
- `--color-retro-gray: #8B8B8B` — disabled / archived / dimmed text (e.g. `Create a location first.` helper, archived strikethrough labels, "em-dash placeholder" cells)
- `--color-retro-hazard-yellow: #FFD700` paired with `--color-retro-hazard-black: #1A1A1A` — the 45° hazard stripe on HazardStripe error headers only
- `--color-retro-orange`, `--color-retro-green`, `--color-retro-blue` — not used by Phase 63 surfaces

**Accent reserved for:** (1) active Sidebar NavLink, (2) Sidebar NavLink hover, (3) Sidebar NavLink focus outline, (4) RetroButton primary hover/pressed. **Nothing else.** If a planner or executor wants to add amber to a dashboard card, empty state CTA, or active tab, that is out of scope for Phase 63 and must be bounced.

## i18n Catalog Contract (REQ-04 success condition)

All user-visible strings introduced in phases 56–62 and in this phase must be present in both `frontend2/locales/en/messages.po` and `frontend2/locales/et/messages.po` with **no orphan keys**.

| File | Role | Required state at phase end |
|------|------|-----------------------------|
| `frontend2/locales/en/messages.po` | Source locale | Every `t\`…\`` usage in `frontend2/src` resolves to an msgid; no entries marked `#, obsolete` remain unless intentionally removed. |
| `frontend2/locales/et/messages.po` | Estonian | Every msgid present in the EN catalog has a non-empty `msgstr` translation (no `msgstr ""` fuzzy placeholders). |
| `lingui.config.ts` | Extract config | Unchanged — already scans `src` and emits to `locales/{locale}/messages`. |

The two new sidebar labels (`BORROWERS`, `TAXONOMY`) MUST have Estonian translations `LAENAJAD` and `TAKSONOOMIA` respectively, matching the page H1 strings already translated in phases 58 and 59.

Catalog operations Phase 63 must perform, in order:
1. `bun lingui extract` (or equivalent) after adding the two new sidebar NavLinks.
2. Translate any new untranslated keys into ET.
3. `bun lingui compile` to regenerate compiled message bundles so the UI actually serves the new strings.
4. Manual inspection: run `grep -n 'msgstr ""' frontend2/locales/et/messages.po` and confirm only intentional empty entries remain (or zero).

## Accessibility Contract (inherited — listed for completeness)

Phase 63 inherits and must not regress:

- Sidebar is a `<nav aria-label="Main navigation">` and every NavLink is a real anchor (`NavLink` from `react-router`). The two new NavLinks MUST use the same wrapper and class pattern so keyboard focus order follows DOM order: Dashboard, Items, Loans, Borrowers, Taxonomy, Settings.
- NavLinks show a `focus-visible:outline-retro-amber` ring on keyboard focus. This is the only colour the amber token is allowed to bleed onto focus outlines in this phase.
- QuickActionCard links remain `<Link aria-label={label}>` so screen readers announce the destination even though the visible label is UPPERCASE and duplicated inside a decorative button.
- Empty-state actions are real `<button>` elements from `RetroButton`, not `<div role="button">`, and remain 44px-tall on mobile.
