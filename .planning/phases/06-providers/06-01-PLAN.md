## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend SSE stream → browser | Server-pushed JSON crosses into the client; untrusted until parsed/validated |
| session cookie → SSE auth | HttpOnly `access_token` cookie authenticates the stream; must never appear in a URL/log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | EventSource URL (`SSEProvider.tsx`) | mitigate | Cookie auth only (`withCredentials:true`); NEVER `?token=`/JWT in the URL. Grep gate in Task 0 + Task 2 verify asserts no `?token=` in the SSE feature dir from the first commit. |
| T-06-02 | Tampering / DoS | SSE `data:` payload parse (event handlers) | mitigate | `JSON.parse` inside try/catch; malformed payload → ignored. Event `data` is never rendered as HTML here (chrome reads only status). |
| T-06-03 | Information Disclosure / Elevation | cross-workspace event leak | mitigate | Dispatcher keys invalidation by `[...prefix, wsId]`, so even a misrouted event cannot invalidate another workspace's cache; backend per-workspace broadcaster is the primary control. |
| T-06-04 | DoS (self) | reconnect loop on auth loss | mitigate | `auth-expired` listener closes the stream + halts reconnect; capped exponential backoff bounds retry pressure. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | Zero installs this phase (locked "no new deps"; native EventSource). RESEARCH §Package Legitimacy Audit confirms slopcheck N/A. No checkpoint needed. |
