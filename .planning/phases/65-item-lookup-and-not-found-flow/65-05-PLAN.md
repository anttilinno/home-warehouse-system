## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL `?barcode=` → RHF defaultValues → zod validation → network | User-controlled query string crosses into form state; validated by zod before submit |
| Enrichment API response → JSX render | Third-party (OpenFoodFacts / OpenProductsDB) string fields `name`, `brand`, `category` render as text |
| Enrichment API response → RHF setValue | User explicitly accepts suggestions via [USE] chip; never auto-written |
| User keypresses in BRAND field → zod validation → API payload | BRAND is now first-class (D-23); zod max-120 cap is the length guard; no regex needed (free-text) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-05-01 | Injection (V5 Input Validation) | `?barcode=<malicious>` URL param | mitigate | zod `itemCreateSchema.barcode` regex (Plan 65-02 loosened to `/^[A-Za-z0-9\-_]+$/`) + max 64 chars enforced on submit. Banner [USE] writes pass through same validation. Page renders barcode via React JSX `{barcode}` auto-escape |
| T-65-05-02 | Tampering (XSS via enrichment response) | UpcSuggestionBanner name/brand/category render | mitigate | React JSX auto-escapes `{product.name}` and `{product.brand}`. No dangerouslySetInnerHTML anywhere in this plan. Category rendered as helper text via `<p>{t\`Category hint: ${product.category} ...\`}</p>` — Lingui's tagged template keeps the interpolation a string |
| T-65-05-03 | Elevation (silent category auto-write) | LOOK-03 D-15 anti-feature | mitigate | UpcSuggestionBanner NEVER calls `setValue("category_id", ...)` — verified by Task 2 test which asserts zero calls with that first arg across the banner's full render/event cycle |
| T-65-05-04 | Repudiation | dirty-guard bypass | mitigate | `RetroConfirmDialog` (existing atom — has focus trap, ESC handling, a11y) is the only discard path when form is dirty. CANCEL button calls handleCancel which checks isDirty |
