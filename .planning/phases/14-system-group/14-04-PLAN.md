  declutter sooner)`. NOTE: the inventory row id for mark-used is the response `id` field (the inventory_id
  path param), NOT item_id.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend → /api/workspaces/{ws}/declutter* + /inventory/{id}/mark-used | workspace-scoped reads + one write cross here; the server scopes the mark-used to {ws} |
| declutter rows → client CSV (formula-evaluating sink) | item_name/sku/location/category cells flow into a spreadsheet app; every cell is formula-injection-escaped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-10 | Tampering | CSV/formula injection via item or location names | mitigate | declutterCsv mirrors loanCsv's escapeCell — leading =,+,-,@,\t,\r get a `'` prefix, quotes doubled, cell wrapped; the CSV test asserts the guard (T-08-CSV parity) |
| T-14-11 | Information Disclosure | a token riding a download URL | mitigate | the CSV is built in-memory from already-fetched rows and streamed via an object URL — no network download link, no token in a URL (T-08-TOKEN parity); declutter uses CLIENT CSV, not the server /export blob |
| T-14-12 | DoS | null currency through money formatting (white-screen) | mitigate | formatCents called with a guarded currency (?? fallback); null currency renders "—", never crashes (VALIDATION landmine) |
| T-14-13 | Tampering | mark-used sent with item_id instead of inventory_id | mitigate | the action uses row.id (the inventory row id = the mark-used path param), explicitly documented + asserted; item_id is never sent to mark-used |
| T-14-SC | Tampering | npm installs | mitigate | none — installs NO packages (composes existing react-query + @/lib/api + retro atoms) |
</threat_model>
