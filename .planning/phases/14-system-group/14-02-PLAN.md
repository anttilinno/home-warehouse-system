<objective>
SYS-02 — the /my-changes page: a read-only activity-table of the authenticated user's own recent
mutations, sourced from GET /my-pending-changes. Unlike /approvals this endpoint is open to ALL roles
(it returns only the caller's own changes), so there is no 403 guard — just an empty-state.

Verified backend surface (planner re-verified pendingchange/handler.go this session):
- `GET /my-pending-changes?status=pending|approved|rejected` → `{ changes: PendingChangeResponse[], total }`.
  The handler fetches FindByRequester(authUser.ID) then filters to the current workspace — every role may
  call it; it returns ONLY the caller's own changes. Envelope key is `changes` (NOT items), identical to
  /pending-changes (handler.go:175-180).
- PendingChangeResponse table fields: `id, entity_type, entity_id?, action (create|update|delete),
  status (pending|approved|rejected), created_at` (requester is always the caller — omit the requester column).

This is the simplest System-group surface: no select, no bulk, no shortcuts — just the activity table.
Mirror LoansListPage's Window + RetroTable composition without the tabs/filters.

| T-14-06 | DoS | retry-storm on a transient error | mitigate | retry:false; a single settle with a calm error line, no backoff loop |
