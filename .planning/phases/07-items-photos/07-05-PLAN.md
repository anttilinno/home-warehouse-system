| form input → POST/PATCH | validated client-side (zod), authoritative server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-14 | Tampering | XSS via ?barcode prefill | mitigate | The barcode value is bound into a controlled RHF input value (React-escaped); never interpolated into HTML or a URL. zod bounds the field. |
| T-07-15 | Tampering | malformed PATCH (accidental field clear) | mitigate | Update builds the PATCH from dirty fields only; untouched fields omitted (Pitfall 4) so an edit never silently nulls data. Proven in useItemFormMutations test. |
| T-07-16 | Information Disclosure | cross-tenant create/edit | accept (backend authoritative) | All paths carry wsId; backend scopes by workspace_id. |
| T-07-SC | Tampering | npm installs | mitigate | RHF/zod/resolvers already installed (confirmed) — zero new installs; slopcheck N/A. |
