<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user file → upload pipeline | untrusted bytes/type/size; client check is UX, server authoritative |
| caption / filename → render | user-supplied strings shown in DOM |
| backend photo URL → `<img>` | absolute URL already neutralized to /api-relative in Plan 01 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-10 | Tampering | XSS via caption/filename | mitigate | React escapes by default — captions/filenames render as text, never dangerouslySetInnerHTML. Server adds CSP/Content-Disposition on served images. |
| T-07-11 | DoS / Tampering | malicious upload (size/type) | mitigate (UX) + accept (server) | validateUploadFile (accept-list + 10MB) pre-check; backend MaxFileSize/AllowedMimeTypes authoritative. |
| T-07-12 | Tampering | zip-slip on download | accept (server authoritative) | Server `sanitizeUploadFilename` guards the zip; client only triggers an anchor download of an application/zip blob. |
| T-07-13 | Information Disclosure | `<img>` hitting wrong origin / leaking cookie | mitigate | Gallery + lightbox render ONLY the Plan-01 /api-relative URLs (same-origin, proxy, cookie carried) — never the raw absolute backend URL. |
| T-07-SC | Tampering | npm installs | mitigate | Zero new installs (@dnd-kit/next-image/lucide/sonner-direct all declined per UI-SPEC); slopcheck N/A. |
</threat_model>
