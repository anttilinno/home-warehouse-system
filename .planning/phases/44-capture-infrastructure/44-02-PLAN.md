  <action>
Create `frontend/lib/contexts/batch-capture-context.tsx` following the existing context pattern in `offline-context.tsx`.

**BatchSettings interface:**
```typescript
interface BatchSettings {
  categoryId: string | null;
  locationId: string | null;
}
```

**BatchCaptureContextValue interface:**
```typescript
interface BatchCaptureContextValue {
  settings: BatchSettings;
  setCategoryId: (id: string | null) => void;
  setLocationId: (id: string | null) => void;
  resetSettings: () => void;
  captureCount: number;
  incrementCaptureCount: () => void;
  /** Resolved display name for the current category (null if none set or not found in cache) */
  categoryName: string | null;
  /** Resolved display name for the current location (null if none set or not found in cache) */
  locationName: string | null;
}
```

**DEFAULT_SETTINGS:** `{ categoryId: null, locationId: null }`

**SESSION_STORAGE_KEY:** `"quickCaptureBatch"`

**Provider implementation:**
1. Initialize `settings` state from `sessionStorage` with SSR guard:
   ```typescript
   const [settings, setSettings] = useState<BatchSettings>(() => {
     if (typeof sessionStorage === "undefined") return DEFAULT_SETTINGS;
     try {
       const saved = sessionStorage.getItem(SESSION_STORAGE_KEY);
       return saved ? JSON.parse(saved) : DEFAULT_SETTINGS;
     } catch {
       return DEFAULT_SETTINGS;
     }
   });
   ```

2. Write to sessionStorage on every settings change via `useEffect`:
   ```typescript
   useEffect(() => {
     if (typeof sessionStorage === "undefined") return;
     sessionStorage.setItem(SESSION_STORAGE_KEY, JSON.stringify(settings));
   }, [settings]);
   ```

3. `captureCount` state initialized to `0` (NOT persisted — resets on remount).

4. `categoryName` and `locationName`: Use `useState<string | null>(null)` for each. Use a `useEffect` that watches `settings.categoryId` / `settings.locationId` and resolves the name from IndexedDB cache:
   ```typescript
   useEffect(() => {
     if (!settings.categoryId) {
       setCategoryName(null);
       return;
     }
     getAll<Category>("categories").then((cats) => {
       const found = cats.find((c) => c.id === settings.categoryId);
       setCategoryName(found?.name ?? null);
     });
   }, [settings.categoryId]);
   ```
   Same pattern for locationName using `getAll<Location>("locations")`.

5. Import `getAll` from `@/lib/db/offline-db`, `Category` from `@/lib/api/categories`, `Location` from `@/lib/types/locations`.

6. `setCategoryId`, `setLocationId`, `resetSettings`, `incrementCaptureCount` wrapped in `useCallback`.

**Exports:**
- `BatchCaptureProvider` component
- `useBatchCapture` hook (throws if used outside provider)

Do NOT persist captureCount to sessionStorage — it is ephemeral within the provider lifecycle.
Do NOT import or use useAutoSKU or useOfflineMutation here — those are consumed directly by the capture form in Phase 45.
