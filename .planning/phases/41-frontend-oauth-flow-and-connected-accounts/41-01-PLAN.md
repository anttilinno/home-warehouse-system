**A. Create the OAuth callback page** at `frontend/app/[locale]/(auth)/auth/callback/page.tsx`:

This is a `"use client"` page under the `(auth)` layout group (no auth guard -- critical, see Pitfall 1 in research). The page:

1. Uses `useSearchParams()` wrapped in a Suspense boundary (same pattern as OAuthErrorHandler, see Pitfall 6).
2. Uses a `useRef(false)` flag (`exchanged`) to prevent React Strict Mode double-execution (Pitfall 2). Set the flag to `true` BEFORE the async call.
3. In a `useEffect`:
   - Check for `error` search param -- if present, redirect to `/login?oauth_error=${error}` using `router.replace()`.
   - Check for `code` search param -- if missing, redirect to `/login`.
   - Call `authApi.exchangeOAuthCode(code)` which POSTs to `/auth/oauth/exchange` and stores the token.
   - After successful exchange, call `loadUserData()` from auth context to load user profile and workspaces (this is how OAUTH-08 works -- the backend already stored full_name from the provider, and loadUserData fetches it).
   - Read `sessionStorage.getItem("oauth_return_to")` for returnTo redirect (OAUTH-07). Remove it from sessionStorage.
   - Also check `sessionStorage.getItem("oauth_linking")` -- if `"true"`, this is a link flow from settings. Remove the flag, and redirect to `/dashboard/settings/security` instead of returnTo/dashboard.
   - Falls back to `/dashboard` if no returnTo.
   - On error, redirect to `/login?oauth_error=server_error`.
4. Renders a centered loading spinner (Loader2 from lucide-react) while processing.

Use `useRouter` from `@/i18n/navigation` for redirect (matches project convention, supports locale prefix).

Structure the page with an inner component wrapped in Suspense (same pattern as OAuthErrorHandler):
```tsx
function OAuthCallbackInner() { /* useSearchParams + effect logic */ }
export default function OAuthCallbackPage() {
  return <Suspense fallback={<Loader2 spinner/>}><OAuthCallbackInner /></Suspense>;
}
```
