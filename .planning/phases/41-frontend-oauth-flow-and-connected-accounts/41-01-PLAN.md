**Create the OAuth callback page** at `frontend/app/[locale]/(auth)/callback/page.tsx`:

This is a `"use client"` page that:
1. Reads `code`, `error`, and `redirect_to` from URL search params via `useSearchParams()`.
2. If `error` param exists, immediately redirects to `/login?error={errorParam}` using `window.location.href`.
3. If `code` is missing, redirects to `/login?error=missing_code`.
4. Uses a `useRef(false)` guard (`exchanged`) to prevent double-exchange in React StrictMode.
5. On mount, calls `authApi.exchangeOAuthCode(code)` which sets the token.
6. On success, performs full page redirect via `window.location.href` to `redirect_to` param value or `/dashboard` (NOT `router.push()` -- full reload ensures AuthProvider picks up the new token from localStorage).
7. On failure, shows error text briefly then redirects to `/login?error=exchange_failed` after 2 seconds via `setTimeout`.
8. Shows a centered loading spinner (`Loader2` from lucide-react with `animate-spin`) and "Completing sign in..." text while processing.
9. Shows error text in `text-destructive` when exchange fails.
10. Import `Loader2` from `lucide-react`, `authApi` from `@/lib/api/auth`, `useSearchParams` from `next/navigation`.
11. Also reads `redirect_to` from `sessionStorage.getItem("oauth_redirect_to")` as fallback if not in URL params; clear sessionStorage after reading.

The page lives under the `(auth)` layout group, so it gets the auth layout styling (centered card, logo, background decoration).
