Create `frontend/components/settings/connected-accounts.tsx` following the pattern established by `active-sessions.tsx`:

**Component structure:**
1. `"use client"` directive.
2. Imports: `useState`, `useEffect`, `useCallback` from React; `useSearchParams` from `next/navigation`; `useTranslations` from `next-intl`; `Link2`, `Unlink`, `Loader2` from `lucide-react`; `Button` from `@/components/ui/button`; `Badge` from `@/components/ui/badge`; `Skeleton` from `@/components/ui/skeleton`; `authApi`, `OAuthAccount` from `@/lib/api/auth`; `useAuth` from `@/lib/contexts/auth-context`; `toast` from `sonner`.

**Constants:**
- `API_URL = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000"`
- `PROVIDERS = ["google", "github"] as const`

**State:**
- `accounts: OAuthAccount[]` (initially empty)
- `isLoading: boolean` (initially true)
- `unlinkingProvider: string | null` (tracks which provider is being unlinked)

**Data loading:**
- `loadAccounts` callback: calls `authApi.getConnectedAccounts()`, sets accounts, handles errors silently (empty list on failure), wrapped in try/finally with isLoading.
- Call `loadAccounts()` in `useEffect` on mount.

**Handle `?linked=` query param:**
- Second `useEffect` reads `searchParams.get("linked")`.
- If present: show success toast `t("linkSuccess", { provider: linked })`, clean URL via `window.history.replaceState({}, "", url.toString())` after removing `linked` param, call `loadAccounts()` to refresh.

**Connect handler:**
- `handleConnect(provider)`: redirects via `window.location.href = \`${API_URL}/auth/oauth/${provider}?action=link\``.

**Disconnect handler:**
- `handleDisconnect(provider)`: sets `unlinkingProvider`, calls `authApi.unlinkAccount(provider)`, optimistically removes from accounts state, shows success toast, handles error with error toast.

**Lockout guard:**
- `canDisconnect(provider)`: returns true only if `user.has_password` OR there are other connected providers besides this one. When false, the Disconnect button is disabled.

**Rendering:**
- Loading state: two `Skeleton` placeholders (h-16 w-full).
- For each provider in PROVIDERS, render a row (border rounded-lg p-3, flex items-center justify-between):
  - Left side: Provider icon (reuse the Google/GitHub SVGs from social-login.tsx -- create a small `ProviderIcon` component inline or just use conditional SVG rendering), provider name (capitalized), "Connected" Badge if connected, email if connected.
  - Right side: If connected, ghost Button with Unlink icon (disabled when !canDisconnect or unlinkingProvider matches, with Loader2 spinner when unlinking). If not connected, outline Button with Link2 icon and "Connect" text.
- Below the provider list, if `!user?.has_password && accounts.length <= 1 && accounts.length > 0`, show a warning paragraph: "Set a password to allow disconnecting your last linked account."
