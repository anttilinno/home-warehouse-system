**A. Backend: Modify UpdatePassword in `backend/internal/domain/auth/user/service.go`**

Change the `UpdatePassword` method (around line 172) to skip the current password check when the user has no password set. This allows OAuth-only users to set their first password without providing a current password.

Before (current):
```go
if !user.CheckPassword(currentPassword) {
    return ErrInvalidPassword
}
```

After:
```go
// OAuth-only users (no password) can set their first password
// without providing a current password
if user.HasPassword() {
    if !user.CheckPassword(currentPassword) {
        return ErrInvalidPassword
    }
}
```

The handler at line 446 (`updatePassword`) passes `input.Body.CurrentPassword` which will be an empty string for OAuth-only users. The service now skips validation in that case. The entity's `UpdatePassword` method already sets `hasPassword = true` (line 222 of entity.go), so subsequent password changes will require the current password.

No other backend changes needed -- the handler, entity, and repository all work as-is.
