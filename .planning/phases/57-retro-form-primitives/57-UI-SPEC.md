# Phase 57 — UI Design Contract

> Visual and interaction contract for Retro Form Primitives. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Source of truth:** All tokens pre-populated from `frontend2/src/styles/globals.css` (v2.0 retro design system, shipped). Phase 57 extends the existing retro primitive set — it does NOT introduce new design tokens.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom retro component library — v2.0 decision locked) |
| Preset | not applicable |
| Component library | custom retro primitives under `@/components/retro` (no Radix, no shadcn) |
| Icon library | lucide-react (existing, used by v2.0 retro components) |
| Font | System sans (`system-ui, -apple-system, sans-serif`) for UI chrome; `IBM Plex Mono`/`JetBrains Mono` for inputs, SKUs, barcodes, codes, pagination counters |

**Rationale:** v2.0 shipped a fully custom retro component library because shadcn/ui fights the retro aesthetic (STATE.md decision). Phase 57 MUST reuse that pattern. No new runtime dependencies except `react-hook-form` + `zod` (already adopted in v2.1 decisions).

## Spacing Scale

Values pre-populated from `globals.css` `@theme` block. Multiples of 4, 8-point scale. This section contains ONLY layout-rhythm tokens. Per-component minimum dimensions (touch targets) live under "Component Constraints" below.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps in inline labels, error icon → message gap |
| sm | 8px | Checkbox label gap, select option vertical padding, pagination button gap |
| md | 16px | Default field vertical rhythm, form field label → control gap, dialog body padding |
| lg | 24px | Section padding inside forms, confirm-dialog action row top margin |
| xl | 32px | Form section breaks, empty-state internal padding |
| 2xl | 48px | Empty-state icon → heading gap, major layout gaps |
| 3xl | 64px | Empty-state outer vertical padding on full-page empties |

All values above are strict multiples of 4. No exceptions appear in the spacing scale.

## Component Constraints (Accessibility Touch Targets)

These are **per-component minimum dimensions**, NOT spacing tokens. They are enforced via `min-height` / `min-width` on the component itself and never participate in layout-rhythm math.

| Constraint | Value | Rationale |
|------------|-------|-----------|
| Mobile touch target (min-height/min-width) | 44px | iOS HIG & WCAG 2.5.5 AAA; Success Criterion 3 explicit |
| RetroPagination button height (desktop ≥768px compact mode) | 36px | Compact desktop pagination only; mobile breakpoint still enforces 44px |

Applies to: RetroSelect trigger, RetroCombobox trigger, RetroCheckbox hit area, RetroPagination buttons, RetroFileInput button, RetroConfirmDialog action buttons. The Component Inventory table below records the specific touch-target constraint per primitive.

## Typography

Exactly 3 sizes, 2 weights. All values already proven in shipped v2.0 RetroInput/RetroButton.

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body / Control text | 16px | 400 (regular) | 1.5 | mono for input values, SKU, barcode, pagination counter; sans for select-option label text and checkbox labels |
| Label / Small | 14px | 600 (semibold) | 1.4 | sans, uppercase tracking for field labels and table headers (matches RetroInput label style) |
| Heading (dialog title, empty-state heading) | 20px | 600 (semibold) | 1.2 | sans, uppercase |

**Rules:**
- Inline validation error text: 14px / 400 / line-height 1.4, color `--color-retro-red`, mono font to distinguish from prose.
- Helper/description text under a field: 14px / 400 / 1.4, sans, color `--color-retro-charcoal` at 70% opacity.
- No display-size text introduced in this phase.

## Color

Values pre-populated from `globals.css`. 60/30/10 split honored by existing retro system.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F0E1` (`--color-retro-cream`) | Page background, form field background, dialog panel background |
| Secondary (30%) | `#2A2A2A` (`--color-retro-charcoal`) | Text on cream, thick borders, table header backgrounds (inverted) |
| Accent (10%) | `#D4A017` (`--color-retro-amber`) | Primary CTA button fill, focus ring, active pagination page indicator, checked RetroCheckbox fill, selected combobox option highlight |
| Destructive | `#CC3333` (`--color-retro-red`) | Destructive confirm button, inline validation error text/border, "remove photo" icon in file list |

**Accent reserved for (explicit list — not "all interactive elements"):**
1. Primary CTA button background (RetroButton variant="primary")
2. `:focus-visible` outline ring on any focusable primitive (2px solid amber)
3. Active/current page indicator in RetroPagination
4. Checked state fill of RetroCheckbox
5. Highlighted (keyboard-focused) option in RetroCombobox/RetroSelect dropdown

**Other palette usage (NOT counted as accent):**
- `--color-retro-ink` `#1A1A1A` — box-shadow offset, hazard-stripe black band
- `--color-retro-orange` `#E67E22` — warning badges only (e.g. "Usage: 12 items" on archive-category confirm)
- `--color-retro-green` `#4A7C4A` — success toast only
- `--color-retro-blue` `#3366AA` — info toast/badge only
- `--color-retro-gray` `#8B8B8B` — disabled control borders and placeholder text
- `--color-retro-hazard-yellow` `#FFD700` — hazard stripes on destructive confirmation dialog header only

## Copywriting Contract

Phase 57 ships primitives, not features. Copy below is default/demo copy shown on `/demo` and the contract that downstream phases (58–62) MUST follow when composing these primitives.

**Cancel-button convention (applies to every RetroConfirmDialog variant below):** the non-destructive escape button is labelled `KEEP ITEM` when the dialog targets a single concrete record (item, borrower, photo, loan), and `← BACK` when it targets a structural entity (category, location, container) where "keep" reads awkwardly. Generic `CANCEL` is forbidden — every confirm dialog MUST carry a noun or direction signal.

| Element | Copy |
|---------|------|
| Primary CTA (demo form) | `SAVE CHANGES` (uppercase, verb + noun — matches v2.0 RetroButton convention) |
| Empty state heading (RetroEmptyState default demo) | `NO RECORDS YET` |
| Empty state body | `Create your first entry to populate this list.` followed by a primary action button labelled with concrete verb+noun (e.g. `ADD ITEM`, `ADD BORROWER`) |
| Error state (field-level) | `{Field name} is required.` / `Must be a valid email address.` / `Must be 255 characters or fewer.` — concrete problem, imperative voice |
| Error state (form submit, server failure) | `COULD NOT SAVE` heading + `The server rejected the request. Check your connection and try again.` + retry button |
| Destructive confirmation (RetroConfirmDialog default — delete item) | Title: `CONFIRM DELETE` • Body: `This action cannot be undone.` • Escape button: `KEEP ITEM` • Destructive button: `DELETE ITEM` (red fill, hazard-stripe header band) |
| Destructive confirmation (archive variant — structural entity) | Title: `CONFIRM ARCHIVE` • Body: `This {entity} will be hidden from default views. You can unarchive it later.` • Buttons: `← BACK` / `ARCHIVE {ENTITY}` |
| Combobox empty query | `Type to search…` (placeholder); `No matches found.` (empty result); `Loading…` (async state) |
| Pagination label | `Page {N} of {M}` (mono font, uppercase) with `← PREV` / `NEXT →` buttons |
| File input idle | Button: `CHOOSE FILES` • Helper: `JPEG, PNG, or HEIC up to 10 MB each.` |
| File input with selection | Chip per file: `{filename} · {size}` with a `×` remove icon button carrying `aria-label="Remove {filename}"` (retro-red on hover) |
| Checkbox label pattern | Sentence case, no trailing punctuation: `Include archived items` |

**Destructive action inventory consuming RetroConfirmDialog (pre-declared for downstream phases):**

| # | Action (phase) | Escape label | Destructive label | Notes |
|---|----------------|--------------|-------------------|-------|
| 1 | Delete item (Phase 60) | `KEEP ITEM` | `DELETE ITEM` | Confirm body echoes item name |
| 2 | Delete borrower (Phase 59) | `KEEP BORROWER` | `DELETE BORROWER` | Only reachable when no active loans |
| 3 | Archive/delete category \| location \| container (Phase 58) | `← BACK` | `ARCHIVE CATEGORY` / `ARCHIVE LOCATION` / `ARCHIVE CONTAINER` (or `DELETE …` when usage count is 0) | MUST show `Currently assigned to {N} item(s).` above the body |
| 4 | Delete item photo (Phase 61) | `KEEP PHOTO` | `DELETE PHOTO` | Confirm body shows thumbnail preview |
| 5 | Mark loan returned (Phase 62) | `KEEP OPEN` | `MARK RETURNED` | Soft confirmation — amber primary, no red, no hazard stripes |

**Tone rules:**
- UPPERCASE for button labels, dialog titles, section headings, empty-state headings, pagination counter.
- Sentence case for body copy, helper text, field labels (field labels may be UPPERCASE when rendered — CSS decision, not copy decision).
- No emoji. No exclamation marks. No "oops"/"whoops" — factory/industrial voice.
- No generic `CANCEL` / `OK` / `YES` / `NO` labels anywhere. Every button carries verb+noun or direction signal.
- i18n: every literal string MUST go through Lingui `t` macro and land in both `en` and `et` catalogs before Phase 57 checkpoint.

## Interaction Contracts

- **Focus ring:** every focusable control gets `outline: 2px solid var(--color-retro-amber); outline-offset: 2px` on `:focus-visible`. Never remove outline without replacement.
- **Keyboard:** RetroSelect/RetroCombobox support Up/Down/Home/End/Enter/Esc/Typeahead. RetroConfirmDialog supports Tab-loop and Esc-to-cancel (Esc triggers the escape-button handler, never auto-confirms).
- **ARIA:** RetroCombobox uses `role="combobox"` + `aria-expanded` + `aria-controls`; RetroCheckbox uses native `<input type="checkbox">`; RetroConfirmDialog uses `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing at the title.
- **Icon-only buttons:** every icon-only control MUST carry an `aria-label`. Specifically: the RetroFileInput chip remove button uses `aria-label="Remove {filename}"` interpolated with the actual filename so screen-reader users can distinguish which file they're removing.
- **react-hook-form integration:** RetroFormField accepts `name`, `label`, `helper`, and children; wires `register`/`control` from the parent form. Error message is read from `formState.errors[name]`. zod resolver is the expected validator.
- **Loading:** any primitive that can be "in flight" (RetroConfirmDialog action button, RetroFileInput during upload) disables itself and swaps label to `WORKING…` in mono.
- **Motion:** honor `prefers-reduced-motion` — dialog fade/scale and toast slide skipped when reduced; instant state swaps instead.
