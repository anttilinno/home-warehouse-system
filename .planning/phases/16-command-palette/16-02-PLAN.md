<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /search endpoints | untrusted query string crosses to workspace-scoped search; backend enforces workspace_id, client must always pass wsId + clamp limit |
| localStorage MRU → render | persisted ids/labels are attacker-tamperable; read-side must safe-parse + treat ids as opaque |
| keydown (window) → palette open | global chord fires from any focus incl. inputs; must preventDefault so the literal key is not typed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02 | Information disclosure | cross-tenant entity search | mitigate | useEntitySearch ALWAYS passes `currentWorkspaceId`; `enabled` is false without wsId; backend scopes by workspace_id (verified by-barcode test, CLAUDE.md). Keys are [domain, wsId, "search", q] so a workspace switch never serves stale-tenant rows. |
| T-16-03 | Tampering | entity names rendered as result rows (XSS) | mitigate | render names as `{row.name}` text children — React escapes by default; NEVER dangerouslySetInnerHTML. |
| T-16-04 | Tampering | localStorage MRU (hws-palette-recent) | mitigate | getRecent() safe-parses (returns [] on bad JSON), drops malformed entries, treats ids as opaque; navigation re-validates via existing route guards. |
| T-16-05 | DoS | limit > 100 → backend 422 / debounce storm | mitigate | clamp limit<=100 (use 5); fire only when trimmed length>=2; 250ms debounce with the timer in a ref (no per-keystroke fan-out). |
| T-16-SC | Tampering | npm installs | mitigate | none — this plan installs NO packages; it runs `bun install --frozen-lockfile` against the lockfile 16-01 committed behind its legitimacy gate. |
</threat_model>
