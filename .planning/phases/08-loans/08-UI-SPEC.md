## Loan wire shape (from the shipped `loansApi` + `Loan` type)

Recorded because the design depends on the exact fields the read model carries
(`frontend2/src/lib/api/loans.ts`, `LoanPanels.tsx`):

- `id`, `is_active` (the partition flag), `borrower` (`{ id, name }` тАФ name resolves
  on the wire), embedded `item` (with `primary_photo_thumbnail_url`, absolute),
  `loaned_at`, `due_date?`, `returned_at?`, plus `notes?`.
- **There is no `is_overdue` / `status` enum on the read model.** **Overdue is
  computed client-side**: `due_date != null && is_active && due_date < now`. **Returned
  loans are never overdue** (only `is_active` loans qualify). This drives the three
  tabs and the status pill (see ┬зStatus mapping). **Resolved (autonomous): overdue is
  a derived client predicate, not a backend flag** тАФ if RESEARCH confirms a backend
  `/loans/overdue` endpoint or an `is_overdue` field, the planner may swap the source,
  but the tab partition + pill mapping below is the contract either way.
- **Extend body shape** (`PATCH /loans/{id}/extend`) and the **CSV export endpoint**
  (`/export/loan` or similar) are confirmed by RESEARCH against backend source; the UI
  treatment below is endpoint-agnostic (it consumes whatever shape RESEARCH locks).

### Status тЖТ StatusPill variant mapping (Resolved тАФ autonomous)

`StatusPill` is a thin preset over `RetroBadge` (ok/warn/info/danger pastel fill +
**ink** text). The three loan states map to existing variants тАФ **no new pairings.**
The state is computed client-side from `is_active` + `due_date` (see Loan wire shape).

| Loan state | Predicate | Variant | Display | Rationale |
|------------|-----------|---------|---------|-----------|
| **Active** | `is_active && (!due_date || due_date тЙе now)` | **info** (blue) | `Active` | Out but on-time, neutral-informational (matches the Phase 7/7b `on loan тЖТ info`). |
| **Overdue** | `is_active && due_date && due_date < now` | **danger** (pink) | `Overdue` | Problem state тАФ past due, needs attention. |
| **Returned** | `!is_active` | **ok** (mint) | `Returned` | Completed, positive. |

**Resolved (autonomous): the existing `LoanHistoryList` stub pill mapping
(`is_active ? info "OUT" : ok "RETURNED"`) is upgraded** to this three-way map so the
overdue state is visible everywhere a loan pill renders (list, panels, history). The
stub's `OUT` becomes `Active`/`Overdue` per the due-date predicate; `RETURNED` stays.

### Overdue-row highlight (Resolved тАФ autonomous, must-specify #1 тАФ NOT color-only)

An overdue row in the Active/Overdue tabs is signalled by **THREE non-redundant cues**
so it is never color-only (sketch-findings + AA discipline):

1. **Row tint** тАФ overdue rows get a `bg-danger-bg` (#fbe3e8) row fill (the pale danger
   wash), distinct from the white/`#fcf8f0`-stripe normal rows and the
   `bg-titlebar-blue` selected fill. Hover keeps the existing `bg-info-bg` rule layered
   over it.
2. **Status pill** тАФ the row's Status column shows the **danger `Overdue`** pill (the
   word + the pink fill).
3. **Due-date chip** тАФ the Due column renders a **danger** chip `тЪа тИТ{n}d` (with the
   leading `тЪа` glyph + `тИТ` sign) instead of the neutral `due in {n}d`.

The pill word + the `тЪа`/`тИТ` chip prefix carry the meaning for color-blind users; the
row tint is a reinforcing, not sole, signal. **No new contrast pairing** тАФ `text-danger`
on `bg-danger-bg` and ink-on-row-tint both already pass `tokens.test.ts`.

**No NEW contrast pairing is introduced this phase.** Every text/bg pair already passes
`tokens.test.ts` (ink-on-panel/panel-2/blue/ok/warn/info/danger-bg,
danger-on-danger-bg + danger-on-cream, deep companions on white, muted-on-panel).
**Flag for AA test: none.**

## 1 ┬╖ Loans List Page (`/loans`) тАФ tabbed, mirrors items/inventory density (sketch 008)

One **mint-titled `Window`** (`LOANS тАФ {workspace name}`), `bodyClassName=""` (flush,

### Tabs (RetroTabs тАФ folder-tab idiom, must-specify #1)

The window body opens with a **`RetroTabs`** row of three System-7 folder tabs:
**`ACTIVE`** ┬╖ **`OVERDUE`** ┬╖ **`HISTORY`**. The active tab is the blue folder
(accent reserved #3); inactive tabs are panel-2 folders with ink rules. Each tab label
is Silkscreen 16px UPPERCASE (the tab label is display chrome).

- **`ACTIVE`** тАФ `is_active && !overdue` loans (out, on-time).
- **`OVERDUE`** тАФ `is_active && overdue` loans (out, past due) тАФ the attention tab.
- **`HISTORY`** тАФ `!is_active` loans (returned).

**Resolved (autonomous): tab state is URL-driven via `?tab=` (default `active`)** тАФ
mirrors the Phase 7 list's URL-driven sort/filter discipline and gives deep-link
consistency (CONTEXT discretion preferred URL; the `?itemId=`/scan-flow forward-compat
benefits from deep-linkable tabs). `?tab=active|overdue|history`.

**Resolved (autonomous): the source partition depends on RESEARCH's endpoint finding** тАФ
if the backend exposes `/loans/active` + `/loans/overdue`, each tab fetches its own
endpoint; if only a single list (or a status filter) exists, the loans are fetched once
and partitioned client-side on the `is_active` + `due_date < now` predicate (the same
predicate the `LoanPanels` stub already uses). Either way the tab partition contract is
the three predicates above; the planner wires whichever endpoint shape RESEARCH locks.

### Window-body composition (top тЖТ bottom)

1. **`RetroTabs`** strip (the Active/Overdue/History folders).
2. **`FilterBar`** (recessed `bg-bg-panel-2` strip), per tab:
   - Search input (`type="search"`, sunken, `w-[260px]`, placeholder `Filter loansтАж`) тАФ
     filters the current tab's rows by item or borrower name substring.
   - `flex-1` spacer, then a mono `{n} loans` count for the current tab.
   - **`тдУ EXPORT CSV`** (`BevelButton` neutral) тАФ **placement: right end of the FilterBar
     strip, before the primary CTA** (Resolved тАФ autonomous). Exports the **current tab's**
     loan set via the Phase 7 `downloadBlob` helper against the RESEARCH-confirmed export
     endpoint. Disabled (with hint toast `Nothing to export.`) when the tab is empty.
   - **Primary CTA slot**: `BevelButton variant="mint"` `тКХ NEW LOAN` тЖТ `/loans/new`.
3. **`RetroTable`** (column set below) тАФ the current tab's rows.
4. **`RetroPagination`** strip (server-driven; meta reads `page {n} of {m} ┬╖ 25 / page`).
   **Resolved (autonomous): per-page = 25** (matches the Phase 7/7b cadence). Pager hidden
   when a single page or empty.

**Per-tab row actions:**
- **Active tab**: `RETURN` (тЖТ return confirm, ┬з3) ┬╖ `EXTEND` (тЖТ extend dialog, ┬з4) ┬╖ `тЖз` overflow (`EDIT` тЖТ edit dialog ┬з5).
- **Overdue tab**: same as Active (`RETURN` ┬╖ `EXTEND` ┬╖ `тЖз EDIT`) тАФ overdue loans are still returnable/extendable.
- **History tab**: no lifecycle actions (returned loans are terminal); `тЖз` overflow may expose `EDIT` (notes only) тАФ planner's call; default ship = no actions on History rows (read-only), keeping the tab a clean archive.

- **Row click тЖТ `/items/{item_id}`** (`cursor-pointer`); the trailing action buttons `stopPropagation`.

- **Hover**: `bg-info-bg` (existing `.rtable` rule). **Overdue rows**: `bg-danger-bg` row tint (see ┬зOverdue-row highlight) with hover layering on top.

### Empty + filtered-empty states (`RetroEmptyState`, per tab)

- **Overdue tab, none overdue**: heading `NOTHING OVERDUE`, body `No active loans are past their due date.`, no action (good-news state).
- **History tab, no returned loans**: heading `NO LOAN HISTORY`, body `No loans have been returned yet.`, no action.
- **Filtered, no matches** (any tab): heading `NO MATCHES`, body `No loans match this search.`, action `CLEAR SEARCH` (clears `?q=`, keeps the tab).
- States render **inside the table window body** (replacing `<tbody>`); pager hidden when empty.

## 3 ┬╖ Return Action (must-specify #3)

A loan is returned via a **`RetroConfirmDialog` with a BLUE titlebar** (not the pink
default тАФ return is a positive, reversible completion, NOT destructive; see Titlebar
semantics). Opened from a row's `RETURN` action (Active/Overdue tabs) AND from the

## 4 ┬╖ Extend Action (must-specify #4)
