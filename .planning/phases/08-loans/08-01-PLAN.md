<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (loans/borrowers reads + mutations) | wsId-scoped; server is authoritative for tenancy + overdue |
| fetched rows → CSV file → spreadsheet app | untrusted borrower-name/notes cell values cross into a formula-evaluating sink |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-CSV | Tampering | loanCsv.loansToCsvBlob | mitigate | Prefix cells beginning with `= + - @ \t \r` with `'` and quote-wrap/escape every cell (Task 3 escape fn + test) — prevents CSV/formula injection from borrower name or notes. |
| T-08-01 | Information Disclosure | loansApi.* endpoints | mitigate | All calls go through `/workspaces/{wsId}/...`; wsId from useWorkspace SSOT (consumer plans). Backend re-scopes by workspace_id (V4); frontend never sends a foreign wsId. |
| T-08-TOKEN | Information Disclosure | CSV export path | mitigate | Client-gen in-memory Blob (URL.createObjectURL) — no server export URL, no token in a download link (override 3). |
</threat_model>
