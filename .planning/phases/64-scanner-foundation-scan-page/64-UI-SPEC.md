# Phase 64 — UI Design Contract

> Visual and interaction contract for the `/scan` route (viewfinder + Manual + History tabs) plus the four scanner error panels. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (fully custom retro system, established in Phase 50 + Phase 54) |
| Preset | not applicable — shadcn explicitly rejected per v2.0 decision ("shadcn/ui fights retro aesthetic"); `frontend2/components.json` absent by design |

## Spacing Scale

Declared values (must be multiples of 4). Sourced verbatim from existing `frontend2/src/styles/globals.css` `@theme` tokens — Phase 64 adds NO new tokens.

| Token | Value | Usage in Phase 64 |
|-------|-------|-------------------|
| xs | 4px | Icon-to-text gaps inside RetroButton; gap between history row timestamp and code |
| sm | 8px | Compact stacking inside the result banner; RetroPanel close-button inset |
| md | 16px | Default padding inside every RetroPanel; gap between tab strip and active-tab body; manual input label-to-input |
| lg | 24px | Outer ScanPage container padding; vertical gap between viewfinder and tab strip |
| xl | 32px | Gap between the viewfinder viewport and result banner when both visible on mobile |
| 2xl | 48px | Major section breaks on history tab (used between list header and first entry) |
| 3xl | 64px | Not used in Phase 64 |

**Exceptions (with justification):**

- **44px minimum touch target** for `RetroButton`, the torch toggle, history-entry tap targets, and history-entry delete affordances. Matches existing `RetroButton` `h-[44px]` baseline + WCAG 2.5.5 AA (v1.3 precedent). Not an 8-point multiple but universally required for mobile.
- **Viewfinder aspect ratio container**: `aspect-square` on mobile, `aspect-video` on ≥768px. Height derives from width; no magic px value.
- **Reticle bracket inset**: 16px from the viewfinder inner edge (`spacing-md`, clean match).
- **Result banner inline code/format typography**: monospace tracking inherited; no custom letter-spacing token.

## Typography

Four roles declared; two weights total (regular 400 + bold 700). No italics. All scanner UI uses monospace by default; only long-form error instruction copy may use `font-sans` if readability requires it.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Error-panel instruction copy, history timestamps, manual-input helper text |
| Label | 14px | 700 (bold) | 1.2 | All RetroButton labels, tab labels, section labels ("SCAN HISTORY", "MANUAL ENTRY") — always UPPERCASE |
| Heading | 20px | 700 (bold) | 1.2 | RetroPanel `title` (e.g. "SCAN", "CAMERA ACCESS DENIED") — always UPPERCASE |
| Display | 24px | 700 (bold) | 1.2 | Decoded code rendered inside result banner (monospace, tracked, emphatic) |

**Font-family contract:**

- Scanner UI primary: `font-mono` (IBM Plex Mono stack)
- Decoded code in banner + history entries: `font-mono` (aids scan-ability)
- Per-platform permission-denied body copy: `font-sans` permitted only if a sentence exceeds 120 chars (prose readability); otherwise keep `font-mono`

**Casing contract:**

- Headings, labels, button text: UPPERCASE (retro convention, enforced by existing atoms)
- Decoded codes: preserve original case exactly (alphanumeric QR/Code128 codes are case-sensitive)
- Error body copy: Sentence case (readability) — this is the one place where UPPERCASE is wrong

## Color

Retro 60 / 30 / 10 split sourced from `globals.css` `@theme` tokens. Phase 64 adds NO new color tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F0E1` (`--color-retro-cream`) | Page background, RetroPanel body, active-tab surface, manual-input background |
| Secondary (30%) | `#1A1A1A` (`--color-retro-ink`) | All 3px `border-retro-thick` strokes, corner reticle brackets in viewfinder, typography ink, active-tab text; also the viewfinder matte (the area around the camera frame) so the live preview floats on ink-dark surround |
| Accent (10%) | `#D4A017` (`--color-retro-amber`) | Primary CTA fill (SCAN AGAIN, RETRY, LOOK UP CODE); active scanline sweeping the viewfinder; focus-visible outline; torch-ON state |
| Destructive | `#CC3333` (`--color-retro-red`) | CLEAR HISTORY button, RetroConfirmDialog danger framing, error-panel HazardStripe accent variant (lib-init-fail uses red; permission-denied uses amber) |

**Accent reserved for** (explicit — never "all interactive elements"):

- Primary CTA button fill (SCAN AGAIN, RETRY, LOOK UP CODE) — one per panel max
- The single animated scanline inside the viewfinder (D-13)
- Focus-visible outline (`outline-retro-amber`, inherited from RetroButton)
- Torch toggle fill when torch is ON (off state is neutral cream)
- Result-banner format badge (e.g. "QR_CODE", "EAN_13") — small pill, amber-on-ink

**Not accent:**

- Tab strip background — inactive tabs are `retro-gray` (`#8B8B8B`), active tab is `retro-cream`; amber never appears in the tab strip
- History list rows — rows are cream with ink text; tapping fires the banner, not a color change
- RetroButton `neutral` variant — cream, hover becomes amber (this is the only legal amber hover in the phase)

**Semantic color for errors:**

- `library-init-fail` panel: `retro-red` HazardStripe header (transient infra failure, user should notice)
- `permission-denied`, `no-camera`, `unsupported-browser`: `retro-hazard-yellow` / ink hazard stripe (environmental, not a crash — instruction + fallback)
- `retro-orange` (`#E67E22`) is reserved for future "warning" use; NOT used in Phase 64

**Contrast gates (WCAG AA, 4.5:1 minimum for body text):**

- Ink on cream: 16.1:1 ✓
- Amber on ink: 8.3:1 ✓ (button label inside amber button is ink, not white)
- White on red: 5.5:1 ✓ (CLEAR HISTORY)
- Cream on gray (disabled button): 3.1:1 — acceptable because disabled labels are not required to meet 4.5:1

## Interaction Contract

> Explicit post-scan + error-recovery behavior spec. Checker and executor both consume this. Derived from CONTEXT.md D-01..D-16.

### Page boot sequence (Scan tab default, D-05 / D-07)

1. User navigates to `/scan` — ScanPage mounts.
2. Default active tab = `SCAN`. Scanner mounts with `paused=false` immediately.
3. Browser camera-permission prompt fires (OS-owned UI; not styled by us).
4. First `pointerdown` anywhere in the `RetroTabs` / page wrapper region resumes the singleton `AudioContext` (D-08). This also satisfies iOS first-gesture rule (Pitfall #19).
5. Scanline animation begins sweeping top-to-bottom over the viewfinder at ~2s linear (D-13). Under `prefers-reduced-motion`, scanline is rendered static in the middle of the viewfinder — corners remain unchanged.

### Post-decode flow (D-02 / D-03)

1. Library fires decode callback with `{ rawValue, format }`.
2. Scanner **pauses** (library's `paused` prop toggled true; `MediaStreamTrack` NOT stopped — stays mounted).
3. `useScanFeedback.trigger()` fires simultaneously: 880 Hz oscillator beep (~100 ms), haptic pulse (iOS `ios-haptics`, Android `navigator.vibrate([60])`), visual flash (200 ms amber overlay fade-out on the viewfinder).
4. Visual flash animates over 200 ms using CSS opacity; under `prefers-reduced-motion` the flash is replaced with a 400 ms amber solid hold then snap-off (no fade).
5. `useScanHistory.add({ code, format, entityType: 'unknown', timestamp })` writes to localStorage. Dedupe: if `code` already present, the existing entry moves to position 0 and its `timestamp` updates; no new entry.
6. `ScanResultBanner` renders below the viewfinder (not over it — the viewfinder remains visible in paused state so the user gets spatial continuity).
7. Banner layout: `SCANNED` heading + `CODE: {monospace value, 24px display}` + format pill + timestamp · `SCAN AGAIN` primary button bottom-right.
8. Tapping `SCAN AGAIN` → banner unmounts, scanner `paused=false`, scanline restarts.

### History tap (D-15)

1. User taps a history row (full-row tap target, ≥44px height).
2. Same code path as a live decode: `useScanHistory.add(existing)` → dedupe-to-top, pause scanner (if currently on Scan tab; if on History tab, no scanner to pause — the banner still renders at top of page wrapper).
3. Banner renders identically to live-decode banner. `SCAN AGAIN` button behavior is identical.
4. Phase 65 replaces the banner with QuickActionMenu; no rewire needed at callsite.

### Manual tab submit (D-14)

1. User types in `RetroInput`; `onChange` trims and truncates to 256 chars max (`maxLength={256}` attribute + runtime guard).
2. `LOOK UP CODE` button is disabled when trimmed value is empty.
3. On submit: fire the same post-scan flow with synthetic format `'MANUAL'`; write to history; render banner.
4. Format pill in the banner reads `MANUAL` for this path.

### Error-panel flow

- `permission-denied`: panel replaces the viewfinder area (not the tab strip; tabs stay usable). `USE MANUAL ENTRY` sets active tab to `MANUAL`. No auto-switch — the user must tap.
- `no-camera`: identical layout; secondary `RELOAD PAGE` present.
- `library-init-fail`: panel replaces viewfinder area. `RETRY` re-runs the dynamic `import("@yudiel/react-qr-scanner")`; on success, panel unmounts and Scanner remounts. On repeated failure (2nd retry within 30s), panel stays visible — the Retry button remains enabled, user can keep trying.
- `unsupported-browser`: panel replaces viewfinder area; only `USE MANUAL ENTRY` action.

All four panels log a structured `console.error({ kind, errorName, userAgent, timestamp })` on first render (D-12). No backend telemetry.

### Torch toggle (D-16)

- Rendered ONLY when `MediaStreamTrack.getCapabilities().torch === true` on the active stream.
- Absent (not rendered, no placeholder, no tooltip) when capability missing — matches SCAN-04 wording.
- Positioned as a floating retro button overlaid on the viewfinder, bottom-right of the viewfinder viewport, 16px inset from corner.
- OFF state: neutral cream fill, ink border, ASCII glyph `[◉]` + label `TORCH OFF`.
- ON state: amber fill, ink border, ASCII glyph `[◉]` + label `TORCH ON`.
- Tapping applies track constraint `{ advanced: [{ torch: !current }] }` and flips state on success; silently reverts on rejection.

### Reduced-motion / accessibility

- `prefers-reduced-motion: reduce`: scanline is static (rendered at viewfinder vertical center, no animation); post-decode visual flash is simplified (solid amber hold, snap-off); RetroTabs transitions use instant toggle (already CSS-instant; no change needed).
- All interactive elements retain `focus-visible:outline-retro-amber` 2px / 2px offset (inherited from retro atoms).
- Tab strip is keyboard-navigable (← / → to move, Enter to activate) — inherited from `RetroTabs`; no change needed in Phase 64.
- Manual input is keyboard-first; no placeholder-only labeling (label + placeholder both present).
- Torch toggle has `aria-pressed={torchOn}` and an accessible name from its visible label.

## Viewfinder Visual Spec (D-13 — corner reticle + amber scanline)

Matches the ASCII preview accepted during discussion:

```
┌──   ──┐
│       │
 ──═════──   <- amber scanline (animated top→bottom, ~2s linear)
│       │
└──   ──┘
```

**Dimensions + composition:**

- Outer container: full viewport width up to 480px (matches existing ScanPage `max-w-[480px]`), aspect-square on mobile, aspect-video ≥768px.
- Inner viewport: 3px ink border (`border-retro-thick border-retro-ink`) framing the live `<video>` stream.
- Matte (area between border and video edge if aspect-ratio mismatch): ink-dark (`bg-retro-ink`) so the live preview floats on an ink field.
- Corner brackets: four SVG or CSS-pseudo L-shapes, each leg 24px long × 3px thick, ink color, inset 16px from the inner viewport corners.
- Scanline: 2px tall, 100% inner width, amber fill (`bg-retro-amber`), 60% opacity, box-shadow `0 0 8px 0 #D4A017` for soft glow. Positioned absolutely; `@keyframes scan-sweep` translates Y from `0%` to `100%` over `2s`, `linear`, `infinite`.
- Reduced motion: scanline stays at `50%` vertical position, opacity 0.4, no animation.

**Animation token (add to `globals.css` only if planner deems necessary — preferred is keyframe + utility class in the scan component stylesheet):**

```css
@keyframes scan-sweep {
  from { transform: translateY(0%); }
  to { transform: translateY(calc(100% - 2px)); }
}
```

Scanline is the ONLY animated element in Phase 64 (post-decode flash excepted). No CRT overlay, no pulsing reticle, no corner flicker.
