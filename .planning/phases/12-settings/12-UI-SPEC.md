## Color

Inherits the locked retro-os palette. 60/30/10 maps onto the existing token system.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg-panel` white `#ffffff` over cream desktop `#fdf6ec` | Window bodies, the desktop behind |
| Secondary (30%) | `--bg-panel-2` `#f7f0e2` | Recessed strips (grouped-row hover, footer action strips, table headers) |
| Accent (10%) | `--titlebar-blue` (default window chrome + primary `BevelButton` + selected row) | Reserved for: default Window titlebars, the ONE primary submit per form, focus outlines, the selected/current row fill |
| Destructive | `--danger #b73348` on `--danger-bg #fbe3e8` | Delete account, revoke, remove member, unlink, separator-conflict error band |

**Accent reserved for:** default window titlebars; the single primary submit button per form
(`BevelButton variant="primary"`); focus outline (`outline-titlebar-blue`); the
selected/current table row. Never on plain rows, never on every interactive element. Semantic
titlebars: **pink** = danger windows (Danger Zone, Remove Member confirm context),
**butter** = warning notes (the OAuth "no password yet" note pattern), **mint** = positive
confirmation only. The Appearance/Language/Formats/Notifications windows use the default
**blue** titlebar.

**3-non-color-cue discipline (WCAG, no color-only signalling):**
every status/destructive cue carries TWO non-color reinforcements in addition to hue:
1. **Text label** вҖ” e.g. "THIS DEVICE", "LINKED", "Light (current)", "Owner".
2. **Shape/glyph** вҖ” square check `вң“` (on), badge border, `вң•` on errors, chevron `вҖә` on
   rows, pinstriped pink titlebar on danger windows.
3. **Position/structure** вҖ” danger actions live in a recessed footer strip or a pink
   `Danger Zone` Window; the selected theme sits first with a `RetroBadge`.
