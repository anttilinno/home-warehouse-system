- OAuth initiate: `GET /auth/oauth/{provider}` — BROWSER NAVIGATION (window.location.href), NOT fetch. provider in {google, github}. Through the same-origin `/api` proxy (`/api/auth/oauth/{provider}`) per RESEARCH RESOLVED Q3 (proxy passes the 302).
- OAuth callback (backend -> browser): backend redirects to `${AppURL}/auth/callback?code=...` OR `?error=<code>`.
- OAuth exchange: `POST /auth/oauth/exchange { code }` -> `{ access_token, refresh_token }` + cookies (one-time Redis code — SINGLE USE).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> OAuth provider -> backend callback | the provider redirect chain crosses; PKCE/state live server-side (HttpOnly cookie) |
| browser -> ingress -> Authelia | Authelia forward-auth runs at the BARE path; the SPA must not proxy it through /api |
| browser <- backend redirect (?code) | the one-time code crosses the URL once; single-use Redis store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-14 | Spoofing/CSRF | OAuth state/PKCE | mitigate (backend) | frontend does NO PKCE — verifier+state are server-side HttpOnly cookies; client only initiates + exchanges (Don't Hand-Roll) |
| T-05-15 | Information disclosure | one-time code in localStorage/logs | mitigate | code read from URL, passed straight to exchange, never stored or logged; no console.log of code; URL replaced after exchange (navigate replace) |
| T-05-16 | Elevation | Authelia routed through /api (trust bypass) | mitigate | Authelia button hard-codes the BARE `/auth/authelia/login`; grep gate asserts NOT /api-prefixed (8e13faf) — ingress, not SPA, owns Authelia trust |
| T-05-17 | Tampering | double-exchange of single-use code under StrictMode | mitigate | useRef latch fires exchange once; test asserts single call under double-invoke (Pitfall 5) |
| T-05-18 | Tampering | npm installs (brand-icon pack) | mitigate | NO icon packs — provider tiles are Silkscreen initials (UI-SPEC); NO new packages; Package Legitimacy Audit N/A |
</threat_model>
