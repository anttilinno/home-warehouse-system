## Tech Debt by Phase

### Phase 52 — Dashboard (4 items)

1. **Sidebar missing ITEMS and LOANS nav entries** — Phase 52's Sidebar extension (4 items) was developed in a worktree that was never merged into the branch that Phase 51 AppShell consumed. Current Sidebar.tsx has 2 NavLinks only. Users navigate Items/Loans via QuickActionCards only. Fix: add two NavLink entries to `frontend2/src/components/layout/Sidebar.tsx`.

2. **Spurious logout on transient network error** — `AuthContext.tsx` `catch` block calls `setRefreshToken(null)` on any error including temporary connectivity loss, not just 401/session-expired. Low probability in practice.

3. **entity_name type mismatch** — `lib/types.ts` types `entity_name` as `string | undefined` but backend returns JSON `null`. Truthiness check `if (entry.entity_name)` handles this correctly at runtime; type is technically wrong.

4. **NotFoundPage untranslated strings** — 3 hardcoded English strings in `routes/index.tsx` NotFoundPage component. Inconsistent with rest of codebase's i18n pattern.

### Phase 53 — Settings Hub (3 items)

5. **DataPage null-guard missing on workspaceId** — export URL becomes `/api/workspaces/null/export/workspace` if workspace disappears mid-session. Add `disabled={exporting || !workspaceId}` to Export button.

6. **SET-08 scope deviation** — Cache management and manual sync trigger deferred to v2.1+ (intentional per D-11). DataPage export/import satisfies the implemented scope; original cache management scope is not addressed.

7. **SecurityPage test act() warnings** — `act()` warnings in tests for async `useEffect` fetches. Tests pass; not a code defect.

### Phase 50 — Design System (2 items)

8. **AuthCallbackPage inline hazard stripe** — `AuthCallbackPage.tsx` uses inline `bg-hazard-stripe` div instead of `<HazardStripe>` component. Out-of-scope for Phase 50 refactoring; minor inconsistency.

9. **useToast import path inconsistency** — some settings pages import from barrel (`@/components/retro`), others from direct path (`@/components/retro/RetroToast`). Both resolve correctly; style-only.

### Phase 46 — Photo Sync Pipeline (1 item)

10. **Pre-existing test failures** — 2 failures in `lib/hooks/__tests__/use-offline-mutation.test.ts` (assert `queueMutation` without `workspaceId`; hook now always includes it). Pre-dated this phase; tracked in phase documentation.

### Requirements File (documentation, no code)

11. **Design System section lists v1.9 SYNC requirements** — The "Design System" section in `v1.9-REQUIREMENTS.md` shows SYNC-01 through SYNC-04 instead of DS-01 through DS-10. DS requirements covered only in ROADMAP.md and phase VERIFICATIONs.

12. **Dashboard section lists v1.9 COMP requirements** — The "Dashboard" section shows COMP-01 through COMP-04 instead of DASH-01 through DASH-03. DASH requirements covered only in ROADMAP.md and phase VERIFICATIONs.

13. **v2.0 requirement checkboxes all unchecked** — All SCAF, AUTH, SET, LAY checkboxes remain `[ ]` despite verified satisfaction. Traceability table has no entries for any v2.0 REQ-IDs.
